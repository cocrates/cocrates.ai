# EP13. Menggunakan Cocrates Harness dengan Baik

![Evolusi Timbal Balik](/img/13-evolution-of-cocrates.png)

---

Di Ep1 kita bertanya: *"Berapa banyak rekan tim AI yang Anda miliki?"*

Di Ep2 kita menetapkan prinsip: *"Output yang tidak diperiksa tidak layak dihasilkan."*

Di Ep3 kita menyiapkan alat—menginstal Cocrates Harness.

Di Ep4–6 kita mempraktikkan—belajar, generasi, pembuatan skill.

Di Ep7–10 kita memahami internal—Agent, Skill, routing, prinsip arsitektur.

Di Ep11–12 kita mempelajari skill inti dan meta.

**Kita sekarang tahu seluruh Cocrates.**

Lalu apa yang harus kita lakukan? Hanya menggunakannya?

Tidak. Bagian sebenarnya dimulai sekarang.

---

## 💭 "Apakah Ini Akhirnya?"

Jujur, saya bertanya pada diri sendiri:

*"Saya sudah menginstal Cocrates, belajar cara menggunakannya, memahami internalnya. Sekarang apa—hanya menggunakannya keras-keras?"*

Tapi nilai sebenarnya Cocrates bukan dalam **menggunakan**nya—melainkan dalam **mengevolusikannya**.

Cocrates bukan produk jadi. Ini **framework yang tumbuh**. Dan ia tumbuh bersama pengguna.

---

## 🧑‍🎓 Pengguna Harus Berevolusi

Kondisi pertama menggunakan Cocrates dengan baik: **Anda berevolusi**.

**Internalisasi pertanyaan Sokratik.**

Awalnya Cocrates bertanya: *"Apa tujuan desain ini?"* *"Mengapa pendekatan ini?"* Anda menjawab dan belajar berpikir.

Akhirnya Anda bertanya pada diri sendiri—tanpa Cocrates—*"Apa alasan saya untuk keputusan ini?"*

**Biasakan tinjauan (U→A→E→A).**

Pahami, analisis, evaluasi, setujui—dari Ep2. Awalnya sadar; akhirnya harus jadi kebiasaan.

Ketika Anda mendapat output AI, bukan *"kelihatannya baik"* tetapi U→A→E→A yang alami.

**Biasakan persetujuan eksplisit.**

Tinjauan tanpa persetujuan sama dengan tidak ada tinjauan. Bukan *"Saya baca sekilas—mungkin oke"* tetapi *"Saya sudah memeriksa. Kita pakai ini. Saya bertanggung jawab."*

Ketiga itu adalah inti evolusi pengguna.

---

## 🤖 Cocrates Harness Juga Harus Berevolusi

Bukan hanya pengguna yang harus tumbuh—**Cocrates Harness juga harus berevolusi**.

**Tambah, edit, perluas skill.**

Mulai dengan skill bawaan; seiring waktu Anda butuh milik sendiri. Gunakan generating-skill-creation dari Ep12—bangun, edit, sempurnakan skill.

Ketika pola kerja berubah, skill berubah. Skill adalah aset hidup, bukan beku.

**Perbaiki prompt agen.**

Prompt Cocrates Agent juga target evolusi. Prompt awal generik; pengalaman bisa mempertajamnya.

*"Jenis permintaan ini sering muncul—bagaimana saya ubah prompt agar Agent menanganinya lebih baik?"*

Anda harus bisa bertanya itu pada diri sendiri.

---

## 🔄 Evolusi Timbal Balik — Loop Umpan Balik

Evolusi pengguna dan evolusi Cocrates bukan terpisah—mereka membentuk **satu loop umpan balik**.

**Pertumbuhan pengguna → permintaan lebih baik → respons Cocrates lebih baik → wawasan lebih dalam → pertumbuhan pengguna**

Ulangi itu dan keduanya menguat.

Putus loop? Pengguna berhenti tumbuh → Cocrates stagnan. Tidak perbarui Cocrates → pengguna menemui batas.

**Itulah makna sebenarnya pertanyaan Ep1.** *"Berapa banyak rekan tim AI yang Anda miliki?"* bukan hanya produktivitas—melainkan seberapa baik Anda memimpin rekan tim AI dan tumbuh bersama.

---

## 🌍 Filosofi Open Source — Kode Menunggu Evolusi Anda

Cocrates adalah open source. Semua kode dan prompt publik.

[**github.com/cocrates/cocrates.ai**](https://github.com/cocrates/cocrates.ai)

Anda tidak harus tetap konsumen. Anda bisa berkontribusi—bagikan skill, perbaiki bug, usulkan prompt yang lebih baik.

Dan Cocrates tidak terkunci pada satu IDE atau plugin. Claude Code, Cursor, Codex, opencode—Anda bisa mengimplementasikan prinsip dan skill Cocrates di mana saja. Cocrates adalah metodologi, bukan hanya alat.

---

## 💡 Wawasan Hari Ini: The Unexamined Code Berlaku untuk Framework Juga

Ingat prinsip inti seri?

> *"The unexamined code is not worth generating."*

Ini tidak hanya berlaku untuk kode—melainkan untuk **Cocrates Harness itu sendiri**.

Cocrates harus terus ditinjau, dianalisis, dievaluasi, diperbaiki. *"Ini Cocrates—pasti baik"* adalah kembali ke kepercayaan AI-assisted membabi buta.

Menggunakan Cocrates saja tidak cukup. **Anda harus mengevolusikan Cocrates.**

---

## 📌 Poin Penting

1. **Pengguna harus berevolusi**—internalisasi pertanyaan Sokratik, biasakan U→A→E→A, persetujuan eksplisit.
2. **Cocrates Harness harus berevolusi**—skill dan prompt agen.
3. **Evolusi timbal balik adalah loop umpan balik**—mesin pertumbuhan berkelanjutan.
4. **Cocrates adalah open source**—jadi kontributor, bukan hanya konsumen.
5. **Prinsip berlaku untuk framework juga**—unexamined code mencakup kode Cocrates sendiri.

---

- Apakah saya sudah membaca kode sumber Cocrates?
