# EP4. Pembelajaran Sokratik dalam Praktik

## 💡 Saya Bertanya "Ceritakan tentang Taksonomi Bloom"—Inilah yang Terjadi

"AI, jelaskan Taksonomi Bloom dengan sederhana."

Pertanyaan biasa yang banyak dari kita ajukan. ChatGPT atau Claude biasa mungkin membuang esai panjang: *"Taksonomi Bloom adalah kerangka kerja yang diusulkan Benjamin Bloom pada 1956..."* Fokus memudar dalam tiga detik.

**Cocrates** menjawab berbeda. Satu atau dua kalimat esensi—lalu **misi** tajam tiba-tiba.

> 🦉 **Cocrates:** "Taksonomi Bloom terkenal dengan piramida enam tingkatnya. Bayangkan seorang guru merancang kursus yang berjalan jujur dari bawah (Remember) ke atas (Create). **Apa yang sangat salah dengan pendekatan itu?**"
> 
> 

Bukan curahan informasi satu arah—pertanyaan kembali pada Anda. Bagaimana Anda akan menjawab? Berhenti sejenak dan pikirkan.

---

## 🗺️ Pipeline Pembelajaran Tiga Langkah Cocrates

Di episode ini kita akan menelusuri pipeline **Learning** Cocrates menggunakan percakapan nyata itu.

1. **Education:** Alih-alih mesin penjawab—contoh cacat dan misi yang membuat Anda menemukan.
2. **Knowledge Capture:** Jangan menyalin dinding teks—catat ke basis pengetahuan (KB) yang berpusat pada **celah** dalam pemahaman Anda.
3. **Reflection:** Beralih ke mode pewawancara dan verifikasi apakah Anda benar-benar tahu.

---

## 🎓 Langkah 1. Education — Tiga Wawasan yang Mengubah Pikiran

Saat saya bergulat dengan pertanyaan Cocrates, asumsi yang saya anggap remeh berbalik dengan menyenangkan.

### 💡 Wawasan 1: Piramida bukan 'urutan pelajaran'

Kita sering membaca piramida sebagai *kuasai dasar sebelum puncak*. Bloom berbicara tentang **struktur ketergantungan**, bukan urutan sederhana.

Dalam praktik, kebalikannya sering lebih baik.

> 💻 Tugas: *"Bangun situs web pemrograman Anda sendiri hari ini! (Create)."* Siswa menemui dinding, googling sintaks (Remember), dan bergulat dengan mengapa kode berperilaku begini (Understand).
> 
> 

Strategi **Pull**: tugas tingkat tinggi (Create) **menarik** pengetahuan tingkat rendah (Remember, Understand). Mitos bahwa *dasar harus sempurna sebelum lanjut* runtuh di sini.

### 💡 Wawasan 2: Bukan piramida 1D—matriks 2D

Cocrates menjatuhkan twist lain: revisi 2001 adalah **matriks dua dimensi**, bukan tangga tunggal.

|  | Remember | Understand | Apply | Analyze | Evaluate | Create |
| --- | --- | --- | --- | --- | --- | --- |
| **Pengetahuan faktual** |  |  |  |  |  |  |
| **Pengetahuan konseptual** |  |  |  |  |  |  |
| **Pengetahuan prosedural** |  |  |  |  |  |  |
| **Pengetahuan metakognitif** |  |  |  |  |  |  |

Belajar `map` JavaScript terbagi di berbagai dimensi:

* **Faktual:** Hafalkan sintaks `map`.
* **Konseptual:** Pahami bagaimana array sumber dan callback berhubungan.
* **Prosedural:** Ketik `map` dalam proyek nyata.
* **Metakognitif:** *"Saya sering salah binding `this` saat menggunakan `map`"*—perhatikan kebiasaan sendiri.

Banyak dari kita hanya tahu piramida 1D. Percakapan mengupas satu lapisan ketidaktahuan.

### 💡 Wawasan 3: Push dan Pull—tarian yang disesuaikan dengan pelajar

Klimaksnya adalah menyelaraskan **Push** dan **Pull**.

* **Push (injeksi berurutan):** Tahap rendah → tinggi langkah demi langkah (baik untuk pemula, beban kognitif berlebih).
* **Pull (berbasis misi):** Lempar tugas tingkat tinggi; pelajari tahap rendah sesuai kebutuhan (pelajar dengan dasar, motivasi kuat).

Cocrates default ke **Pull**—misi cacat yang membuat Anda berpikir. Ketika Anda mengalami kolaps kognitif, ia dengan baik beralih ke **Push** dan membangun konsep.

Untuk puncak Education, Cocrates menugaskan **Create**: *"Rancang kurikulum branching Git 4 jam untuk developer baru."* Melalui itu, saya memiliki ide-ide tersebut.

## 💾 Langkah 2. Knowledge Capture — Mencatat Celah

Ketika saya minta merangkum yang sudah dipelajari, Cocrates membuat `kb/bloom-taxonomy.md`.

Bagian yang mencolok: bukan ringkasan buku teks. Ia berpusat pada **apa yang saya salah asumsikan (Gap)**.

> **📝 Asumsi salah / Celah**
>
> * Saya pikir Bloom adalah piramida 1D sederhana → model sebenarnya adalah matriks 2D.
> * Saya pikir pengajaran harus bottom-up → Pull sering lebih dinamis dan efektif.
> 

Ringkasan membosankan nanti. Catatan kesalahpahaman masa lalu memulihkan *aha* asli dengan intensitas 200%.

---

## 🧠 Langkah 3. Reflection — "Yakin Anda Tahu?"

Ketika saya minta memverifikasi pemahaman, Cocrates menjadi **pewawancara** yang keras.

Pertanyaan tidak berkompromi. Strategi Pull menggali dalam.

> 🦉 **Cocrates:** "Bagus. Petakan setiap sesi kurikulum Git Anda ke sel dalam matriks 2D."
> 
> 

Saya membeku. Menjelaskan konsep (Understand) vs. menempatkannya tepat di matriks (Analyze, Apply)—celahnya jelas.

Setelah wawancara, tanpa pujian kosong—rapor yang jelas:

* ✅ **Kokoh:** Apa arti piramida sebenarnya; Push vs. Pull
* ⚠️ **Goyah:** Menerapkan matriks 2D sempurna dalam desain instruksional nyata

Menemukan batas ⚠️ itu adalah nilai Reflection. **Anda tumbuh ketika tahu apa yang tidak Anda ketahui.**

---

## 📌 Poin Penting

1. **Education:** Cocrates bukan mesin jawaban—ia menggunakan misi dan Pull untuk memprovokasi pemikiran.
2. **Knowledge Capture:** Lebih kuat dari ringkasan—**catat asumsi salah (Gaps)** dan lacak ketidaktahuan.
3. **Reflection:** Pertanyaan menantang menggambar garis antara apa yang Anda tahu dan apa yang hanya Anda kira tahu.
