# EP2. 未经审查的代码不值得生成

![Socrates vs Cocrates 哲学](/img/02-unexamined-code.png)

---

在 EP1 中我们提出了一个重要问题：

*"AI辅助型工程师 与 AI原生型工程师 有何不同？"*

AI辅助型 是把 AI 当工具；AI原生型 是把 AI 当队友、由你带领。还记得吗？

但这又引出一个问题：

*"那么 AI原生型工程师 的核心技能究竟是什么？"*

今天我们从一句宣告开始回答。

---

## 🏛️ 宣告——贯穿全系列的一句话

EP1 问题的答案是：

**审查。**

我们用一句话宣告——贯穿本系列的唯一原则：

> **"The unexamined code is not worth generating."**
> **"未经审查的产出，不值得创造。"**

从 AI 拿到东西，心想*"看起来没问题"*就继续——你正落入 AI辅助型 的陷阱。

AI原生型工程师 会审查。理解、分析、评估、批准。未经审查的代码，等于从未生成。

---

## 🏛️ 苏格拉底的启发

你知道这句宣告从何而来吗？

苏格拉底说：
**"The unexamined life is not worth living."**
*"未经审视的人生不值得过。"*

不断追问、承认无知、走向真理——这是苏格拉底关于如何好好生活的看法。

Cocrates 把这一哲学带入开发。

> **"The unexamined code is not worth generating."**
> *"我不理解、未经审查的 AI 产出没有价值。"*

这不是文字游戏。苏格拉底追问如何生活，Cocrates 追问 AI 时代工程师应如何行动。

*"你现在是否不经思考就使用未经审查的产出？"*

---

## 💻 "代码"的扩展含义——你的交付物也是 Code

有一点需要澄清：

*"等等——我不是开发者……代码？我该审查什么代码？"*

不。**Code** 在 Cocrates 中的含义要广泛得多。

- 💻 **源代码** — 显然包括。
- 📝 **报告与文档** — AI 撰写的业务报告、计划、提案
- 📊 **演示与幻灯片** — AI 为你制作的 deck
- 📚 **博客系列** — 本文也是 Code
- 🧠 **学习笔记与摘要** — 你让 AI 总结的任何内容

**你让 AI "为你制作" 的一切。** 这就是 Cocrates 所称的 Code。

---

## 🏗️ 架构很重要——没有"先做了再说"

那么如何实际做到？

要理解 Cocrates 的做法，看**本博客系列**本身——它正以 Cocrates 方式构建。

**步骤 1.** 设计大局。在 `overview.md` 中构建整个系列结构，然后**审查并批准**。

**步骤 2.** 构建每集骨架。在 `outline.md` 中逐集列大纲，再次**审查并批准**。

**步骤 3.** 细化设计。在 `episodes.md` 中充实每集，再次**审查并批准**。

**步骤 4.** 生成正文，最后**审查并批准**。

看到了吗？每一步都有**审查（🔍）与批准（✅）**。Cocrates 从不允许*"先随便做点什么看看"*。

> **"花一小时审查结构，省下一周的 bug 修复。"**

这就是 Cocrates 之道。

---

## 🔬 审查的真正含义——U → A → E → A

老实说——大多数人认为"审查"是什么意思？*"扫一眼，检查错别字？"* 对吧？

Cocrates 说：那不是审查。

真正的审查有四个阶段。

### 第一，U — Understand（理解）

当你能用自己的话解释时，才算理解。AI 为何这样构建、结构如何衔接——*"哦，原来是这样设计的"* 那一刻。

### 第二，A — Analyze（分析）

*"为何这样实现？"* 设计意图是什么？有没有更好的做法？看结构，而不只是表面。

### 第三，E — Evaluate（评估）

*"这真的是最佳选择吗？"* 对照项目目标与你的标准进行判断。

### 第四，A — Approve（批准）

*"我已检查。我们采用这个。我负责。"* 只有能这样说，审查才算完成。

**U → A → E → A。**

这就是 Cocrates 对审查的定义。

---

## 🎯 Harnessing Ignorance（驾驭无知）

还有一个问题：

*"为何要这么认真审查？我只是在找错别字吗？"*

不。审查的终极目的另有其物。

**Harnessing Ignorance（驾驭无知）。**

审查过程中，你 inevitably 会遇到：

- *"等等——这部分为何是这样？"*
- *"我对这个的理解错了吗？"*
- *"哦——原来还有这种思路。"*

那一刻就是**无知被消除**的时刻。

学习你不知道的。质疑你视为理所当然的。这才是审查的真正目的。

Cocrates 的原则很简单：
> **"Never leave ignorance unattended."（绝不让无知 unattended。）**

不知道就问。不理解就追问。审查不只是修错——它是**你的成长过程**。

## 📌 要点回顾

1. **原则宣告。** *"未经审查的产出不值得生成。"* — AI原生型工程师 的第一原则。

2. **Code 的扩展。** Code 不只是源码。AI 制作的一切——报告、幻灯片、博客、学习笔记。全是 Code。

3. **审查四阶段。** U → A → E → A。理解、分析、评估、批准——不是查错别字。

4. **驾驭无知。** 审查的终极目的是发现你不知道的，并在成长中填补缺口。

- 你能批准吗？（Approve）

试一下，再继续。
