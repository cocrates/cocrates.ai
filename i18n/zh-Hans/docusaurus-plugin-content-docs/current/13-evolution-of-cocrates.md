# EP13. 用好 Cocrates Harness

![相互演进](/img/13-evolution-of-cocrates.png)

---

在 EP1 中我们问：*"How many AI teammates do you have?"*

在 EP2 我们设定原则：*"Unexamined output isn't worth generating."*

在 EP3 我们准备工具——安装了 Cocrates Harness。

在 EP4–6 我们实践——学习、生成、skill 创建。

在 EP7–10 我们理解内部——Agent、Skill、路由、架构原则。

在 EP11–12 我们学了核心与 meta-skills。

**我们现在了解 Cocrates 的全部。**

那么该做什么？只用就行吗？

不。真正的部分现在开始。

---

## 💭 "这就结束了吗？"

老实说，我问自己：

*"我安装了 Cocrates，学会了怎么用，理解了内部。现在呢——就使劲用就行？"*

但 Cocrates 的真正价值不在**使用**——在**演进**。

Cocrates 不是成品。它是**成长中的框架**。它与用户一起成长。

---

## 🧑‍🎓 用户必须演进

用好 Cocrates 的第一条件：**你要演进**。

**内化苏格拉底式提问。**

起初 Cocrates 问：*"这个设计的目的是什么？"* *"为何选这种方案？"* 你回答并学会思考。

Eventually 你问自己——没有 Cocrates——*"我做这个决定的依据是什么？"*

**习惯审查（U→A→E→A）。**

理解、分析、评估、批准——从 EP2 起。起初是有意识的；最终应成为习惯。

拿到 AI 产出，不是*"看起来没问题"*而是自然的 U→A→E→A。

**习惯明确批准。**

没有批准的审查等于没有审查。不是*"我扫了一眼——大概行"*而是*"我已检查。我们采用这个。我负责。"*

这三条是用户演进的核心。

---

## 🤖 Cocrates Harness 也必须演进

不只用户要成长——**Cocrates Harness 也必须演进**。

**添加、编辑、扩展 skills。**

从 bundled skills 开始；随着时间你需要自己的。用 EP12 的 generating-skill-creation——构建、编辑、完善 skills。

工作模式变化，skills 也要变化。Skills 是 living assets，不是 frozen。

**改进 agent prompts。**

Cocrates Agent prompts 也是演进目标。早期 prompts 较 generic；经验可以 sharpen。

*"This request type comes up often—how should I change the prompt so Agent handles it better?"*

你应该能问自己这个。

---

## 🔄 相互演进 — 反馈循环

用户演进与 Cocrates 演进不是分开的——它们形成**一个反馈循环**。

**用户成长 → 更好的请求 → 更好的 Cocrates 回应 → 更深的洞察 → 用户成长**

重复，两者 strengthen。

Break the loop？User stops growing → Cocrates stagnates。Don't update Cocrates → user hits limits。

**That's the real meaning of EP1's question.** *"How many AI teammates do you have?"* 不只是 productivity——是你如何 well lead AI teammates 并一起成长。

---

## 🌍 开源哲学 — 等待你演进的代码

Cocrates 是 open source。全部 code 与 prompts 公开。

**github.com/cocrates/cocrates.ai**

你不必 stay consumer。你可以 contribute——share skills、fix bugs、propose better prompts。

Cocrates 不 locked 到 one IDE 或 plugin。Claude Code、Cursor、Codex、opencode——你可在 anywhere 实现 Cocrates 原则与 skills。Cocrates 是 methodology，不只是 tool。

---

## 💡 今日洞察：Unexamined Code 也适用于框架本身

记得系列核心原则？

> *"The unexamined code is not worth generating."*

它不只适用于 code——也适用于 **Cocrates Harness 本身**。

Cocrates 必须 continuously reviewed、analyzed、evaluated、improved。*"It's Cocrates—it must be fine"* 是 slide back 到 blind AI-assisted trust。

用好 Cocrates 不够。**You must evolve Cocrates.**

---

## 📌 要点回顾

1. **Users must evolve**——内化 Socratic questions、习惯 U→A→E→A、explicit approval。
2. **Cocrates Harness must evolve**——skills 与 agent prompts。
3. **Mutual evolution 是 feedback loop**——sustainable growth engine。
4. **Cocrates 是 open source**——be contributor，not only consumer。
5. **原则也适用于 framework**——unexamined code 包括 Cocrates 自己的 code。

---

**问自己：**

- 我 satisfied with using Cocrates——还是 evolving it？
- 我最近 create 或 edit 过 Cocrates skill 吗？
- 我读过 Cocrates source code 吗？
