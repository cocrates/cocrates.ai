## Behind the Chapter (Writer's Notes)

Ch 005는 Ch 004의 **여운**을 다룬다. 사고 현장이라는 극적인 사건 이후, 유진이 개인적인 방식으로 그날을 소화하는 과정. R-17 데이터 분석은 단순한 기술 작업이 아니라, 유진이 재민을 '알게 되는' 과정이다.

**핵심 설계 원칙:**
1. 유진은 재민을 직접 만나지 않는다. 데이터 속의 재민을 발견한다. 이 '간접적 발견'이 Ch 007(병원 재회)에서의 긴장감을 배가시킴.
2. 유진의 로봇관이 다시 한 번 드러난다. R-17을 단순한 장비로 대하지 않는 그녀의 태도 — 데이터 분석 과정에서 R-17의 '기록'을 R-17의 '목소리'처럼 대함.
3. 나나 기억이 처음으로 유진의 과거에 대한 실마리를 제공한다. 그러나 노골적으로 드러내지 않음 — "나나가 생각난다"는 수준으로만.
4. 이수는 유진의 데이터 분석을 지원하며, 동시에 자신의 관찰 데이터(재민 관련)를 보완한다.

**Ch 004와의 톤 차이:** Ch 004는 빠른 템포, 액션, 충돌. Ch 005는 느린 템포, 침묵, 관찰. 새벽의 고요함, 데이터 패드 위의 숫자, 복구된 이미지 속의 얼굴. Ch 004가 '사건'이라면 Ch 005는 '여운'.

## Reader Engagement (Chapter Level)
### Personal Stake
- **유진:** R-17은 단순한 업무용 로봇이 아니다. 3년간 그녀가 정비하고 신뢰해온 '동료'였다. 데이터 복구는 그녀의 의무이자, 그녀의 방식으로 R-17과 작별하는 의식. 데이터 속에서 발견한 재민 — 그녀에게 새로운 궁금증을 던짐.
- **이수:** 유진의 데이터 분석 과정을 지원하며, 인간이 로봇의 데이터를 어떻게 '의미'로 해석하는지 관찰. 재민 관련 데이터 보완 — '분류 중' 상태 유지.

### Reader Questions
- 유진에게 R-17은 무엇이었는가? '장비'를 넘어선 관계의 증거는 무엇인가?
- 유진은 재민의 수술 데이터를 보고 무엇을 느끼는가?
- R-17의 시각 센서에 잡힌 재민의 얼굴 — 유진이 그 프레임을 오래 바라보는 이유는?
- '나나' — 유진의 과거에 무슨 일이 있었는가?
- 이수는 이 모든 과정을 어떻게 데이터화하는가?

### Chapter Arc

**현장 복구(잔해 수습, 새벽의 고요) → 데이터 분석(기술 작업 속 재민 발견) → 이미지 발견(재민의 얼굴, 나나의 기억) → 여운(저녁, Ch 007로의 연결)**
