# Revisions Plan: 거부된 미래 (Rejected Future)

> **작성일**: 2026-07-16
> **기반**: `revisions.md` (수정 사항 정리)
> **목적**: revisions.md의 각 항목을 구체적인 파일별 작업으로 변환. 병렬 실행 가능 여부 명시.

---

## 실행 순서 (3 Layers)

```
Layer 1: 아키텍처 파일 (병렬 가능)
    ↓
Layer 2: 챕터 설계 파일 (파일 간 독립적, Layer 1 완료 후)
    ↓
Layer 3: 원고 파일 (Layer 2 완료 후)
```

- **Layer 1**은 서로 독립적이므로 **동시에** 수정 가능
- **Layer 2**는 각 챕터가 독립적이므로 **챕터별로 병렬** 가능 (단, 같은 Part 내 일관성 확인 필요)
- **Layer 3**는 Layer 2 설계 승인 후 생성
