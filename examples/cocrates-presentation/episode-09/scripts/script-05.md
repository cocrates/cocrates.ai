# Script: Slide 05 — 🔥 3블록 출력 포맷 + DIP 실제 사례

**Time:** ~1분

**[SLIDE 05: 3블록 출력 포맷 코드 블록 + DIP 예시 대화 흐름도]**

자, 이제 가장 중요한 부분입니다. 이 모든 규칙이 실제로 어떻게 하나의 응답으로 형체를 갖추는지, Education 스킬의 3블록 출력 포맷을 보여드리겠습니다.

SKILL.md에는 모든 응답이 다음 세 블록을 따라야 한다고 명확히 정의되어 있습니다.

(슬라이드의 코드 블록을 가리키며)

**첫 번째 블록, Concept Briefing.** 핵심 원리를 단 1~3문장, 일상적인 비유로 전달합니다. 응답의 20%를 넘지 않아야 합니다.

**두 번째 블록, Thought Lab.** 일부러 결함을 심어두었거나 비어있는 불완전한 예시를 제시합니다. 사용자가 직접 오류를 발견하고 수정하도록 유도하는 겁니다.

**세 번째 블록, MISSION.** 사용자가 다음 턴에 생각하고 답해야 할 정확히 하나의 인지적 과제입니다. 절대 두 개 이상을 주지 않습니다.

(잠시 멈춤)

실제 예시를 보겠습니다. 사용자가 "DIP 원칙에 대해 알려줘"라고 물었을 때 Cocrates의 응답입니다.

**Briefing:** "DIP는 전기 플러그와 같습니다. 고수준 모듈이 저수준 세부사항에 직결되지 않고, 둘 다 공유 인터페이스에 의존해야 합니다."

**Thought Lab:**
```typescript
class OrderService {
  constructor(private db: MySQLDatabase) {} // flaw
}
```

**MISSION:** "자신의 언어로, MySQLDatabase를 PostgresDatabase로 바꾸면 무엇이 깨질지 설명해보세요. 그리고 DIP를 위반하는 의존 방향이 어디인지 지적해보세요."

이 세 블록이 하나의 완전한 학습 턴을 이룹니다. Briefing이 없으면 맥락을 모르고, Lab이 없으면 생각할 재료가 없고, MISSION이 없으면 수동적으로 끝납니다. 이 구조가 Cocrates의 모든 교육 대화를 생성하는 단일한 엔진입니다.
