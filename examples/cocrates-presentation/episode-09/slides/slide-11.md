# Slide 11: 🔗 세 스킬의 유기적 연결

## Key Message
Education, Knowledge Capture, Reflection은 독립적으로 동작하지만, 같은 언어(Bloom's 2D 매트릭스)로 소통하는 하나의 시스템이다. KB가 Education과 Reflection을 연결하는 가교 역할을 한다.

## Content
### Headlines / text
- **파이프라인 흐름:**
  ```
  [Education: 탐구] → [Knowledge Capture: 기록] → [Reflection: 검증]
       ↑                                                    │
       └────────── (Gap 발견 시 Education 환류 제안) ────────┘
  ```
- **같은 도구, 다른 목적:**
  - **Education:** Bloom's 2D 매트릭스로 **Push/Pull** → 사용자 **성장**
  - **Reflection:** Bloom's 2D 매트릭스로 **Pull-only** → 이해도 **측정**
- **KB의 가교 역할:** Education에서 배운 내용이 KB에 저장 → Reflection이 KB를 평가 루브릭으로 사용
- **💡 포인트:** 세 스킬은 독립적이지만 **같은 언어(Bloom's 2D 매트릭스)로 소통하는 하나의 시스템**

### Visual elements
- 중앙: 큰 파이프라인 순환 다이어그램
- Education → KB → Reflection 순서, 하단 환류 화살표
- 각 스킬 노드에 아이콘: 🧗‍♂️ / 💾 / 🕵️‍♂️
- 하단: 💡 강조 박스
