# Slide 09: 🔍 Reflection — 진짜 아는 것과 아는 척하는 것

## Key Message
Reflection의 목표는 점수를 매기는 것이 아니라, 사용자가 자신의 무지를 정확히 인식하게 만드는 것. 반례·경계·전이 질문으로 진짜 이해도를 측정한다.

## Content
### Headlines / text
- **핵심 원칙:** 기습 질문으로 진짜 이해도 측정
- **어떻게 검증하는가:**
  - "이 개념의 반례를 들어보세요" (경계 사례)
  - "이 원칙이 깨지는 조건은 무엇인가요?" (한계 조건)
  - "이 개념을 다른 상황에 적용하려면 어떻게 바꿔야 하나요?" (전이/응용)
- **결과물 — 명세표:**
  - ✅ **진짜 완전히 아는 것** (반례·경계·응용까지 설명 가능)
  - ⚠️ **안다고 착각했던 것** (설명하려니 막힘)
- **순환 구조:** ⚠️ 발견 시 Education 단계로 돌아가 보완 → 다시 Reflection
- **💡 포인트:** Reflection의 목표는 점수를 매기는 것이 아니라 **시청자가 자신의 무지를 정확히 인식하게 만드는 것.**

### Visual elements
- 상단: 🔍 Reflection 타이틀과 핵심 원칙
- 중앙: 3가지 질문 유형을 카드로 배열 (각각 말풍선 아이콘)
- 중간: ✅ / ⚠️ 결과표 (시각적 대비 강하게)
- 하단: 순환 구조를 미니 다이어그램으로, 💡 강조 박스

## Layout Notes
- 질문 카드는 청중이 실제로 질문을 받는 느낌이 들게 인용문 스타일로
- ✅와 ⚠️는 큰 폰트로 시각적 대비 극대화
- 순환 다이어그램은 하단에 작게 배치
