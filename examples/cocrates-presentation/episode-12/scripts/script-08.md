# Script — Slide 8 (1분 30초)

## Slide: 각 단계별 상세 — Kernel부터 Verification까지

**Time:** ~1분 30초

"각 단계를 조금 더 자세히 들여다보겠습니다.

**Kernel** — 스킬의 정체성을 한 문장으로 정의합니다.
'이 스킬은 [대상 상황]에서 [산출물 유형]을 [검토 가능한 단계]를 거쳐 생성하도록 돕는다.'
Kernel이 흔들리면 모든 후속 단계가 흔들립니다. define 단계에서 가장 중요한 활동입니다.

**Components** — 5대 차원을 산출물 유형에 맞게 명명하고 범위를 설정합니다.

**Frame** — 5단계 Snowflake 워크플로우와 디렉토리 구조, 승인 지점을 설계합니다.

**여기가 핵심입니다 — Outline.**

Outline 단계에서는 각 단계별로 생성될 파일 산출물을 **5가지 항목**으로 정의합니다:

- **Input**: 이 파일이 의존하는 산출물은 무엇인가?
- **Creation activity**: 이 파일을 만들기 위해 어떤 작업을 수행하는가?
- **Completion criteria**: 이 파일이 완료된 것으로 간주되는 조건은?
- **Review questions**: 사용자 확인을 위해 어떤 질문을 던질 것인가?
- **Approval criteria**: 다음 단계로 진행하기 위한 승인 조건은?"

(잠시 멈춤)

"이 **Outline Format**이 중요한 이유는 단 하나입니다.

중간 산출물이 **채팅에 휘발되지 않고 파일로 관리**되도록 보장하기 때문입니다.

그리고 마지막으로 **Spec → Skill → Verification**.
확정된 명세로 frontmatter, 원칙, 절차, 금지 사항, 완료 조건을 담은 SKILL.md를 생성하고, 이 스킬이 Cocrates 생성 하네스에서 정상 작동하는지 검증합니다."
