# Script — Slide 6 (1분 30초)

## Slide: 단계별 구성 요소 구체화 — 소설 창작의 구성 요소별 진화

**Time:** ~1분 30초

"그런데 여기서 중요한 점이 하나 더 있습니다.

각 구성 요소는 **동일한 속도로 구체화되지 않습니다**.

이 표를 한번 보시죠.

**과업과 제약** — 장르와 분량. 이건 define 단계에서 확정되고, 이후에는 변하지 않습니다. 한 번 정해지면 바꾸기 어려운 근본적인 조건들이죠.

**개체, 즉 등장인물** — define에서는 '주인공, 조연, 악당'이라는 역할 유형만 식별합니다.
plan에서는 이들의 관계를 요약합니다.
architecture design에서는 인물 카탈로그와 계층 구조를 만듭니다.
detail design에서 비로소 인물별 상세 설정을 완성합니다.
그리고 generation에서 **인물별 챕터가 비로소 완성**됩니다.

**구조와 흐름, 즉 플롯** — define에서 5막 구조라는 최상위 골격을 잡고, generation에서 최종 조립될 때까지 점점 구체화됩니다.

여기서 **지연 구체화(Lazy Refinement)** 라는 전략이 등장합니다.
플롯이 특정 인물을 참조할 때, 우리는 먼저 인물 카탈로그를 architecture design에서 완성합니다. 하지만 인물별 상세 설정은 detail design에서, 필요할 때 채웁니다.

(잠시 멈춤)

핵심은 이것입니다. 모든 것을 한 번에 완벽히 설계하려 하지 마십시오.

**적시에, 적절한 수준으로 구체화하는 것** — 이것이 Snowflake 방법론의 진정한 힘입니다."
