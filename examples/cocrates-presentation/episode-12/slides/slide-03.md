# Slide Content Specification — Slide 3

## Slide 3: 🧬 두 가지 접근법 — 구조를 아는 것과 모르는 것

**Section:** 본론 1 — 메타-스킬의 정체성
**Type:** 개념/비유
**Time:** ~1분 30초

### Visual Design
- **Background:** 시리즈 공통 템플릿
- **Layout:** 좌우 비교 레이아웃
  - 상단: 제목 "두 가지 접근법" (32pt)
  - 좌측 컬럼: "구조를 모를 때" — 빨간 계열 강조
  - 우측 컬럼: "구조를 알 때" — 초록/파랑 계열 강조
  - 하단: 메타-스킬 연결 — 💡 포인트 텍스트

### Content Elements
| 요소 | 내용 | 비고 |
|------|------|------|
| 제목 | 🧬 두 가지 접근법 — 구조를 아는 것과 모르는 것 | 상단 |
| 좌측: 구조 모를 때 | 호텔, 식당, 활동을 개별 검토 → 막연한 계획 | 빨간 계열 |
| 좌측: 방식 | ASR 식별 → ADR → 사용자 결정 → Spec → 생성 | (spec-driven-generation) |
| 우측: 구조 알 때 | 여행 계획 = **timeline 구조** | 초록/파랑 계열 |
| 우측: timeline | 숙박 → 조식 → 활동 → 중식 → 활동 → ... | 일별 timeline |
| 우측: 경로 | 여행지1 → 여행지2 → 여행지3 → ... | 경로 구조 |
| 우측: 결과 | 더 구체적인 설계 결정 가능 (호텔/식당/활동 검토) | 하이라이트 |
| 하단 💡 | "두 접근법의 차이는 '구조를 아는가 모르는가' — 메타-스킬은 알게 된 구조를 영구적인 스킬로 전환" | 강조 |

### Layout Notes
- 좌우 비교 명확하게. 중간에 수직 구분선.
- 우측(timeline 구조)을 더 시각적으로 강조 (그림/아이콘으로 timeline 표현).
- 하단 💡 포인트는 별도 박스로 강조.
