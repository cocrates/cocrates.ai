# Slide 11: 4️⃣ 4단계 — Generation & Verification

## Key Message
검증 가능해야 생성할 가치가 있다. Undocumented ASR을 경계하고 순환적 개선하라.

## Content
- **Step indicator (상단):** 4️⃣ **4단계: Generation & Verification**
- **Left column — Generation:**
  - **🏗️ Generation — Spec 기준 생성**
  - Spec을 유일한 기준으로 산출물 생성 (최초 프롬프트가 아님)
  - **건축 비유:** 설계도를 시공팀에 넘기고 시공
- **Right column — Verification:**
  - **🔍 Verification — Spec 대조 검증**
  - ✅ Spec 요구사항 충족 여부
  - ❌ **Undocumented ASR** — 설계도에 없는 AI의 임의적 결정 밀수입 확인 (예: 설계도에 옥탑방 → 그냥 옥상으로 끝남)
  - **건축 비유:** 완공 후 설계도와 대조 검수
- **Bottom left 💡:** 검증 가능해야 생성할 가치가 있다 (Ep2 원칙)
- **Bottom right 💡:** Undocumented ASR → 다음 Spec 개정 재료 (순환적 개선)
- **Center bottom:** 🔄 Generation → Verification → Spec 개정 → Re-Generation
