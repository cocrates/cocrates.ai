# Slide 09: 2️⃣ 2단계 — ADR: 대안 분석 및 결정

## Key Message
ADR은 AI의 독단을 막고 사용자가 설계도의 주인으로 남게 하는 장치다.

## Content
- **Step indicator (상단):** 2️⃣ **2단계: ADR — 대안 분석 및 결정**
- **Concept (좌측):**
  - **What:** 최소 2~3가지 명확한 선택지 제시, 장단점 비교, 사용자 결정
  - **세 가지 규칙:**
    - ✓ 사용자가 위임한 영역에 대해서만 진행
    - ✓ 각 선택지의 장단점을 명확히 비교
    - ✓ 사용자의 결정은 Audit Trail로 기록
  - **건축 비유:** "지붕으로 하면 비용은 저렴하지만 공간 활용 불가. 옥상으로 하면 테라스 가능 but 방수 필요. 옥탑방으로 하면 추가 공간 but 공사비 최고. 선택은 당신의 몫입니다."
- **ADR 예시 mockup (우측):**
  - **선택지 A (지붕):** 비용 저렴 ✅ / 공간 활용 불가 ❌
  - **선택지 B (옥상):** 테라스 활용 ✅ / 방수 관리 필요 ❌
  - **선택지 C (옥탑방):** 추가 생활 공간 ✅ / 공사비 가장 높음 ❌
  - **결정:** B ✅
- **💡 Point box:** ADR은 AI의 독단을 막고 사용자가 설계도의 주인으로 남게 하는 장치다

- 결정 표시: ✅ + 사용자 아이콘
