# Script: Slide 07 — 🏡 전원주택 건축으로 보는 4단계 파이프라인 개관

**Time:** ~1분

**[SLIDE 07: Full-width horizontal pipeline diagram — 4 boxes with arrows: ① Concern/ASR 식별 → ② ADR → ③ Spec → ④ Generation & Verification / 아래: 전원주택 건축 비유 대응표]**

(전체 구조를 한눈에 보여주며)

자, 드디어 핵심입니다. Cocrates의 구조 기반 생성은 다음 4단계 파이프라인으로 작동합니다.

(화면의 파이프라인을 따라가며)

**1단계 — Concern/ASR 식별.** 구조를 좌우할 중요한 질문들을 수면 위로 올립니다.

**2단계 — ADR: 대안 분석 및 결정.** 각 질문에 대해 최소 2~3가지 선택지를 제시하고, 사용자가 결정합니다.

**3단계 — Spec: 설계 명세서.** 결정된 모든 내용을 하나의 통합 설계도로 만듭니다.

**4단계 — Generation & Verification.** 설계도를 기준으로 생성하고, 생성물이 설계도와 일치하는지 검증합니다.

(비유로 연결하며)

이해를 돕기 위해 전원주택 건축에 비유해볼게요.
**1단계:** 건축가가 "지붕을 할까, 옥상을 할까, 옥탑방을 할까?" 같은 중요한 질문들을 식별합니다.

**2단계:** "지붕 vs 옥상 vs 옥탑방? 각각의 장단점은?" 선택지를 비교하고 결정합니다.
**3단계:** 결정된 모든 내용을 하나의 통일된 설계도로 통합합니다.
**4단계:** 설계도대로 시공하고, 완공 후 설계도와 대조하며 검수합니다.

(자신 있게)

**💡 이 4단계 파이프라인이 Cocrates 구조 기반 생성의 핵심 엔진입니다.** 모든 산출물 — 보고서, 발표자료, 블로그 포스트, 코드 — 전부 이 파이프라인을 거쳐 탄생합니다.

(화면 전환)
