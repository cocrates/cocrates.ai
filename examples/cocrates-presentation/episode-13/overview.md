# Presentation Definition — Episode 13

## Topic
올바른 Cocrates Harness 활용 — 사용자와 에이전트의 공진화

## Purpose
Cocrates Harness를 설치하고, 실습하고, 구조의 원리까지 이해했다. 그런데 이게 끝일까? 아니다. **진짜 사용은 지금부터 시작**이다.

Cocrates는 사용자가 AI를 올바르게 사용하는 방법을 제시하고, 사용자의 **구조 주도 생성 역량(Architecture-Driven Generation Capability)**을 개선하도록 돕는다. 하지만 **Cocrates를 사용하는 것만으로는 충분하지 않다.** 더 나아가, AI-native Engineer(팀장)는 AI 팀원과 함께 일하는 도구인 Cocrates Harness를 지속적으로 진화시켜야 한다. 즉, 주어진 도구를 소비하는 데서 그치지 않고, **우리 팀이 일을 제대로 하는 도구로 계속 발전시켜 나가는 것** — 이것이 이 에피소드의 핵심 메시지다.

이 에피소드에서는 다음을 다룬다:
1. **사용자가 진화해야 한다** — Cocrates의 소크라테스식 질문이 사용자 자신의 사고방식으로 내면화되는 과정
2. **Cocrates Harness도 진화해야 한다** — 스킬 확장, 프로세스 커스터마이징, 에이전트 프롬프트 개선
3. **함께 진화하는 피드백 루프** — 사용자의 검토가 에이전트를 개선하고, 에이전트의 결과물이 사용자를 성장시킴
4. **오픈소스 철학과 플랫폼 독립성** — Cocrates는 특정 IDE/플러그인에 종속되지 않는 열린 설계도
5. **지속적 개선의 원칙** — "The unexamined code is not worth generating"은 프레임워크 자신에게도 적용
6. **🎯 궁극적 메시지: "Cocrates를 사용하는 것으로 충분하지 않다 — Cocrates를 진화시켜야 한다"**

## Audience
- **Background:** 소프트웨어 개발자(초급~중급), AI 도구를 업무에 활용하는 지식 노동자
- **Knowledge level:** Ep1~12 전체 시리즈를 통해 Cocrates의 모든 구조와 원리를 학습한 상태
- **Interests/expectations:** 배운 것을 실제 업무에 적용하고, 도구를 자신의 필요에 맞게 진화시키는 방법을 알고 싶어 함

## Logistics
- **Time limit:** 12분 (10슬라이드)
- **Format:** 시리즈 내 독립 발표 세션 (in-person)
- **Tone/style:** Part 4(결론) — 지금까지의 여정을 되돌아보고, 앞으로의 방향을 제시. 격려와 다짐의 톤.

## Success Criteria (발표자 관점 — "전달에 성공했는가?")
1. **공진화 개념 전달** — 청중이 "사용자가 진화해야 도구도 진화하고, 도구가 진화해야 사용자도 더 성장한다"는 **양방향 순환 관계**를 자신의 말로 설명할 수 있다
2. **핵심 메시지 전달** — 청중이 **"Cocrates를 사용하는 것으로 충분하지 않다 — AI-native Engineer로서 우리 팀의 도구를 지속적으로 진화시켜야 한다"** 는 메시지에 공감하고 스스로의 상황에 대입해볼 수 있다
3. **실천 의지 형성** — 청중이 발표 후 **"내 Cocrates Harness를 어떻게 개선할까?"** 라는 질문을 스스로에게 던지게 된다

## Constraints & Taboos
- 이미 배운 개념(Ep1~12)의 재설명 금지 — 여정을 요약하되 새로운 내용에 집중
- Ep14(사용자 선언문)의 7계명 내용을 미리 다루지 않음
- 기술적 설치 방법(Ep3)이나 실습(Ep4~6) 재설명 금지
- Part 4의 스타일 — 격려와 다짐의 톤, 선언적 어조
