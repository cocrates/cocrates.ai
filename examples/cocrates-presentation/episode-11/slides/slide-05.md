# Slide Content Specification — Slide 5

## Slide 5: 📝 adr-writing — "왜 이런 결정을 내렸는가"

**Section:** 본론 2 — 4개 스킬 상세
**Type:** 개념/상세
**Time:** ~1분 30초

### Visual Design
- **Background:** 동일 템플릿
- **Layout:** 상단 제목 + 중앙 핵심 원칙 + 하단 3개 금지 규칙
  - 상단: 스킬명 + 역할 설명
  - 중앙: 핵심 원칙 (1 ADR = 1 Concern)
  - 하단: 세 가지 금지 규칙 — 3열 배열 (각각 ❌ 아이콘)

### Content Elements
| 요소 | 내용 | 비고 |
|------|------|------|
| 제목 | 📝 adr-writing — "왜 이런 결정을 내렸는가" | 상단 |
| 핵심 원칙 | **1 ADR = 1 Concern** (하나의 ADR 파일 = 하나의 관심사) | 중앙, 큰 텍스트 |
| 적용 범위 | 기술, 문서 구조, 발표 전개 방식 — 모든 구조적 결정 | 중앙, 작은 텍스트 |
| 금지 1 | ❌ **가짜 대안 제시 금지** — 모든 대안은 실질적이고 장단점이 팽팽해야 함 | 좌측 카드 |
| 금지 2 | ❌ **산문 스타일 금지** — 이름 + 2~4개 불릿으로만 트레이드오프 요약 | 중앙 카드 |
| 금지 3 | ❌ **무단 승인 금지** — 사용자 명시적 구두 승인 후에만 proposed → approved | 우측 카드 |
| 예시 | "보고서 서론 구조 — chronological vs problem-first vs executive summary" | 하단 작은 박스 |
| Key Point | 💡 ADR은 결정을 **강요하는** 것이 아니라 결정을 **가능하게 하는** 장치 | 하단 |

### Visual Notes
- 각 금지 규칙 카드에 아이콘 + 짧은 설명
- 금지 규칙 3개가 시각적으로 균형 잡히도록 배치

### Transitions
- **In:** 제목 → 핵심 원칙 → 금지 규칙 순차 등장
- **Out:** Key Point 표시
