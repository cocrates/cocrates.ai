# Slide 08: 📜 Spec — 룰북과 금지 조항

## Key Message
Spec은 합의된 행동 규범이다. 무엇을 해야 하는지(Do)뿐 아니라, 무엇을 하지 말아야 하는지(Don't)도 명시한다.

## Content
- **6가지 맞춤형 구성 방식 선택지 (사용자의 통찰이 반영됨):**
  - 설득/제안: 서론-본론-결론
  - 스토리/사례: 기승전결
  - 문제 해결: 문제-원인-해결-제안
  - 비교/분석: 기준-대안-평가-결론
  - (기타 2가지)
- **7가지 금지 사항 (Anti-Patterns):**
  - 승인 게이트 통과 전 본문 작성 금지
  - 중간 산출물을 파일로 저장하지 않고 채팅창에만 남기지 말 것
  - (등)
- **포인트:** Do & Don't가 모두 명시되어야 AI가 혼란 없이 일관된 방식으로 작업한다

## Visual Elements
- Left column: "✅ Do" — 6가지 구성 방식
- Right column: "❌ Don't" — 7가지 금지 사항
- Top: Spec document icon with checkmark
- Bottom: "합의된 행동 규범" label

## Layout Notes
- Two-column layout: Do / Don't
- Use checkmark (✅) for Do items, prohibition (🚫) for Don't items
- List items concisely — use keywords not full sentences
