# Slide 03: "보고서 써줘" 대신 "보고서 쓰는 법"을 AI에게 가르친다

## Key Message
일회성 요청(좌)과 스킬(우)의 차이. 스킬은 AI의 SOP(Standard Operating Procedure)로서 한 번 만들면 영원히 재사용 가능한 자산이다.

## Content
- **좌측: 일반적인 접근 (일회성 요청)** — "보고서 써줘"
  - 매번 새로운 요청, 매번 다른 품질, 반복 피로
  - AI가 그때그때 다르게 응답 → 일관성 없음
- **우측: Cocrates의 접근 (스킬)** — "보고서 자동화 스킬 만들어줘"
  - 한 번 만들면 영원히 재사용
  - 같은 유형의 요청 → 항상 같은 워크플로우로 진행 → 일관된 품질
- **비유:** "물고기를 잡아달라" vs "물고기 잡는 법을 가르쳐달라"
- **핵심 정의:** 스킬 = AI의 행동 규범(Standard Operating Procedure)
- **오늘 만들 것:** `document-authoring` 스킬
