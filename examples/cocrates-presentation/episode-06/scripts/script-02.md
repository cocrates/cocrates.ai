# Script: Slide 02 — 학습 목표

**Time:** ~1.5분

**[SLIDE 02: 학습 목표 3가지 — 🔄 스킬 vs 일회성 요청 / 🔧 6단계 파이프라인 / ❄️ Snowflake 실전 적용]**

시작하기 전에, 여러분께 한 가지 질문을 드리고 싶습니다. "여러분, AI에게 같은 부탁을 여러 번 해보신 적 있나요? '보고서 써줘', '설명문 작성해줘' — 매번 비슷한 결과물을 받으면서도 왠지 만족스럽지 않았던 경험, 있으시죠?"

오늘 우리는 이 문제를 근본적으로 해결합니다. 일회성 요청을 넘어서, AI의 **일하는 방식 자체를 설계**하는 겁니다.

오늘의 학습 목표는 세 가지입니다.

첫째, **스킬과 일회성 요청의 차이**를 이해합니다. 스킬이 왜 지속 가능한 자산인지 설명할 수 있어야 합니다.

둘째, **스킬 생성 파이프라인 6단계** — Kernel, Frame, Outline, Spec, Skill, Verification — 각 단계가 무엇을 하는지 이해합니다.

셋째, **Snowflake 점진적 확장**이 실제로 어떻게 적용되는지 실전 사례를 통해 익힙니다. 작게 시작해서 점진적으로 확장하는 방법, 이것이 오늘의 핵심입니다.

자, 그럼 본격적으로 들어가볼까요?
