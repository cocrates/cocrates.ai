# Script: Slide 09 — 🔄 Cocrates는 완성된 하네스가 아니다

**Time:** ~1분

**[SLIDE 09: 진화/성장 개념 이미지 — "사용 → 이해 → 진화" 3단계 화살표. 하단: "구조 이해 = 커스터마이징과 진화의 첫걸음"]**

자, 여기서 가장 중요한 메시지를 전하겠습니다.

**Cocrates는 최종 완성된 하네스가 아닙니다.**

처음에 말씀드렸죠? 여러분의 업무 패턴에 맞춰 지속적으로 발전시키고 진화시켜야 합니다.

그렇다면 구조를 이해한 사용자는 무엇을 할 수 있을까요?

첫째, **기존 스킬을 자신의 업무 패턴에 맞게 수정**할 수 있습니다. 예를 들어 `document-authoring` 스킬의 문서 구조를 자신의 회사 템플릿에 맞게 바꾸는 거죠.

둘째, **새로운 산출물 유형에 대한 스킬을 직접 생성**할 수 있습니다. Ep6에서 배운 `generating-skill-creation`이 바로 이 도구입니다.

셋째, **Agent Prompt의 원칙을 자신의 맥락에 맞게 조정**할 수 있습니다. 성공 기준이나 원칙을 자신의 팀 문화에 맞게 수정하는 거죠.

**구조를 이해하는 것이 커스터마이징과 진화의 첫걸음입니다.** 지금까지의 에피소드가 Cocrates를 사용하는 법이었다면, 여기서부터는 여러분이 Cocrates의 주인이 되는 법을 배워가는 겁니다.
