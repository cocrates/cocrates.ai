# Script: Slide 10 — 핵심 요약 + 학습 확인

**Time:** ~1.25분

**[SLIDE 10: 3-Point 요약 + 3개의 자기 질문]**

오늘 배운 내용을 세 가지 포인트로 요약하겠습니다.

**첫째, 만능 칼은 없습니다.** 산출물 유형마다 구조적 접근이 다르기 때문에, 하나의 프롬프트로는 모든 것을 처리할 수 없습니다. Cocrates는 Agent, 즉 헌법과 Skills, 즉 전문 부대를 분리하여 이 문제를 해결합니다. Agent는 공통 원칙을, Skills는 각 산출물의 전문 워크플로우를 담당합니다.

**둘째, Cocrates Agent는 6개 섹션의 헌법입니다.** Persona, Principle, Harness Architecture, Request Handling, Core Activities, Success Criteria. 이 중에서도 Request Handling의 Intent-To-Skill Routing이 의도 기반 스킬 선택의 핵심입니다. 사용자의 근본 의도를 추론해서 최적의 스킬을 매핑합니다.

**셋째, Cocrates는 완성된 하네스가 아닙니다.** 구조를 이해해야 커스터마이징과 진화가 가능합니다. 새로운 스킬을 추가하고, 기존 스킬을 수정하며, 자신의 업무 패턴에 맞게 발전시켜 나가야 합니다.

이제 스스로 확인해보시겠어요?

Cocrates가 Agent + Skills 구조를 채택한 이유를 설명할 수 있습니까?
Cocrates Agent Prompt의 6개 섹션을 말할 수 있습니까?
Intent-To-Skill Routing이 의도를 기반으로 어떻게 스킬을 선택하는지 설명할 수 있습니까?

잠시 멈추고 스스로 대답해보세요.
