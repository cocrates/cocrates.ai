# Slide 05: 📜 Cocrates Agent Prompt — 6개 섹션 개관

## Key Message
Cocrates Agent(cocrates.md)는 6개 섹션으로 구성된 헌법이다: Persona → Principle → Harness Architecture → Request Handling → Core Activities → Success Criteria.

## Content
- **전체 구조를 한눈에:** Persona → Principle → Harness Architecture → Request Handling → Core Activities → Success Criteria
- **각 섹션 간략 설명:**
  1. **Persona (정체성):** "불확실성을 체계적인 탐구로 전환하고, 사용자가 결과물을 완전히 이해할 때까지 안내하는 존재"
  2. **Principle (원칙):** Harness Ignorance — 무지의 통제, 검토 없는 생성 금지
  3. **Harness Architecture (구조):** Agent + Skills 분리 원칙 선언
  4. **Request Handling (요청 처리):** 의도 기반 스킬 라우팅 — Intent-To-Skill Routing
  5. **Core Activities (핵심 활동):** 산출물 생성 + 학습 두 파이프라인
  6. **Success Criteria (성공 기준):** 사용자가 산출물을 스스로 설명할 수 있는가?

## Visual Elements
- 6개 섹션을 가로로 연결한 순서도 (horizontal flow)
- 각 섹션에 번호와 아이콘
- Section 4 (Request Handling)만 다른 색상으로 강조 (다음 슬라이드에서 상세)
- 하단: "→ Slide 6에서 Intent-To-Skill Routing 상세"

- Horizontal flow diagram taking most of the slide

- 블로그 에피소드보다 압축된 정보량 (구조적 이해에 초점)
