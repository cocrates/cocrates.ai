### 2. 본론 (슬라이드 3~9) — 약 6분

#### 2.1 원칙 선언 (슬라이드 3~4) — 2분

##### 슬라이드 3: 선언 — "The Unexamined Code Is Not Worth Generating"
- **Key message:** 이 시리즈를 관통하는 단 하나의 선언. 검토되지 않은 산출물은 생성할 가치가 없다.
- **Key points:**
  - Ep1의 질문: "AI-native Engineer의 핵심 역량은?"
  - 답변: **검토** — 그리고 이 한 문장으로 선언한다
  - **"The unexamined code is not worth generating."**
  - "검토되지 않은 산출물은 생성할 가치가 없다"

##### 슬라이드 4: 소크라테스의 영감
- **Key message:** 이 선언은 소크라테스의 명언 "성찰하지 않는 삶은 살 가치가 없다"에서 영감을 받았다.
- **Key points:**
  - 소크라테스: "The unexamined life is not worth living"
  - Cocrates: "The unexamined code is not worth generating"
  - 끊임없는 질문과 성찰이 없는 삶은 의미가 없다
  - **내가 이해하고 검토하지 않은 AI의 결과물도 마찬가지로 가치가 없다**

#### 2.2 Code의 확장 (슬라이드 5) — 1분

##### 슬라이드 5: 'Code'의 확장
- **Key message:** Cocrates가 말하는 'Code'는 소스 코드만이 아니다. AI를 사용해 만드는 모든 최종 산출물이 Code다.
- **Key points:**
  - "어? 나는 개발자 아니라서...?" — 걱정 마세요
  - Cocrates의 Code = AI로 만드는 **모든 최종 산출물**
  - 💻 소스 코드 | 📝 보고서 | 📊 슬라이드 | 📚 블로그 | 🧠 학습 노트
  - 당신이 AI에게 만들어 달라고 한 것, 그것이 Code입니다

#### 2.3 Architecture의 중요성 (슬라이드 6) — 1분

##### 슬라이드 6: 뼈대(Architecture)의 중요성
- **Key message:** 최종 산출물을 처음부터 한 번에 찍어내지 않는다. 먼저 구조를 설계하고, 검토하고, 승인한 뒤에 알맹이를 채운다.
- **Key points:**
  - 블로그 시리즈 자체가 이 과정의 예시
  - (1) 전체 그림 설계(overview) → 검토/승인
  - (2) 각 에피소드 뼈대(outline) → 검토/승인
  - (3) 상세 설계(episodes.md) → 검토/승인
  - (4) 본문 생성 → 검토/승인
  - 매 단계마다 검토와 승인이 들어간다

#### 2.4 검토의 진짜 본질 (슬라이드 7~8) — 2분

##### 슬라이드 7: 검토의 진짜 본질 — U → A → E → A
- **Key message:** 검토는 '대충 읽어보기'가 아니다. 이해 → 분석 → 평가 → 승인의 4단계로 이루어진다.
- **Key points:**
  - 대부분의 사람들이 생각하는 검토: "대충 읽어보고 오타 확인"
  - Cocrates가 말하는 검토:
  - **① 이해 (Understand):** 내 말로 다시 설명할 수 있을 때 비로소 이해한 것
  - **② 분석 (Analyze):** "왜 이렇게 구현되었는가?" — 설계 의도, 구조, 로직 파악
  - **③ 평가 (Evaluate):** "이게 정말 최선인가?" — 프로젝트 목적/기준에 부합하는지 판단
  - **④ 승인 (Approve):** "내가 확인했다. 이걸로 간다. 책임은 나다."
  - 검토 = U→A→E→A

##### 슬라이드 8: 무지의 제거 (Harnessing Ignorance)
- **Key message:** 검토의 궁극적 목적은 '무지의 제거' — 내가 무엇을 모르는지 인식하고 채워가는 과정이다.
- **Key points:**
  - 검토의 목적은 단순한 오류 찾기가 아니다
  - **무지의 제거 (Harnessing Ignorance)**
  - 내가 몰랐던 것을 알게 되는 과정이 진짜 성장
  - Cocrates의 원칙: "do not leave ignorance unattended" — 무지를 방치하지 마라
  - 검토를 통해 모르는 것을 발견하고 채워 넣는 것

#### 2.5 Cocrates Harness의 역할 (슬라이드 9) — 1분

##### 슬라이드 9: Cocrates Harness의 리듬
- **Key message:** Cocrates는 검토 없이 날림으로 넘어가는 순간을 절대 넘기지 않는 깐깐한 페이스메이커다.
- **Key points:**
  - 👤 유저: "귀찮아, 그냥 로그인 모듈 대충 만들어줘!"
  - 🦉 Cocrates: "검토되지 않은 산출물은 생성할 가치가 없습니다. 구조부터 함께 설계해볼까요?"
  - 처음에는 잔소리처럼 느껴질 수 있다
  - 하지만: **"뼈대를 검토하는 1시간이, 일주일짜리 버그 수정을 막아준다"**
  - Cocrates = 답을 주는 자판기가 아니라, 사용자의 주도권을 지켜주는 가드레일

### 3. 결론 (슬라이드 10~12) — 약 2분

#### 슬라이드 10: 핵심 요약
- **Key message:** 오늘의 4가지 핵심 포인트
- **Estimated time:** 1분
- **Key points:**
  1. **원칙 선언:** "The unexamined code is not worth generating"
  2. **Code의 확장:** 소스 코드 + 보고서 + 슬라이드 + 블로그 + 학습 노트
   3. **검토의 4단계:** 이해(U) → 분석(A) → 평가(E) → 승인(A)
  4. **무지의 제거:** 검토를 통해 모르는 것을 발견하고 성장한다

#### 슬라이드 11: 학습 목표 확인
- **Key message:** 스스로에게 질문하라. "지금 나는 AI의 결과물을 검토하고 있는가, 대충 넘어가고 있는가?"
- **Estimated time:** 1분
- **Key points:**
  - 질문 1: "오늘 AI에게 받은 결과물 중, U→A→E→A를 모두 거친 것은 몇 개인가?"
  - 질문 2: "방금 AI가 생성해준 이 결과물을, 내 말로 설명할 수 있는가?"
  - 미션: 영상을 멈추고, 자신이 가장 최근에 AI에게 받은 결과물을 떠올려 U→A→E→A로 검토해보라

#### 슬라이드 12: 다음 에피소드 연결
- **Key message:** 원칙을 알았으니 이제 실천할 차례. 다음 편에서는 Cocrates Harness를 직접 설치하고 첫 대화를 시작한다.
- **Estimated time:** 1분
- **Key points:**
  - 오늘: "무엇"이 옳은 일인지 알게 되었다 (원칙)
  - 다음: "어떻게" 실천할지 배운다 (설치와 첫 대화)
  - **"Cocrates Harness 설치 — 생각보다 정말 간단합니다"**
  - 진짜 여정의 첫걸음, 다음 편에서 함께 내딛자
