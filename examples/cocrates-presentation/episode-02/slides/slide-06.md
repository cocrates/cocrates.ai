# Slide 06: 뼈대(Architecture)의 중요성

## Key Message
최종 산출물을 한 번에 찍어내지 않는다. 단계별로 구조 설계 → 검토 → 승인 → 다음 단계로 진행하는 것이 Cocrates의 방식이다.

## Content
- **Headline:** Cocrates의 4단계 워크플로우
- **4단계 수직 흐름 다이어그램:**
  - (1) 전체 그림 설계 (overview.md) → 🔍 검토 → ✅ 승인
  - (2) 각 에피소드 뼈대 (outline.md) → 🔍 검토 → ✅ 승인
  - (3) 상세 설계 (episodes.md) → 🔍 검토 → ✅ 승인
  - (4) 본문 생성 → 🔍 검토 → ✅ 승인
- **하단 메시지:** "이것이 Cocrates가 작동하는 방식이다"
- **Visual Elements:**
  - 4단계 수직 흐름도
  - 각 단계 우측에 🔍 (검토)와 ✅ (승인) 아이콘 배치
  - 단계 간 연결 화살표

## Layout Notes
- 수직 흐름이 가장 직관적 — 좌측에 단계명, 우측에 검토/승인 아이콘
- 각 단계는 카드 형태로 구분
- 블로그 시리즈 예시임을 작은 레이블로 표시
