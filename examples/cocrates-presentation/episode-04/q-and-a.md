# Q&A Plan — Episode 04

## Anticipated Audience Questions

### Q1: "블룸 분류학을 처음 들었는데, 6단계를 다 외워야 하나요?"

**Answer:**
외울 필요는 없습니다. 중요한 것은 **개념적 이해**입니다.

(1) 6단계는 아래로 갈수록 낮은 차원의 사고, 위로 갈수록 높은 차원의 사고입니다.
(2) 이 6단계는 **진도 순서**가 아니라 **의존성 구조**입니다.
(3) 진짜 블룸 분류학은 **2차원 매트릭스**입니다.

이 세 가지만 기억하세요. 6단계 이름은 자주 보다 보면 자연스럽게 익혀집니다.

**관련 슬라이드:** 03, 04, 05

---

### Q2: "2차원 매트릭스는 처음 알았어요. 이걸 실무에서 어떻게 활용하나요?"

**Answer:**
훌륭한 질문입니다. 몇 가지 실용적인 예를 들어드리죠.

1. **학습 설계:** 교육 커리큘럼을 설계할 때, 각 세션이 매트릭스의 어느 셀을 목표로 하는지 매핑해보세요. 예: "Git 기초 문법 암기" = 사실적×기억, "브랜치 충돌 해결 실습" = 절차적×적용

2. **자기 진단:** 지금 내가 배우고 있는 내용이 매트릭스의 어느 셀에 해당하는지 생각해보세요. 나는 설명만 할 수 있는 수준(이해)인가, 실제로 적용할 수 있는 수준(적용)인가?

3. **교육 전략:** 초보자에게는 Push 전략(낮은 단계→높은 단계)이, 경험자에게는 Pull 전략(높은 단계 과제 먼저)이 효과적입니다.

이 에피소드의 목적은 블룸 분류학을 깊이 가르치는 것이 아니라, **개념적으로 이해하고 자신의 수준을 진단할 수 있게** 하는 것입니다. 깊은 활용은 Ep8~9(원리와 스킬)에서 더 다룹니다.

**관련 슬라이드:** 05, 06, 11

### Q3: "Cocrates가 면접관으로 변신하는 게 신기했어요. Reflection 단계는 항상 이렇게 하나요?"

**Answer:**
네, Reflection 단계에서 Cocrates는 **인터뷰어(면접관) 모드**로 전환됩니다.

일반 AI는 "잘하셨어요!"라고 무조건 칭찬하지만, Cocrates는 **진짜 이해도를 검증**합니다.
- 단순 암기한 내용을 반복하는지 (X)
- 자신의 언어로 설명하고, 다른 상황에 적용할 수 있는지 (O)

Pull 전략을 사용해서 높은 수준의 질문을 먼저 던지고, 사용자가 답하지 못하면 한 단계씩 낮춰서 **이해의 경계**를 찾아냅니다.

**관련 슬라이드:** 09, 10

### Q4: "⚠️ 영역이 있다는 게 좀 부담스러워요. 부정적인 건가요?"

**Answer:**
전혀 아닙니다! 오히려 반대입니다.

⚠️는 **성장의 신호**입니다. 자신의 ⚠️를 아는 사람만이 다음 단계로 나아갈 수 있습니다.

생각해보세요:
- ⚠️가 없다면 = "나는 다 안다" = 더 배울 게 없다 = 성장 정체
- ⚠️가 있다면 = "여기는 아직 부족하다" = 무엇을 공부할지 안다 = 성장 가능

Cocrates는 ⚠️를 지적할 때 결코 부정적인 어조를 사용하지 않습니다. 오히려 "여기서부터 더 공부하면 된다"는 이정표를 제시합니다.

**관련 슬라이드:** 10

### Q5: "Push와 Pull 전략 중 어떤 게 더 좋은 건가요?"

**Answer:**
어느 한쪽이 "더 좋다"고 말할 수 없습니다. **상황에 따라 선택**해야 합니다.

| 상황 | 추천 전략 |
|------|-----------|
| 초보자, 개념이 완전히 처음 | Push (순차적 주입) |
| 기본 지식이 있는 학습자 | Pull (미션형 인출) |
| 학습자가 인지적 붕괴 상태 (뇌정지) | Push로 전환 |
| 동기부여가 필요할 때 | Pull (도전 과제) |
| 시간이 촉박할 때 | Pull (핵심 only) |
| 체계적인 기초가 필요할 때 | Push (기초부터) |

Cocrates의 기본 전략은 **Pull**입니다. 이유는? 성인이 스스로 학습할 때 Pull이 더 강력한 동기부여와 깊은 이해를 만들어내기 때문입니다. 하지만 학습자가 헤매면 즉시 Push로 전환합니다. 이 판단이 바로 **AI-native Engineer의 역량**입니다.

**관련 슬라이드:** 06

### Q6: "Git 커리큘럼을 설계하라는 미션, 실제로 어떻게 했나요?"

**Answer:**
블로그 에피소드에는 구체적인 유저의 응답이 나와 있지 않지만, 미션의 의도는 다음과 같습니다.

1. **Create 수준의 도전:** 단순히 Git 브랜치 전략을 설명하는 것이 아니라, **4시간 분량의 커리큘럼을 설계**하라는 것
2. **통합적 사고 요구:** Push/Pull 전략, 2차원 매트릭스, 학습자 수준 등 모든 요소를 고려해야 함
3. **자기 것으로 만드는 과정:** 설계를 해야만 개념이 완전히 내 것이 됨

직접 해보시겠어요? 잠시 생각해보세요:
- 4시간을 어떻게 나눌 것인가?
- 각 세션의 목표는 매트릭스의 어느 셀인가?
- 초보자와 경험자를 어떻게 고려할 것인가?

**관련 슬라이드:** 07

### Q7: "Knowledge Capture에서 Gap을 기록한다고 했는데, 실제로 어떻게 하나요?"

**Answer:**
Cocrates는 학습이 끝난 후 사용자가 "배운 내용을 정리해 줘"라고 하면 `kb/{topic-slug}.md` 파일을 생성합니다.

기존의 일반 요약처럼 "블룸 분류학이란..."이라고 시작하지 않고, **"Wrong Assumptions / Gaps"** 섹션을 먼저 작성합니다.

예를 들어:
```
## Wrong Assumptions / Gaps
- ❌ 1차원 피라미드인 줄 알았음 → ✅ 2차원 매트릭스
- ❌ 기초부터 순서대로 가르쳐야 하는 줄 알았음 → ✅ Pull 전략
```

이 Gap 기록이 강력한 이유는:
1. **생생함:** 나중에 읽어도 그때의 '아하!' 순간이 재생됨
2. **자기 진단 도구:** "내가 무엇을 몰랐는지"가 명확히 보임
3. **성장 증거:** 시간이 지나고 다시 보면 얼마나 성장했는지 알 수 있음

**관련 슬라이드:** 08

### Q8: "오늘 배운 내용을 직접 실천해보려면 어떻게 해야 하나요?"

**Answer:**
3단계로 실천해보세요.

**Step 1 — 자기 진단:** 지금 자신이 블룸 분류학의 어느 단계에 있는지 진단해보세요.
- 기억: 블룸 분류학이 뭔지 대략 안다
- 이해: 6단계와 2차원 매트릭스를 설명할 수 있다
- 적용: 내 학습에 Push/Pull 전략을 활용할 수 있다
- 분석: 교육 자료를 보고 매트릭스의 어느 셀을 목표로 하는지 분석할 수 있다
- 평가: 다른 사람의 학습 설계를 평가하고 개선점을 제시할 수 있다
- 창조: 직접 교육 커리큘럼을 설계할 수 있다

**Step 2 — Cocrates에게 물어보기:** 관심 있는 주제를 Cocrates에게 물어보고 3단계 파이프라인(Education → KC → Reflection)을 경험해보세요.

**Step 3 — Gap 기록하기:** 배운 내용을 Cocrates가 기록한 `kb/` 파일을 확인하고, 자신의 언어로 Gap을 정리해보세요.

**관련 슬라이드:** 11
