# Script: Slide 12 — 다음 에피소드 연결

**Time:** ~1분

**[SLIDE 12: 상단에 "다음 에피소드" 타이틀. 중앙에 "The Unexamined Code Is Not Worth Generating"이라는 문장이 강조되어 표시. 우측 하단에 시리즈 맵에서 현재 위치(Ep1)와 다음 위치(Ep2)가 표시]**

오늘 우리는 '무엇'이 문제인지 알게 되었습니다. 바로 **태도**였죠. AI-assisted Engineer와 AI-native Engineer의 차이는 기술이 아니라 태도의 차이라는 점을 알게 되었습니다.

그렇다면, AI-native Engineer가 되기 위해 꼭 필요한 핵심 역량은 무엇일까요?

바로 **검토**입니다. AI가 한 일을 이해하고, 분석·평가하고, 승인하는 것. 이 검토라는 행위가 AI-native Engineer의 가장 중요한 업무입니다.

**다음 편**에서는 이 시리즈를 관통하는 강력한 원칙을 선언합니다.

(천천히, 강조하며)

**"The unexamined code is not worth generating."**
**검토되지 않은 산출물은 생성할 가치가 없다.**

이 한 문장이 왜 AI-native Engineer의 무기가 되는지, '검토한다'는 행위의 진짜 본질은 무엇인지, 다음 편에서 낱낱이 파헤쳐 보겠습니다.

시청해주셔서 감사합니다. 다음 편에서 봅시다!

→ (시리즈 계속)
