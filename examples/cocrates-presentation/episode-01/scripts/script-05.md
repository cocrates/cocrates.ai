# Script: Slide 05 — AI-native Engineer — B님의 사례

**Time:** ~1분

**[SLIDE 05: A님(좌측)과 B님(우측)의 대비표. A님: "지시 → 복붙 → 리스크 폭발". B님: "협의 → 구조 설계 → 이해 완료". 하단에 B님 쪽에 강조 표시]**

이번에는 B님을 볼까요? 같은 연차의 개발자입니다.

B님도 로그인 모듈이 필요합니다. 하지만 AI를 대하는 방식이 완전히 다릅니다.

"우리 프로젝트는 사용자가 10만 명 정도고 보안이 철저해야 해. 먼저 적용할 수 있는 인증 방식들을 제안해 주고, 각각 장단점을 비교해 줘."

(잠시 멈춤)

AI가 세션 기반, JWT 토큰 기반, OAuth 2.0 등 다양한 선택지를 제시하자, B님은 AI와 티키타카를 주고받으며 우리 프로젝트의 미래까지 고려한 최적의 구조를 함께 설계합니다.

1시간 치열한 논의 끝에 구조를 확정 짓고 나서야 비로소 코드를 요청합니다. B님이 받은 코드는 A님의 것과 겉보기엔 비슷할지 몰라도... B님은 이 코드가 왜 이렇게 짜였는지, 에러 처리는 왜 이 패턴인지, 모든 설계 이유를 완벽하게 이해하고 있습니다.

이게 무슨 차이일까요? 기술의 차이일까요? 아닙니다. 바로 **태도**의 차이입니다.
