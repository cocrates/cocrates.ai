## SLIDE 05: Step 1 — opencode 설치 (Mac / Linux 유저)

먼저 감각적인 Mac 유저들과 리눅스를 쓰는 능력자분들, 10초 만에 끝내드릴게요.

(자신 있게 미소 지으며)
맥북을 켜고 '터미널(Terminal.app)'만 실행하세요. 뭐 다운로드할 필요 전혀 없습니다. 맥에는 curl과 bash라는 도구가 이미 숨 쉬듯 내장되어 있거든요.

(화면의 명령어를 가리키며)
터미널 창에 이 한 줄을 복사해서 붙여넣고 엔터를 치세요.

```bash
curl -fsSL https://opencode.ai/install | bash

```

(설명하며)
인터넷에서 설치 파일을 안전하게 끌고 와서(curl), 알아서 실행(bash)해 주는 명령어입니다. 띵- 하고 완료되면 잘 설치됐는지 확인사살을 해야겠죠?

```bash
opencode --version

```

숫자가 예쁘게 출력되면 성공입니다! 만약 초보자분들 중에 `"command not found"`라는 빨간 에러가 뜨면서 협박한다면 당황하지 마세요. 환경 변수(PATH)에 `~/.local/bin` 경로가 안 잡혀서 그런 거니, 터미널을 껐다 켜거나 구글에 검색하면 1초 만에 해결됩니다. 자, 맥/리눅스 유저 설치 끝!

## SLIDE 06: Step 1 — opencode 설치 (Windows 유저 구원)

자, 이제 윈도우 사용자분들! 윈도우는 개발 세팅할 때 에러가 많이 나서 항상 서러우셨죠? 오늘 제가 아주 깔끔하게 구원해 드릴게요.

윈도우는 리눅스용 bash가 기본으로 없어서 맥이랑 똑같이 하면 터집니다. 대신 우리는 개발자들의 만능 보물창고인 'Chocolatey(초콜러티)'라는 패키지 매니저를 쓸 겁니다.

(화면의 Step A를 짚으며)

1. 먼저 시작 버튼에서 마우스 우클릭을 하고, 반드시 'Windows PowerShell'을 관리자 권한으로 실행해 주세요.

2. 그다면 화면에 보이는 복잡한 세 줄의 스크립트를 `chocolatey.org/install` 에서 복사해서 붙여넣고 엔터를 치세요.

```powershell
Set-ExecutionPolicy Bypass -Scope Process -Force
[System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072
iex ((New-Object System.Net.WebClient).DownloadString('https://chocolatey.org/install.ps1'))
```

(Step B를 가리키며)
이렇게 초콜러티 설치가 끝나면, 이제 진짜 opencode 설치는 초등학생도 할 수 있습니다. 딱 이 한 줄만 치세요.

```powershell
choco install opencode
```

그리고 `opencode --version`을 쳐서 확인해 보세요. 어때요? 윈도우에서도 완벽하게 엔진 설치가 끝났습니다. 이 초콜러티라는 녀석은 한 번 깔아두면 앞으로 다른 개발 도구 깔 때도 `choco install` 한 줄로 다 해결되니까 엄청 편하실 겁니다.

## SLIDE 07: VS Code + OpenChamber 확장 프로그램 설치

엔진을 장착했으니 이제 우리가 일하는 작업실인 **VS Code**와 연결해 주겠습니다. 터미널의 검은 화면이 두려웠던 주니어분들에게는 단비 같은 소식이죠.

방법은 윈도우 게임 설치보다 쉽습니다.

(하나씩 짚으며)

* **첫째,** 혹시 아직도 컴퓨터에 VS Code나 Cursor 에디터가 없다면, 지금 당장 `code.visualstudio.com`에 가서 다운받아 설치하세요. 그냥 '다음, 다음, 완료' 누르면 끝납니다.

* **둘째,** VS Code를 켜고 왼쪽 사이드바에 레고 블록처럼 생긴 **확장(Extensions) 아이콘**을 누르세요. (단축키 Ctrl+Shift+X)
검색창에 돋보기를 대고 "OpenChamber"를 검색한 뒤, 파란색 '설치(Install)' 버튼을 누르면 끝!

이렇게 하면, 마치 Cursor 에디터나 깃허브 코파일럿을 쓰는 것처럼, 익숙하고 편안하게 AI를 사용할 수 있는 환경이 구축됩니다.

AI가 설치하면서, 파일 생성에 관한 권한 승인을 요청합니다. 승인해 줘야 설치가 진행됩니다. 실수로 승인하지 않았으면 어쩌지? 걱정마세요. AI에게 다시 설치해달라고 요청하면 됩니다.

AI가 "주인님, 설치 끝났습니다!"라고 하면, AI-native Engineer는 어떻게 해야 하죠? 네, **그 산출물을 매서운 눈으로 U-A-E-A 검토하고 최종 '승인'해야 합니다.** 다음 슬라이드에서 그 검토 방법을 알려드릴게요.

## SLIDE 09: 설치 확인 — Cocrates = Plugin + Skill (인간의 검토)

AI가 지멋대로 세팅해 놓은 작업실을 검토해 볼 시간입니다.
Cocrates Harness가 완벽히 작동하려면 딱 두 가지만 눈으로 검증하시면 됩니다. 플러그인(Plugin)과 **스킬(Skill)**!

(좌측의 설정 파일 코드를 가리키며)

* **첫째, Plugin 검토:** 설정 파일인 `~/.config/opencode/opencode.jsonc` 파일을 메모장이나 VS Code로 열어보세요. 그 안에 AI가 `"plugin": ["@cocrates/cocrates-harness"]`라는 코드를 똑바로 집어넣었는지 눈으로 확인합니다. 있으면 1단계 패스!

(우측의 폴더 트리를 가리키며)

* **둘째, Skill 검토:** 설정 폴더인 `~/.config/opencode/skills/` 디렉토리를 열어보세요. 그 안에 `adr-writing`나 `spec-writing`, `education` 같은 스킬 폴더가 있는지 그 안에 `SKILL.md` 파일이 있는지 확인합니다.

(두 손을 모아 쾅 치며)
이 두 가지를 눈으로 확인하고, **"음, AI 네가 에러 없이 똑바로 설치했군. 승인한다!"** 하고 도장을 찍는 순간, 비로소 설치가 끝나는 겁니다.

AI에게 일은 시키되, 검토와 책임은 인간 리더가 진다! U-A-E-A의 짜릿한 손맛이죠. 이제 준비가 끝났으니 opencode를 가볍게 재시작하고 에이전트 목록에서 'Cocrates Agent'를 활성화해 주세요.
