# Script: Slide 03 — "jsondb를 만들어줘" — 하지만 AI는 설계부터 권했다

**Time:** ~1.5분

**[SLIDE 03: 일반 AI vs Cocrates 비교]**

**좌측: 일반 AI**
"jsondb? 알겠습니다!" → 즉시 수백 줄 코드 생성.

**우측: 🦉 Cocrates**
"이 jsondb는 어떤 용도인가요? 아키텍처를 먼저 설계하고 검토받은 후 생성하겠습니다."

**핵심:** Cocrates는 설계 없는 생성을 거부한다 (Ep2 원칙).

---

자, 이제 실제 상황을 한번 볼까요?

여러분이 Cocrates에게 이렇게 요청했다고 상상해보세요. "jsondb를 개발해 줘."

일반적인 AI라면 어떻게 반응할까요? 아마 이럴 겁니다. "jsondb? 알겠습니다!" 하고는, 바로 수백 줄의 코드를 쏟아낼 거예요. 검토도 없이, 설계도 없이, 그냥 코드부터 만들어버립니다.

하지만 Cocrates는 다릅니다. Cocrates는 이렇게 응답합니다.

"이 jsondb는 어떤 용도로 쓰실 예정인가요? 아키텍처를 먼저 설계하고, 사용자님께 검토와 승인을 받은 뒤에 안전하게 코드를 생성하겠습니다."

(잠시 멈춤)

여기에 Cocrates의 핵심 가치가 담겨 있습니다. Ep2에서 선언했던 원칙을 기억하시나요? **"검토되지 않은 산출물은 생성할 가치가 없다."** Cocrates는 이 원칙을 지키는 AI입니다. 설계 없이, 검토 없이 코드를 만들어내지 않습니다.

오늘 만들 것은 jsondb. 아주 간단해 보이는 로컬 JSON 저장소입니다. 하지만 이 단순한 프로젝트조차도 **여러가지 아키텍처 논쟁**이 필요했습니다.

그럼 그 논쟁의 현장으로 함께 가보시죠.
