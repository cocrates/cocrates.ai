# 01. 들어가며 — "나는 내가 모르는 것을 안다"

철학자 소크라테스는 이런 말을 남겼습니다.

> "I know that I know nothing."
> (나는 내가 아무것도 모른다는 것을 안다.)

이 말은 단순한 겸손이 아닙니다. **자신이 무엇을 모르는지 정확히 아는 것** — 이것이 모든 학습의 출발점입니다. 자신의 무지를 인식하지 못하면, 무엇을 배워야 할지도 알 수 없기 때문입니다.

## 당신은 한글 창제에 대해 얼마나 알고 있나요?

잠시 생각해보세요.

- "한글은 누가 만들었지?" — **세종대왕**
- "언제 만들어졌지?" — **1443년**
- "한글은 어떻게 만들어졌을까?" — **글쎄... 잘 모르겠다**

이렇게 자신이 **어디까지 알고 있고, 어디부터 모르는지**를 아는 것, 그것이 바로 블룸 분류학이 가르쳐주는 가장 높은 수준의 지식입니다.

## 이 문서를 통해 당신은

이 문서는 블룸 분류학(Bloom's Taxonomy)이라는 **학습 목표 분류 도구**를 소개합니다. 블룸 분류학은 1956년 Benjamin Bloom이 처음 제안하고, 2001년 Anderson과 Krathwohl이 개정한 이론으로, **"무엇을 알고 있는가"** 와 **"어느 정도로 알고 있는가"** 를 함께 분석할 수 있는 2차원 프레임워크입니다.

이 문서를 읽고 나면 당신은:

1. 블룸 분류학의 **2차원 구조**(지식 차원 × 인지 과정 차원)를 설명할 수 있습니다
2. **한글 창제**라는 구체적인 사례를 분류 체계의 각 위치에 매핑할 수 있습니다
3. **자신의 현재 지식 수준**을 진단할 수 있습니다
4. 분류 체계를 활용해 **구체적인 학습 목표**를 설정할 수 있습니다

# 02. 블룸 분류학 — 2차원 프레임워크

"무엇을 알고 있는가"와 "어느 정도로 알고 있는가" — 이 두 질문을 합치면 블룸 분류학의 핵심이 됩니다.

## 02-01. 블룸 분류학 개요

### 블룸 분류학이란?

블룸 분류학(Bloom's Taxonomy)은 교육 목표를 체계적으로 분류하는 프레임워크입니다. 1956년 교육심리학자 Benjamin Bloom이 처음 제안했고, 2001년 Anderson과 Krathwohl이 인지 과정의 재정의와 지식 차원의 추가라는 중요한 개정을 했습니다. 우리가 사용하는 것은 이 **개정판(2001)** 입니다.

### 왜 알아야 할까?

블룸 분류학을 알면:

- **학습 목표를 명확히 설정**할 수 있습니다: "나는 00을 00할 수 있게 되고 싶다"
- **자신의 현재 수준을 진단**할 수 있습니다: "지금 나는 어디쯤 있는가?"
- **더 나은 학습 전략**을 설계할 수 있습니다: "목표에 도달하려면 무엇이 필요할까?"

### 2차원 구조

블룸 분류학의 개정판이 주목받는 이유는 바로 **2차원 구조** 때문입니다. 하나의 축은 **지식 차원**(Knowledge Dimension) — "무엇을 알고 있는가"이고, 다른 축은 **인지 과정 차원**(Cognitive Process Dimension) — "어느 정도로 알고 있는가"입니다. 이 두 축을 교차하면 총 **4 × 6 = 24개의 셀**로 이루어진 매트릭스(Taxonomy Table)가 만들어집니다.

| | **A. 사실적** | **B. 개념적** | **C. 절차적** | **D. 메타인지** |
|---|---|---|---|---|
| **1. 기억** | | | | |
| **2. 이해** | | | | |
| **3. 적용** | | | | |
| **4. 분석** | | | | |
| **5. 평가** | | | | |
| **6. 창안** | | | | |

**학습 목표의 표현:** 블룸 분류학에서 학습 목표는 **동사(인지 과정)** + **목적어(지식 유형)** 형태로 표현합니다. 예를 들어:
- "한글의 제자 원리를 **설명할 수 있다**" → 설명하다(이해, B) + 제자 원리(개념적 지식, 2) → **B2 셀**
- "한글 창제 연도를 **말할 수 있다**" → 말하다(기억, A) + 1443년(사실적 지식, 1) → **A1 셀**

## 02-02. 지식 차원 (Knowledge Dimension)

지식 차원은 **무엇을** 아는가를 구분합니다. 4가지 유형이 있으며, **구체적인 것에서 추상적인 것** 순서로 나열됩니다.

| 유형 | 질문 | 본질 | 난이도 |
|------|------|------|--------|
| **A. 사실적 (Factual)** | "무엇인가?" | 낱낱의 정보 조각 | 비교적 쉬움 |
| **B. 개념적 (Conceptual)** | "왜 그런가?" | 요소 간 관계와 체계 | 보통 |
| **C. 절차적 (Procedural)** | "어떻게 하는가?" | 방법과 기능, 기술 | 보통 |
| **D. 메타인지 (Metacognitive)** | "내가 무엇을 알고 있는가?" | 앎에 대한 앎 | **가장 어려움** |

### 관계: 구체 → 추상의 연속선

이 네 지식 유형은 단순한 목록이 아니라 **계층 관계**를 가집니다:

- **사실적 지식**이 가장 구체적입니다. "1443년", "28자" 같은 개별 정보입니다.
- **개념적 지식**은 사실들을 연결합니다. 상형·가획·합자 원리는 여러 사실(ㄱ은 혀뿌리 모양, ㄴ은 혀 모양...)을 하나의 체계로 묶습니다.
- **절차적 지식**은 개념을 실행으로 옮깁니다. 제자 원리를 안다는 것(개념)과 실제로 한글을 쓸 수 있다는 것(절차)은 다릅니다.
- **메타인지 지식**은 가장 추상적입니다. 위의 세 가지 지식에 대해 **"내가 그것을 얼마나 알고 있는가"** 라는 2차 질문을 던집니다.

> ⚠️ **메타인지 지식 주의:** 지금은 "내가 아는 것에 대해 아는 것"이라는 정의만 기억해두세요. 06장에서 자세히 다룹니다.

## 02-03. 인지 과정 차원 (Cognitive Process Dimension)

인지 과정 차원은 **어느 정도로** 알고 있는가를 구분합니다. 총 6단계이며, **하위 단계에서 상위 단계**로 이어집니다.

| 단계 | 1문장 정의 | 핵심 동사 예 |
|------|-----------|------------|
| **1. 기억 (Remember)** | 장기기억에서 관련 지식을 인출한다 | 말하다, 쓰다, 열거하다 |
| **2. 이해 (Understand)** | 정보의 의미를 구성한다 | 설명하다, 해석하다, 요약하다 |
| **3. 적용 (Apply)** | 주어진 상황에서 절차를 수행한다 | 실행하다, 사용하다, 적용하다 |
| **4. 분석 (Analyze)** | 전체를 부분으로 나누고 관계를 파악한다 | 비교하다, 대조하다, 분석하다 |
| **5. 평가 (Evaluate)** | 기준에 따라 판단한다 | 평가하다, 비판하다, 판단하다 |
| **6. 창안 (Create)** | 요소들을 새롭게 조합해 하나의 전체를 만든다 | 설계하다, 창안하다, 구성하다 |

### 관계: 하위 → 상위, 상위는 하위를 포함

인지 과정 단계는 단순한 단계별 목록이 아닙니다. 상위 단계는 **하위 단계를 포함**합니다.

- "한글의 제자 원리를 **평가(5)** 하려면" 먼저 그 원리를 **기억(1)** 하고, **이해(2)** 하고, **분석(4)** 할 수 있어야 합니다.
- 하위 단계 없이 상위 단계에 도달할 수 없습니다. 창안(6) 수준의 학습은 모든 하위 단계의 인지 과정을 통합적으로 요구합니다.

그러나 실제 학습이 반드시 1→2→3→4→5→6의 **선형적 순서**로만 일어나는 것은 아닙니다. 학습자는 어떤 단계에서 시작하든 필요에 따라 여러 단계를 오가며 학습할 수 있습니다.

블룸 분류학은 두 차원(지식 × 인지 과정)의 조합으로 학습 목표를 정밀하게 정의합니다. 03장부터는 각 지식 차원을 한글 창제 사례를 통해 깊이 탐구해보겠습니다.

# 03. 사실적 지식 (Factual Knowledge) — 기본을 알자

> *"모든 학습은 '무엇인가'라는 사실로부터 출발한다. 하지만 거기에 머물러서는 안 된다."*

## 사실적 지식이란?

**사실적 지식(Factual Knowledge)** 은 학문 분야의 기본 요소, 즉 낱낱의 정보 조각에 대한 지식입니다. Anderson과 Krathwohl(2001)은 사실적 지식을 두 가지 하위 유형으로 나눕니다:

- **용어에 대한 지식 (Knowledge of Terminology):** 언어적·비언어적 기호(용어, 명칭, 레이블). 예: "훈민정음", "상형", "가획"
- **구체적 사실과 요소에 대한 지식 (Knowledge of Specific Details and Elements):** 사건, 인물, 장소, 시기 등. 예: "1443년 창제", "세종대왕"

사실적 지식은 "**무엇?** (What?)"이라는 질문에 답합니다. "한글을 누가 만들었나요?" → "세종대왕." "몇 자였나요?" → "28자."

### 한글 창제의 핵심 사실들

한글 창제를 이해하기 위해 알아야 할 기본 사실들은 다음과 같습니다:

| 분류 | 사실 | 비고 |
|------|------|------|
| **인물** | 세종대왕(1397-1450), 집현전 학자들 | 정인지, 박팽년, 신숙주 등 |
| **시기** | 1443년 창제(훈민정음 반포), 1446년 반포(훈민정음 해례본) | 창제→반포 사이 3년의 정비 기간 |
| **문자** | 초성 17자, 중성 11자 = 총 28자 | 현대 한글은 24자(자음 14, 모음 10) |
| **문헌** | 《훈민정음 해례본》, 《세종실록》 | 해례본은 1997년 유네스코 세계기록유산 등재 |
| **저술** | 《훈민정음》(해례본) — 정인지 서문 | 한문으로 쓰인 한글 해설서 |

## 인지 과정별로 살펴보는 사실적 지식

이제 이 사실들을 블룸 분류학의 인지 과정 6단계에 적용해보겠습니다.

### A1. 기억 (Remember) — "한글 창제 연도를 말할 수 있다"

가장 기본적인 수준입니다. 단순히 정보를 **인출**하는 능력입니다.

예시:
> "한글은 몇 년도에 만들어졌나요?"
> "1443년입니다."

학습 목표: "한글 창제 연도를 **말할 수 있다**" — 말하다(기억) + 1443년(사실적 지식) = **A1 셀**

### A2. 이해 (Understand) — "각 사실의 의미를 이해한다"

단순 암기를 넘어, 그 사실이 **무엇을 의미하는지** 아는 수준입니다.

예시:
> "1443년에 창제되었다는 것은, 세종이 즉위한 지 25년째 되는 해라는 뜻이다. 당시 조선은 백성들의 문맹률이 매우 높았고, 세종은 백성을 위한 문자를 오랫동안 연구해왔다."

학습 목표: "한글 창제 연도가 갖는 역사적 의미를 **설명할 수 있다**" — 설명하다(이해) + 연도의 의미(사실적 지식) = **A2 셀**

### A3. 적용 (Apply) — "한글 창제 타임라인을 그릴 수 있다"

배운 사실들을 활용해 **무언가를 실행**하는 수준입니다.

예시:
> "세종의 생애와 주요 업적을 타임라인으로 그려보세요. 1397년 출생, 1418년 즉위, 1443년 한글 창제, 1446년 해례본 반포, 1450년 승하 — 각 사건을 시간 순서대로 배치하고, 그 사이의 간격을 계산해보세요."

학습 목표: "한글 창제 관련 역사적 사건의 타임라인을 **작성할 수 있다**" — 작성하다(적용) + 타임라인(사실적 지식) = **A3 셀**

### A4. 분석 (Analyze) — "두 역사 기록의 차이를 분석할 수 있다"

사실들을 **비교하고 대조**하며, 요소 간의 관계와 패턴을 파악하는 수준입니다.

예시:
> "《세종실록》과 《훈민정음 해례본》의 기록을 비교해보세요. 두 문헌에서 창제 연도가 어떻게 기록되어 있는가? 차이가 있다면 그 이유는 무엇일까?"

학습 목표: "한글 창제에 대한 두 역사 기록의 차이점을 **분석할 수 있다**" — 분석하다(분석) + 역사 기록(사실적 지식) = **A4 셀**

### A5. 평가 (Evaluate) — "역사 기록의 신뢰성을 평가할 수 있다"

사실들을 **판단의 기준**에 따라 평가하는 수준입니다.

예시:
> "《세종실록》에 기록된 한글 창제 과정은 얼마나 신뢰할 수 있는가? 누가, 언제, 어떤 목적으로 기록했는지를 고려해 기록의 객관성과 신뢰성을 판단해보세요."

학습 목표: "한글 창제 관련 역사 기록의 신뢰성을 **평가할 수 있다**" — 평가하다(평가) + 기록의 신뢰성(사실적 지식) = **A5 셀**

### A6. 창안 (Create) — "한글 역사 연표를 직접 만들 수 있다"

여러 사실들을 **새롭게 조합**해 하나의 완성된 산출물을 만드는 수준입니다.

예시:
> "지금까지 학습한 사실들을 바탕으로, 한글 창제부터 현대 한글까지의 '한글 600년 역사 연표'를 직접 만들어보세요. 각 시대별 주요 사건, 인물, 변화를 포함하고 시각적으로 표현해보세요."

학습 목표: "한글의 역사를 보여주는 연표를 **설계할 수 있다**" — 설계하다(창안) + 역사 연표(사실적 지식) = **A6 셀**

# 04. 개념적 지식 (Conceptual Knowledge) — 원리를 알자

> *"낱낱의 사실을 넘어, 그것들이 어떻게 관계 맺고 체계를 이루는지 아는 것 — 그것이 개념적 지식이다."*

**관계 환기:** 03장에서 배운 사실들(28자, 1443년, 세종대왕)이 이 장에서는 **원리**(상형·가획·합자)로 연결됩니다. 예를 들어 "ㄱ은 혀뿌리 모양을 본떴다(상형)"는 하나의 낱낱 사실처럼 보이지만, 이것은 '상형'이라는 원리 아래 하나로 묶이는 **개념적 관계**입니다.

## 개념적 지식이란?

**개념적 지식(Conceptual Knowledge)** 은 한 영역 내에서 기본 요소들이 서로 어떻게 관계 맺고, 체계를 이루며, 구조화되는지에 대한 지식입니다. 세 가지 하위 유형이 있습니다:

- **분류와 범주에 대한 지식 (Knowledge of Classifications and Categories):** "오음(五音) 분류 — ㄱ(아음), ㄴ(설음), ㅁ(순음), ㅅ(치음), ㅇ(후음)"
- **원리와 일반화에 대한 지식 (Knowledge of Principles and Generalizations):** "상형(象形) — 발음 기관의 모양을 본떠 글자를 만들었다"
- **이론, 모형, 구조에 대한 지식 (Knowledge of Theories, Models, and Structures):** "천지인(天地人) 삼재 사상 — 중성은 하늘·땅·사람을 본떴다"

개념적 지식은 "**왜?** (Why?)"라는 질문에 답합니다. "왜 한글은 ㄱ, ㄴ, ㅁ, ㅅ, ㅇ으로 시작할까?" → "발음 기관의 모양을 본떴기 때문이다(상형 원리)."

### 한글 창제의 핵심 개념들

| 개념 | 설명 | 한글 사례 |
|------|------|-----------|
| **상형 (象形)** | 발음 기관의 모양을 본뜸 | ㄱ(혀뿌리), ㄴ(혀), ㅁ(입), ㅅ(이), ㅇ(목구멍) |
| **가획 (加劃)** | 기본 글자에 획을 더해 소리의 차이 표현 | ㄱ→ㅋ→ㆁ, ㄴ→ㄷ→ㅌ, ㅁ→ㅂ→ㅍ |
| **합자 (合字)** | 초·중·종성을 조합해 음절 구성 | ㄱ+ㅏ+ㅁ = 감 |
| **오음 (五音)** | 발음 위치에 따른 다섯 가지 분류 | 아음·설음·순음·치음·후음 |
| **삼재 (三才)** | 하늘·땅·사람 — 중성의 철학적 기반 | ㆍ(하늘), ㅡ(땅), ㅣ(사람) |
| **연서·병서·부서** | 자음을 결합하는 추가 규칙 | 연서(ㅅㅅ→ㅆ), 병서(ㄱㅅ), 부서(ㄹㅂ) |

## 인지 과정별로 살펴보는 개념적 지식

### B1. 기억 (Remember) — "제자 원리의 정의를 말할 수 있다"

예시:
> "상형(象形)이 무엇인가요?"
> "발음 기관의 모양을 본떠 글자를 만든 원리입니다."

학습 목표: "상형·가획·합자 원리의 **정의를 말할 수 있다**" = **B1 셀**

### B2. 이해 (Understand) — "제자 원리의 '왜'를 설명할 수 있다"

가장 중요한 전환점입니다. 단순히 원리의 이름을 기억하는 것을 넘어, **왜 그런 원리가 생겼는지**를 이해합니다.

예시:
> "'ㄱ'이 왜 저렇게 생겼는지 설명할 수 있나요?"
> "네, 'ㄱ'은 혀뿌리가 목구멍을 막는 모양을 본떴습니다. 발음할 때 혀뿌리가 목젖 쪽으로 올라가면서 공기를 막는 그 모양 자체를 글자로 만든 것입니다."

학습 목표: "상형 원리가 각 글자 모양에 어떻게 적용되었는지 **설명할 수 있다**" = **B2 셀**

### B3. 적용 (Apply) — "원리로 새 글자의 소리를 추론할 수 있다"

원리를 **실제 상황에 적용**하는 수준입니다. 새로운 글자나 상황에서 배운 원리를 사용할 수 있어야 합니다.

예시:
> "만약 'ㅎ'이 후음(목구멍 소리)이라면, 어떤 모양일지 상형 원리에 따라 추론해보세요. 그리고 왜 그렇게 생각했는지 설명해보세요."

학습 목표: "제자 원리를 활용해 낯선 글자의 소리를 **추론할 수 있다**" = **B3 셀**

### B4. 분석 (Analyze) — "한글과 한자의 문자적 특성을 비교·분석할 수 있다"

여러 개념 체계를 **비교하고 관계를 분석**하는 수준입니다.

예시:
> "한글과 한자는 각각 어떤 원리로 만들어졌나요? 두 문자 체계를 **비교**해보세요. 문자와 소리의 관계, 학습 난이도, 표현의 정밀성 측면에서 어떤 차이가 있나요?"

학습 목표: "한글과 한자의 문자적 특성을 **비교·분석할 수 있다**" = **B4 셀**

심화 분석 예시: "최만리는 왜 한글 창제를 반대했을까?" — 당시 한자 중심의 지식 체계와 충돌한 한글의 혁신성을 분석.

### B5. 평가 (Evaluate) — "한글의 독창성을 평가 기준에 따라 판단할 수 있다"

**평가 기준**을 세우고 그에 따라 개념 체계의 가치를 판단하는 수준입니다.

예시:
> "한글이 '가장 과학적인 문자'라는 주장을 평가해보세요. 평가 기준(표현의 정밀성, 학습 용이성, 체계의 일관성 등)을 먼저 세우고, 각 기준에 따라 한글을 평가한 후, 결론을 내려보세요."

학습 목표: "한글의 과학성을 평가 기준에 따라 **판단할 수 있다**" = **B5 셀**

### B6. 창안 (Create) — "상형·가획·합자 원리로 새 문자 체계를 설계할 수 있다 ★"

이것이 개념적 지식에서 가장 높은 수준입니다. 배운 원리들을 **새롭게 조합**해 전혀 새로운 체계를 만들어냅니다.

예시:
> "만약 지구 외계인이 사용하는 언어를 표현할 문자 체계를 설계해야 한다면, 한글의 제자 원리(상형·가획·합자)를 어떻게 활용할 수 있을까? 외계인의 발음 기관을 가정하고(상형), 소리의 차이를 구분하는 방법(가획)과 소리를 조합하는 규칙(합자)을 설계해보세요."

학습 목표: "한글의 제자 원리를 바탕으로 새로운 문자 체계를 **설계할 수 있다**" = **B6 셀**

# 05. 절차적 지식 (Procedural Knowledge) — 방법을 알자

> *"원리를 아는 것(개념)에서 더 나아가, 실제로 실행할 수 있는 것 — 그것이 절차적 지식이다."*

**관계 환기:** 03장의 사실(28자), 04장의 원리(상형·가획·합자)가 이 장에서는 **실제로 읽고 쓰는 절차**로 이어집니다. 상형이 무엇인지 설명할 수 있는 것과, 실제로 한글을 읽고 쓸 수 있는 것은 다른 능력입니다. 바로 이 "할 수 있는 것"이 절차적 지식입니다.

## 절차적 지식이란?

**절차적 지식(Procedural Knowledge)** 은 **무언가를 하는 방법**, 즉 기능, 기술, 알고리즘, 방법론에 대한 지식입니다. 세 가지 하위 유형이 있습니다:

- **구체적 기능과 알고리즘에 대한 지식 (Knowledge of Subject-Specific Skills and Algorithms):** "초성+중성+종성을 합자하여 한 음절을 만드는 방법"
- **기법과 방법에 대한 지식 (Knowledge of Subject-Specific Techniques and Methods):** "한글 맞춤법 규칙에 따라 글을 쓰는 방법"
- **적절한 절차 사용 시기에 대한 지식 (Knowledge of Criteria for Determining When to Use Appropriate Procedures):** "어떤 상황에서 연서를 쓰고, 어떤 상황에서 병서를 써야 하는지"

절차적 지식은 "**어떻게?** (How?)"라는 질문에 답합니다. "한글로 '감'을 어떻게 쓰나요?" → "ㄱ + ㅏ + ㅁ을 왼쪽에서 오른쪽, 위에서 아래로 합자합니다."

### 한글의 핵심 절차들

| 절차 | 설명 | 예시 |
|------|------|------|
| **합자 순서** | 초성→중성→종성 순서로 결합 | ㄱ+ㅏ+ㅁ = 감 |
| **모아쓰기** | 자모를 한 음절로 조합 | ㅎ+ㅏ+ㄴ=한, ㄱ+ㅡ+ㄹ=글 |
| **연서법** | 같은 자음을 겹쳐 쓰기 | ㅅ+ㅅ = ㅆ (쌍시옷) |
| **병서법** | 다른 자음을 나란히 쓰기 | ㄱ+ㅅ (한글 초기, 현대에는 거의 사용 안 함) |
| **맞춤법 규칙** | 표준 발음과 표기법 일치 규칙 | "한글" → [한글] 발음과 일치 |
| **외래어 표기법** | 외국어를 한글로 옮기는 절차 | computer → 컴퓨터 |

## 인지 과정별로 살펴보는 절차적 지식

### C1. 기억 (Remember) — "합자 순서를 기억한다"

예시:
> "한글 음절을 만들 때 초성, 중성, 종성을 어떤 순서로 결합하나요?"
> "초성 다음에 중성, 마지막에 종성을 결합합니다."

학습 목표: "초·중·종성의 **합자 순서를 말할 수 있다**" = **C1 셀**

### C2. 이해 (Understand) — "모아쓰기 원리를 이해한다"

단순 순서 암기를 넘어, **왜 그런 순서인지**, 각 단계의 **의미**를 이해하는 수준입니다.

예시:
> "왜 한글은 자모를 풀어쓰지 않고 모아쓰기를 할까요? 모아쓰기가 음절 단위 인식에 어떤 이점을 주는지 설명해보세요."

학습 목표: "초·중·종성이 음절을 형성하는 **과정을 이해한다**" = **C2 셀**

### C3. 적용 (Apply) — "한글 맞춤법 규칙에 따라 글을 쓸 수 있다"

절차적 지식의 핵심입니다. 배운 절차를 **실제로 수행**할 수 있어야 합니다.

예시:
> "다음 문장을 한글 맞춤법 규칙에 따라 올바르게 써보세요."
> "한글은 세종대왕이 창제하신 과학적인 문자입니다."
> (직접 써보기 — 실제로 손이나 키보드로 작성)

학습 목표: "한글 맞춤법 규칙에 따라 **글을 쓸 수 있다**" = **C3 셀**

### C4. 분석 (Analyze) — "현대와 당시 표기법의 차이를 분석할 수 있다"

여러 절차와 방법을 **비교하고, 차이를 식별**하는 수준입니다.

예시:
> "15세기 훈민정음 당시의 표기법과 현대 한글 맞춤법을 비교해보세요. 어떤 차이가 있는가? 받침 표기, 연서·병서 사용, 띄어쓰기 측면에서 구체적인 예를 들어 분석하세요."

핵심 차이점 예시:
- 당시에는 **연서·병서·부서**가 활발히 사용됨
- 현대에는 **된소리(ㄲ, ㄸ, ㅃ, ㅆ, ㅉ)** 만 연서 형태로 남음
- 띄어쓰기는 현대에 정착된 규칙

학습 목표: "현대와 15세기 한글 표기법의 차이를 **분석할 수 있다**" = **C4 셀**

### C5. 평가 (Evaluate) — "표기 방식의 적절성을 평가할 수 있다"

여러 절차 중 **어느 것이 상황에 더 적합한지** 판단하는 수준입니다.

예시:
> "한글을 처음 배우는 외국인 학습자에게는 어떤 표기 방식(풀어쓰기 vs 모아쓰기)이 더 효과적일까? 학습자의 인지적 부담, 읽기 효율성, 발음 학습 용이성 등을 기준으로 평가해보세요."

학습 목표: "상황에 따른 표기 방식의 적절성을 **평가할 수 있다**" = **C5 셀**

### C6. 창안 (Create) — "새로운 표기 규칙을 제안할 수 있다"

기존의 절차를 **새로운 방식으로 조합**하거나, 전혀 새로운 절차를 고안하는 수준입니다.

예시:
> "스마트폰 시대에 더 효율적인 한글 입력 방식을 제안해보세요. 또는 '한국어 발음을 가장 정확히 표현할 수 있는 새로운 로마자 표기법'을 설계해보세요. 기존 표기법(국어의 로마자 표기법, 매큔-라이샤워 표기법)의 한계를 분석하고, 이를 극복할 새로운 절차를 고안하세요."

학습 목표: "기존과 다른 새로운 표기 규칙을 **제안할 수 있다**" = **C6 셀**

# 06. 메타인지 지식 (Metacognitive Knowledge) — 나를 알자

> *"아는 것(사실), 관계(개념), 방법(절차)을 넘어, 내가 무엇을 알고 무엇을 모르는지 아는 것 — 이것이 가장 높은 지식 차원이다."*

> ⚠️ **이 장은 이 문서에서 가장 도전적인 내용을 다룹니다.** 메타인지는 블룸 분류학의 네 가지 지식 차원 중 가장 이해하기 어려운 개념입니다. 한 번에 완벽히 이해되지 않아도 괜찮습니다. 비유와 사례를 따라 천천히 읽어보세요. 03~05장의 내용을 떠올리며 "나는 이것을 얼마나 알고 있나?"라고 스스로 질문해보는 것만으로도 이 장의 반은 온 것입니다.

**관계 환기 (가장 중요):** 지금까지 배운 지식들(03의 사실, 04의 개념, 05의 절차)은 모두 **'1차 지식'** — 즉 어떤 대상 자체에 대한 지식이었습니다. 메타인지 지식은 이들과 **근본적으로 다릅니다.** 메타인지는 '1차 지식'을 대상으로 삼는 **'2차 지식'** 입니다.

즉, 메타인지 지식은 독립된 내용이 아니라 03·04·05장에서 배운 모든 지식을 바탕으로 **"내가 그것(사실·개념·절차)을 아는가?"** 라고 질문합니다.

## 06-01. 메타인지 지식이란? — "앎에 대한 앎"

> *"메타인지는 '무엇을 아는 것'이 아니라, '내가 그것을 안다는 사실'을 아는 것이다."*

**'메타(meta)'** 는 그리스어로 **'~에 관한'** 이라는 뜻입니다. 따라서 메타인지는 **"인지에 관한 인지", "생각에 대한 생각"** 입니다.

### 1차 지식 vs 2차 지식

| 구분 | 1차 지식 (03·04·05) | 2차 지식 (06 — 메타인지) |
|------|-------------------|------------------------|
| **내용** | "1443년에 세종이 한글을 창제했다" | "**나는** 그 사실을 안다" 또는 "**나는** 그 사실을 모른다" |
| **초점** | 대상(사실, 개념, 절차) 자체 | 자신이 그 대상을 **아는지 모르는지** |
| **질문** | "한글은 언제 만들어졌나?" | "내가 한글 창제 연도를 진짜 아는가?" |

### 비유로 이해하기: 지도와 탐험가

> **지식 자체는 '지도'와 같습니다.** 한글 창제라는 영역의 지도(03 사실, 04 개념, 05 절차)가 있다고 합시다.
>
> **메타인지는 '지도 위에 자신의 위치를 표시하는 탐험가'와 같습니다.** 지도를 보고 "지금 나는 여기 있다"고 아는 것, "이 부분은 아직 탐험하지 못했다"고 아는 것, "이 길은 잘 알지만 저 길은 자신 없다"고 아는 것 — 이것이 메타인지입니다.
>
> 지도가 아무리 완벽해도, 그 위에 자신의 위치를 모른다면 길을 잃습니다. 반대로 지도가 조금 부족해도 자신의 위치를 정확히 안다면, 무엇을 더 찾아야 할지 알 수 있습니다.

### 소크라테스: "I know that I know nothing"

다시 01장의 소크라테스로 돌아옵니다. 소크라테스가 한 말을 다시 음미해보세요:

> "나는 내가 아무것도 모른다는 것을 안다."

여기서 **'안다'는 동사가 두 번 등장**합니다. "아무것도 모른다"는 1차 지식에 대한 진술입니다. "그것을 안다"는 2차 지식, 즉 **메타인지**입니다. 소크라테스는 단순히 겸손한 것이 아닙니다. 그는 자신의 지식의 경계를 정확히 인식하고 있었던 것입니다.

> 이것이 바로 메타인지 지식의 본질입니다.

## 06-02. 메타인지 지식의 세 가지 하위 유형

Anderson과 Krathwohl(2001)은 메타인지 지식을 세 가지 하위 유형(D1, D2, D3)으로 세분화합니다. 각각을 한글 창제 사례와 함께 자세히 살펴보겠습니다.

### D1. 전략적 지식 (Strategic Knowledge)

#### 정의

학습, 사고, 문제 해결을 위한 **방법과 전략**에 대한 지식입니다. "어떻게 하면 더 잘 배울 수 있을까?"라는 질문에 답합니다. 이는 05장의 절차적 지식(어떻게 하는가)과 비슷해 보이지만, 결정적인 차이는 **'무엇을 배우는가'가 아니라 '어떻게 배우는가'에 초점**을 둔다는 점입니다. 절차적 지식이 과제 수행 절차라면, 전략적 지식은 **학습과 사고 자체의 절차**입니다.

#### 한글 창제 사례

| 전략 유형 | 설명 | 한글 예시 |
|-----------|------|-----------|
| **암기 전략** | 정보를 기억하기 위한 방법 | "한글 28자를 외울 때 **연상법**이 효과적이다. 'ㄱ'은 혀뿌리가 막히는 모양, 'ㄴ'은 혀가 입천장에 닿는 모양을 떠올리자" |
| **정교화 전략** | 새로운 정보를 기존 지식과 연결 | "상형 원리를 이해하려면 **발음 기관 모양을 직접 손가락으로 가리키며** 글자를 보자" |
| **조직화 전략** | 정보를 구조화하여 이해 | "제자 원리를 상형·가획·합자로 **분류해 표로 정리**하면 각 원리의 관계가 더 잘 보인다" |
| **초인지 전략** | 자신의 이해도를 점검하며 학습 | "내가 제자 원리를 진짜 이해했는지 **스스로에게 설명해보고 막히는 부분을 확인**한다" |

### D2. 인지 과제에 관한 지식 (Knowledge about Cognitive Tasks)

#### 정의

각 **과제의 성격과 요구 조건**에 대한 지식입니다. "이 과제는 얼마나 어려운가?", "이 과제는 어떤 인지적 노력이 필요한가?"라는 질문에 답합니다.

#### 한글 창제 사례

| 인지 | 설명 | 한글 예시 |
|------|------|-----------|
| **난이도 인식** | 과제의 상대적 난이도를 앎 | "한글 창제 연도를 **기억하는 것(A1)** 보다 제자 원리를 **설명하는 것(B2)** 이 훨씬 더 어렵다. 후자는 전체 체계에 대한 이해가 필요하기 때문이다" |
| **전략 선택** | 과제 성격에 맞는 전략 선택 | "사실 암기(A1·A2)에는 **반복 학습과 플래시카드**가 효과적이지만, 원리 이해(B2)에는 **예시 만들기와 스스로 설명하기**가 더 효과적이다" |
| **맥락 인식** | 과제가 요구하는 맥락 이해 | "한국어 화자에게는 한글 제자 원리가 **자연스러운 배경 지식**에 가깝지만, 외국인 학습자에게는 **처음부터 체계적으로 학습해야 할 새로운 지식**이다. 같은 과제라도 대상에 따라 접근이 달라져야 한다" |

### D3. 자기 자신에 대한 지식 (Self-Knowledge)

#### 정의

자신의 **인지적 강점과 약점, 동기, 신념**에 대한 지식입니다. "나는 어떤 유형의 학습자인가?", "나는 무엇을 잘하고, 무엇을 못하는가?"라는 질문에 답합니다.

#### 한글 창제 사례

| 자기 인식 유형 | 설명 | 한글 예시 |
|---------------|------|-----------|
| **강점·약점 인식** | 자신의 인지적 프로필 파악 | "나는 **사실 암기는 잘하지만**(연도, 인물, 숫자 — 강점), **원리 설명은 약하다**(상형·가획·합자의 관계 설명 — 약점). 따라서 04장 개념적 지식에 더 많은 시간을 투자해야 한다" |
| **자기 효능감** | 자신이 할 수 있다는 신념 | "제자 원리 **이해(B2)는** 할 수 있겠지만, 새로운 문자 체계 **설계(B6 창안)는** 아직 자신이 없다. 먼저 B2~B4를 충분히 익힌 후에 도전하자" |
| **동기 인식** | 자신이 왜 배우는지 아는 것 | "나는 단순히 한글에 대해 **아는 것을 넘어**, 한글이 왜 세계적으로 과학적인 문자로 인정받는지 **증명할 수 있을 정도로** 알고 싶다. 이것이 나의 학습 동기다" |

## 06-03. 메타인지 지식을 학습 목표로 삼는다는 것

> *"자신의 지식 상태를 정확히 아는 것 — 이것이 최고 수준의 학습 목표다."*

블룸 분류학에서 6단계 인지 과정의 최상위는 **창안(Create)** 입니다. 하지만 지식 차원에서 가장 높은 것은 **메타인지(Metacognitive)** 입니다. 이 둘이 만나는 지점인 **D6(메타인지 지식의 창안 수준)** 이 이론상 가장 높은 학습 목표입니다.

하지만 더 중요한 것은 이것입니다:

> **자신이 무엇을 알고 무엇을 모르는지 정확히 아는 것은, 단순히 더 많이 아는 것보다 더 높은 차원의 학습이다.**

### 메타인지가 최고 수준인 네 가지 이유

**1. 통합적이다**
메타인지 지식은 사실적·개념적·절차적 지식을 모두 **통합적으로 조망**해야 가능합니다. 예를 들어 "나는 한글의 제자 원리(개념적)를 설명할 수 있지만, 실제로 외국인에게 가르칠(절차적) 자신은 없다"는 판단은 두 지식 차원을 동시에 고려해야 나옵니다.

**2. 자기 주도적이다**
메타인지는 **남이 가르쳐줄 수 없고 스스로만 얻을 수 있는 지식**입니다. 교사가 "당신은 이 사실을 알고 있나요?"라고 물을 수는 있지만, "당신 스스로 당신이 아는지 모르는지 생각해보세요"라고 할 수밖에 없습니다.

**3. 전이 가능하다**
메타인지 능력은 한 주제에 국한되지 않습니다. 한글 창제에서 훈련한 메타인지는 경제학(수요와 공급), 컴퓨터 과학(구조 명세 언어), 역사 등 **어떤 주제로든 전이**될 수 있습니다.

**4. 무지의 정확한 인식**
소크라테스의 말처럼, **"무엇을 모르는지 아는 것"** 이 진정한 학습의 출발점입니다. 아무것도 모르면서 아는 척하는 것이 가장 큰 장애물입니다.

### 메타인지 학습 목표 예시

| 유형 | 인지 과정 | 학습 목표 예시 |
|------|-----------|---------------|
| D2 | 이해 | "Taxonomy Table 각 셀의 인지적 요구 차이를 설명할 수 있다" |
| D2 | 분석 | "한글 창제 학습 과제를 Taxonomy Table에 매핑하고 각 셀의 관계를 분석할 수 있다" |
| D3 | 평가 | "Taxonomy Table 위에서 **자신의 지식 상태를 정확히 평가**할 수 있다" |
| D3 | 창안 | "자신의 강점과 약점을 분석해 **6개월 학습 계획을 설계**할 수 있다" |

| D6. 창안 | 자신의 지식 프로필에 맞는 학습 계획을 설계할 수 있다 | 6개월 학습 로드맵 설계 |

## 연습: "수요와 공급의 법칙"에 적용하기

### A열: 사실적 지식 — 기본 개념

### B열: 개념적 지식 — 원리와 관계

| 질문 | 인지 과정 |

|------|-----------|

| "주어진 데이터로 수요·공급 곡선을 그리고 균형점을 찾을 수 있는가?" | C3. 적용 |

# 08. 마치며 — 아는 것이 힘이 아니라, 모르는 것을 아는 것이 힘이다

01장에서 소크라테스의 말을 처음 보았을 때를 기억하시나요?

> "I know that I know nothing."

그때는 이 말이 단순한 철학자의 겸손처럼 느껴졌을 수도 있습니다. 하지만 지금, 블룸 분류학의 4가지 지식 차원과 6단계 인지 과정을 모두 살펴본 당신에게 이 말은 다르게 다가올 것입니다.

소크라테스는 이렇게 말한 것이 아닙니다:
- ❌ "나는 아무것도 모른다" — 단순한 1차 지식의 부재
- ✅ "**나는** 아무것도 모른다는 것을 **안다**" — 자신의 지식 상태에 대한 정확한 2차 지식, **메타인지**

## 당신은 이제 24개의 거울을 얻었다

블룸 분류학의 24개 셀은 **당신의 지식을 비추는 24개의 거울**입니다. 이 거울들로 당신이 아는 것과 모르는 것을 구체적으로 볼 수 있습니다.

| | **A. 사실적** | **B. 개념적** | **C. 절차적** | **D. 메타인지** |
|---|---|---|---|---|
| **1. 기억** | | | | |
| **2. 이해** | | | | |
| **3. 적용** | | | | |
| **4. 분석** | | | | |
| **5. 평가** | | | | |
| **6. 창안** | | | | |

당신의 지식은 더 이상 "안다/모른다"라는 이분법이 아닙니다. **"나는 이 주제의 사실적 지식을 이해 수준(B2)으로 알고 있고, 절차적 지식은 적용 수준(C3)으로 할 수 있지만, 메타인지 수준(D)은 아직 개발 중이다"** — 이렇게 정밀하게 표현할 수 있습니다.

## 모르는 셀이 두렵지 않은 이유

이제 당신은 **무엇을 모르는지 알 수 있게 되었습니다.** 이것이 두려움이 아니라 힘인 이유는 간단합니다:

> 모르는 것을 알면, **무엇을 배워야 할지** 알게 됩니다.
> 무엇을 배워야 할지 알면, **어디서부터 시작할지** 알게 됩니다.
> 어디서부터 시작할지 알면, **결국 배울 수 있습니다.**

- **Anderson, L. W. & Krathwohl, D. R. (2001).** *A Taxonomy for Learning, Teaching, and Assessing: A Revision of Bloom's Taxonomy of Educational Objectives.* New York: Longman.
  - 블룸 분류학 개정판의 완전한 이론서. 지식 차원(4가지)과 인지 과정 차원(6단계)의 상세한 정의와 학습 목표 예시를 제공한다.
- **Bloom, B. S. (1956).** *Taxonomy of Educational Objectives: The Classification of Educational Goals. Handbook I: Cognitive Domain.* New York: David McKay.
  - 초판. 2001년 개정판의 기초가 된 원전.

## Taxonomy Table 요약 포스터

> 24셀 완성 매트릭스 — 각 셀에 한글 창제 사례를 채워넣은 일람표

| | **A. 사실적 (Factual)** | **B. 개념적 (Conceptual)** | **C. 절차적 (Procedural)** | **D. 메타인지 (Metacognitive)** |
|---|---|---|---|---|
| **1. 기억 (Remember)** | 한글 창제 연도(1443)를 말할 수 있다 | 제자 원리(상형·가획·합자)의 정의를 말할 수 있다 | 합자 순서(초→중→종)를 말할 수 있다 | 암기 전략의 종류(연상법, 정교화 등)를 말할 수 있다 |
| **2. 이해 (Understand)** | 창제 연도의 역사적 의미를 설명할 수 있다 | 상형 원리가 적용된 방식을 설명할 수 있다 | 모아쓰기 원리를 이해한다 | Taxonomy Table 각 셀의 인지적 요구 차이를 설명할 수 있다 |
| **3. 적용 (Apply)** | 한글 창제 타임라인을 그릴 수 있다 | 제자 원리로 새 글자의 소리를 추론할 수 있다 | 맞춤법 규칙에 따라 글을 쓸 수 있다 | 자신에게 맞는 학습 전략을 선택해 적용할 수 있다 |
| **4. 분석 (Analyze)** | 두 역사 기록의 차이를 분석할 수 있다 | 한글과 한자의 문자적 특성을 비교·분석할 수 있다 | 현대와 당시 표기법의 차이를 분석할 수 있다 | 자신의 강점과 약점을 분석할 수 있다 |
| **5. 평가 (Evaluate)** | 역사 기록의 신뢰성을 평가할 수 있다 | 한글의 과학성을 평가 기준에 따라 판단할 수 있다 | 표기 방식의 적절성을 평가할 수 있다 | 자신의 지식 상태를 정확히 평가할 수 있다 |
| **6. 창안 (Create)** | 한글 역사 연표를 직접 만들 수 있다 | 제자 원리로 새 문자 체계를 설계할 수 있다 | 새로운 표기 규칙을 제안할 수 있다 | 자신의 지식 프로필에 맞는 학습 계획을 설계할 수 있다 |

### 학습 목표 설정 워크시트

**Step 1: 주제 선정**
```
내가 배우고 싶은 주제: ________________________________
```

**Step 2: 현재 위치 찾기**
```
지금 나는 이 주제에 대해:

사실적(A): □ 기억  □ 이해  □ 적용  □ 분석  □ 평가  □ 창안
개념적(B): □ 기억  □ 이해  □ 적용  □ 분석  □ 평가  □ 창안
절차적(C): □ 기억  □ 이해  □ 적용  □ 분석  □ 평가  □ 창안
메타인지(D): □ 기억  □ 이해  □ 적용  □ 분석  □ 평가  □ 창안
```

**Step 3: 목표 설정**
```
나는 "______________________________"을/를
"____________________________________" 할 수 있게 되고 싶다.
                                      (인지 과정 동사)

이 목표는 Taxonomy Table의 (   )열 (   )행에 해당한다.
                   (A/B/C/D)  (1~6)
```

**Step 4: 격차 분석 및 계획**
```
현재 위치와 목표 위치의 차이: ________________________________

극복 계획:
1. ________________________________________________________
2. ________________________________________________________
3. ________________________________________________________
```
