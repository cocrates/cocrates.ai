# 1. 서론 — Bloom의 분류학을 왜 2차원으로 이해해야 하는가

## 문제 제기: "기억보다 창출이 중요하다"는 말의 함정

Bloom의 분류학(Bloom's Taxonomy)은 교육 분야에서 가장 널리 인용되는 프레임워크 중 하나입니다. 많은 사람들이 "단순히 외우는 것(Remember)보다 창의적으로 무언가를 만들어내는 것(Create)이 더 고차원적인 사고다"는 식으로 받아들입니다. 이는 맞는 말이지만 **불완전한 이해**입니다.

왜일까요? 어떤 대상을 창출(Create)하기 위해서는 그 대상에 대한 사실적 지식을 기억하고, 원리를 이해하고, 구성 요소를 분석하고, 대안을 평가하는 능력이 **모두 내재되어 있어야 합니다.** 즉, 인지 과정은 단순한 계단식 위계가 아니라 **상위 수준이 하위 수준을 포함하는 포괄 관계**입니다.

더 중요한 문제는 이것이 **한 축**에 불과하다는 점입니다.

## Anderson & Krathwohl(2001)의 개정: 두 번째 차원의 발견

Bloom의 원래 분류학(1956)은 인지 과정만을 다루었습니다. 2001년 Anderson & Krathwohl이 개정하면서 추가한 가장 큰 변화는 **지식 차원(Knowledge Dimension)** 이라는 두 번째 축입니다. 이로써 Bloom의 분류학은 단순한 '사고의 깊이'만을 측정하는 도구에서, **'무엇에 대해 사고하는가'** 까지 고려하는 2차원 매트릭스로 확장되었습니다.

|  | **인지 과정(Cognitive Process)** → |
|---|---|
| **지식 차원(Knowledge Dimension)** ↓ | 기억 → 이해 → 적용 → 분석 → 평가 → 창출 |
| 사실적(Factual) | |
| 개념적(Conceptual) | *여기 24개의 셀이 각각 독립된 학습 목표가 된다* |
| 절차적(Procedural) | |
| 메타인지적(Metacognitive) | |

지식 차원이 없으면 "같은 '분석(Analyze)'이라도 사실을 분석하는 것과 개념을 분석하는 것은 어떻게 다른가?"라는 질문에 답할 수 없습니다.

## 왜 하필 한글(훈민정음)인가?

이 문서는 Bloom 분류학의 2차원 매트릭스를 설명하기 위해 **한글(훈민정음) 창제의 과학적 원리**를 사례로 사용합니다. 그 이유는 세 가지입니다.

1. **다양한 지식 차원을 자연스럽게 포함한다.** 한글 창제에는 연도·인물 같은 사실(Factual)부터 발음 기관의 상형 원리(Conceptual), 자모 조합 규칙(Procedural), 백성의 인지적 한계를 고려한 전략적 설계(Metacognitive)까지 네 가지 지식 차원이 모두 존재합니다.
2. **인지 과정을 뚜렷이 구분할 수 있다.** 같은 주제(한글)를 가지고 암기에서 창출까지 여섯 인지 과정의 예시를 일관성 있게 구성할 수 있습니다.
3. **실제로 존재하는 체계이다.** 가상의 예시가 아닌 역사적 실체를 다루므로 학습자의 이해와 기억에 더 오래 남습니다.

이제 한글 창제의 사례를 통해 지식 차원과 인지 과정을 각각 살펴본 뒤, 마지막에 두 축이 교차하는 2차원 매트릭스를 종합적으로 제시하겠습니다.

# 2.1 사실적 지식(Factual Knowledge) — 한글 창제의 기본 데이터

## 정의

사실적 지식은 특정 분야의 기본적인 요소 — 용어, 구체적인 사실, 데이터 — 에 대한 지식입니다. 이 지식은 보통 논쟁의 여지가 없으며, 분야에 입문하기 위해 가장 먼저 접하게 되는 정보들입니다. Bloom의 분류학에서 지식 차원의 가장 기초층을 이룹니다.

## 한글 창제의 사실적 지식

한글 창제와 관련된 사실적 지식의 예는 다음과 같습니다.

### 역사적 사실

| 항목 | 내용 |
|------|------|
| 창제 연도 | 1443년(세종 25년) |
| 반포 연도 | 1446년(세종 28년), 《훈민정음》 반포 |
| 창제자 | 조선 제4대 국왕 세종대왕(이도) |
| 참여자 | 집현전 학자들(정인지, 성삼문, 신숙주 등) |
| 기록물 | 《훈민정음 해례본》(국보 제70호) |

### 문자 체계의 용어

- **초성**: 음절의 첫 자음 (예: '한'의 'ㅎ')
- **중성**: 음절의 모음 (예: '한'의 'ㅏ')
- **종성**: 음절의 끝 자음 (예: '한'의 'ㄴ')
- **자음**: 소리를 내는 기관의 모양을 본뜬 글자 (ㄱ, ㄴ, ㄷ, ㄹ...)
- **모음**: 천·지·인의 형상을 본뜬 글자 (ㆍ, ㅡ, ㅣ와 그 결합)
- **훈민정음**: "백성을 가르치는 바른 소리"라는 의미의 문자 이름

이러한 사실적 지식은 **암기**할 수 있는 수준의 정보입니다. "세종이 1443년에 한글을 만들었다"는 말은 참/거짓 판별이 명확한 사실입니다.

## 지식 차원의 시작점

사실적 지식은 그 자체로는 단편적이지만, 상위 지식 차원(개념적·절차적·메타인지적)을 구성하는 **재료** 역할을 합니다. 사실 없이 개념을 세울 수 없고, 개념 없이 절차를 이해할 수 없습니다. 다음 절에서는 이 사실들이 어떤 원리와 관계로 체계화되는지 살펴보겠습니다.

# 2.2 개념적 지식(Conceptual Knowledge) — 한글 창제에 담긴 원리와 철학

## 정의

개념적 지식은 개별 사실들이 어떻게 조직되고 구조화되는지에 대한 지식입니다. 분류와 범주, 원리와 일반화, 이론과 모델이 여기에 속합니다. 사실적 지식이 **벽돌**이라면, 개념적 지식은 그 벽돌을 쌓는 **설계도**에 비유할 수 있습니다.

## 한글 창제의 개념적 지식

한글은 단순한 문자가 아니라, **소리(음성)를 분석하고 이를 시각적 기호로 체계화한 과학적 설계**입니다. 그 핵심 원리들은 다음과 같습니다.

### 1. 상형의 원리 — 발음 기관을 본뜨다

한글 자음의 모양은 사람의 발음 기관 형태를 추상화하여 만들었습니다. 이는 세계의 어떤 문자 체계에서도 찾아보기 어려운 독창적인 접근입니다.

| 자음 | 발음 기관 | 제자 원리 |
|------|----------|----------|
| ㄱ | 혀뿌리(연구개) | 혀뿌리가 목구멍을 막는 모양을 본뜸 |
| ㄴ | 혀끝(치경) | 혀끝이 윗잇몸에 닿는 모양을 본뜸 |
| ㅁ | 입술(양순) | 입 모양을 본뜸 |
| ㅅ | 이(치아) | 이빨 모양을 본뜸 |
| ㅇ | 목구멍(후두) | 목구멍의 모양을 본뜸 |

### 2. 가획의 원리 — 획을 더하여 소리를 더 강하게

기본 자음에 획을 추가하면 소리가 더 거세지거나(유기음) 더 강하게 발음됩니다. 이는 문자 체계가 소리의 음성학적 속성과 **체계적으로 연결**되어 있음을 보여줍니다.

```
ㄱ → ㅋ (획 추가 → 기음이 섞인 소리)
ㄴ → ㄷ → ㅌ (혀끝 소리 → 더 강한 소리)
ㅁ → ㅂ → ㅍ (입술 소리 → 더 강한 소리)
ㅅ → ㅈ → ㅊ (잇소리 → 더 강한 소리)
ㅇ → ㆆ → ㅎ (목구멍 소리 → 더 강한 소리)
```

### 3. 천·지·인 삼재의 모음 철학

모음은 동양 철학의 삼재(三才) 사상을 바탕으로 설계되었습니다.

- **ㆍ (아래아)**: 하늘(天) — 둥글고 열려 있는 모양
- **ㅡ**: 땅(地) — 평평하고 고른 모양
- **ㅣ**: 사람(人) — 서 있는 모양

이 세 기본 모음의 조합으로 모든 한글 모음이 생성됩니다.

### 4. 애민정신과 위민사상 — 창제의 철학적 토대

세종이 한글을 창제한 근본 동기는 **백성을 사랑하고(애민, 愛民), 백성을 중심에 두는(위민, 爲民)** 정치 철학입니다. 《훈민정음》 서문의 유명한 구절이 이를 잘 드러냅니다.

> "백성이 이르고자 하는 바가 있어도 제 뜻을 펴지 못하는 자가 많다. 내가 이를 가엾게 여겨 새로 스물여덟 글자를 만드니, 사람마다 쉽게 익혀 날마다 쓰는 데 편하게 하라."

이는 단순한 사실을 넘어, '왜 한글이 만들어졌는가'라는 인과적 관계와 '문자와 통치 철학'이라는 추상적 연결을 포함하는 **개념적 지식**입니다.

## 사실에서 개념으로의 도약

사실적 지식이 "ㄱ은 혀뿌리를 본떴다"라는 개별 데이터라면, 개념적 지식은 "한글 자음의 자형은 조음 위치에 따라 체계적으로 설계되었고, 이는 보편적인 발음 기관의 형태학에 기반한다"는 **일반화된 원리**입니다. 개념적 지식을 가지면 새로운 자음(예: 외국어의 특수 발음)이 어떻게 설계되었을지 추론할 수도 있습니다.

# 2.3 절차적 지식(Procedural Knowledge) — 한글을 만들고 적용하는 방법

## 정의

절차적 지식은 **무엇을 어떻게 하는가(how-to)** 에 대한 지식입니다. 특정한 방법, 기술, 알고리즘, 기법을 다루며, 개념적 지식이 '원리를 아는 것'이라면 절차적 지식은 '그 원리를 실행에 옮기는 방법을 아는 것'입니다.

개념적 지식이 "이 레스토랑의 셰프는 손님의 취향을 고려한다"는 철학이라면, 절차적 지식은 "재료를 손질하고, 조리하고, 플레이팅하는 단계적 과정"입니다.

## 한글 창제의 절차적 지식

### 1. 글자 생성 절차: 분절 → 조합(모아쓰기)

한글은 소리를 **분절(segment)** 한 뒤 **조합(assemble)** 하는 두 단계의 절차를 따릅니다.

**1단계: 분절**
말소리를 가장 작은 단위인 자음과 모음으로 쪼갭니다.

```
/한/ → /ㅎ/(초성) + /ㅏ/(중성) + /ㄴ/(종성)
```

**2단계: 조합(모아쓰기)**
분절된 자음과 모음을 하나의 음절 블록으로 모아 씁니다.

```
ㅎ + ㅏ + ㄴ → 한
```

이 모아쓰기 방식은 알파벳처럼 자소를 일렬로 늘어놓는 방식과 달리, 음절 단위로 시각적 덩어리를 형성하여 **독서 속도와 의미 인식에 유리**합니다. 이 절차를 알면 어떤 한국어 발음이든 한글로 표기할 수 있습니다.

### 2. 검증 절차: 용비어천가와 불경 언해

세종은 한글이 실제로 소리를 정확히 표현할 수 있는지 검증하기 위해 체계적인 절차를 거쳤습니다.

**① 시범 창작**: 한글로 최초의 장편 문헌인 《용비어천가》를 지어, 문자 체계가 실제 언어 생활에서 작동하는지 시험했습니다.

**② 번역(언해)을 통한 확장**: 불경을 한글로 번역(언해)하여 추상적 개념과 학술적 내용도 표현할 수 있음을 입증했습니다.

**③ 반포와 교육**: 1446년에 정식 반포하고, 백성들이 쉽게 배울 수 있도록 보급 절차를 체계화했습니다.

### 3. 자음 추가 생성 절차: 가획의 규칙 적용

새로운 소리를 표기해야 할 때, 한글은 가획의 원리를 절차적으로 적용하여 새로운 글자를 만들 수 있습니다. 예를 들어, 기본자 ㄱ에 획을 더해 ㅋ을 만들고, 다시 획을 더해 ㄲ을 만드는 방식입니다.

```
기본자 → 획추가(거센소리) → 획중복(된소리)
ㄱ      → ㅋ               → ㄲ
ㄷ      → ㅌ               → ㄸ
ㅂ      → ㅍ               → ㅃ
ㅈ      → ㅊ               → ㅉ
```

이 절차를 알면 기존에 없던 새로운 소리(예: 외래어의 특수 발음)도 체계적으로 표기할 수 있습니다.

## 개념과 절차의 관계

개념적 지식이 "자음은 발음 기관을 본떴고, 획을 더하면 소리가 강해진다"는 **원리**라면, 절차적 지식은 "ㄱ을 기본으로 해서 획을 하나 더 그어 ㅋ을 만들고, 이 원리를 모든 자음 계열에 순차적으로 적용한다"는 **실행 방법**입니다. 개념은 '무엇'과 '왜'를, 절차는 '어떻게'를 다룹니다.

# 2.4 메타인지적 지식(Metacognitive Knowledge) — 자신의 앎에 대한 전략적 인식

## 정의

메타인지적 지식은 **인지에 관한 지식**, 즉 자신의 인지 과정과 그 과정을 통제하는 전략에 대한 지식입니다. Bloom 분류학의 지식 차원 중 가장 높은 수준에 위치하며, 세 가지 하위 유형으로 나뉩니다.

| 하위 유형 | 설명 |
|----------|------|
| **전략적 지식** | 어떤 과제에 어떤 학습/사고 전략이 효과적인지 아는 것 |
| **과제에 관한 지식** | 과제의 성격과 난이도를 판단하는 것 |
| **자기 인식(Self-knowledge)** | 자신의 강점, 약점, 선호하는 방식을 아는 것 |

간단한 비유: **절차적 지식**이 "자동차를 운전하는 방법"이라면, **메타인지적 지식**은 "나는 비 오는 날 야간 운전에 취약하므로, 그럴 때는 속도를 줄이고 차간 거리를 더 넉넉히 확보해야 한다"는 **자신의 운전 능력에 대한 전략적 인식**입니다.

## 한글 창제에서의 메타인지적 지식

흥미롭게도, 세종대왕의 한글 창제 과정에는 메타인지적 지식의 세 가지 하위 유형이 모두 뚜렷이 나타납니다.

### 1. 전략적 지식: "열흘이면 배울 수 있게"

세종은 문자를 설계할 때 **학습 전략의 효율성** 자체를 설계 기준으로 삼았습니다.

> "지혜로운 사람은 아침나절이 되기 전에 깨우치고, 어리석은 사람도 열흘이면 배울 수 있게 하였다."

이것은 단순한 자부심이 아니라 **"사람들은 문자를 어떻게 학습하는가?"에 대한 전략적 분석**의 결과입니다. 즉, 세종은:

- 기존 한자가 실패한 이유가 **인지적 과부하** 때문임을 간파하고
- 음소 단위의 **작은 학습 단위**로 쪼갠 뒤
- 발음 기관이라는 **시각적 연상 장치**를 제공하여 기억을 돕고
- 모아쓰기를 통해 **의미 처리를 병렬화**하는

등의 학습 전략을 문자 체계 자체에 내장했습니다. 이는 '무엇을 아는가'를 넘어 **'인간이 어떻게 효과적으로 아는가'에 대한 지식**입니다.

### 2. 과제에 관한 지식: "한자는 백성을 위한 문자가 아니다"

세종은 당면 과제의 본질을 정확히 진단했습니다.

- **과제의 성격**: 단순히 "문자를 만드는 것"이 아니라 **"평범한 백성이 쉽게 배울 수 있는 문자를 만드는 것"**
- **난이도 판단**: 한자를 개량하는 방식으로는 해결할 수 없고, **완전히 새로운 체계를 창안해야 함**을 인식
- **제약 조건 인식**: 사대부의 반발(최만리의 상소 등)이라는 정치적 제약과, 문자의 사회적 수용에 필요한 조건을 동시에 고려

이는 "무엇을 해야 하는가"뿐 아니라 **"이 과제의 본질과 난이도는 무엇인가"** 를 아는 메타인지적 판단입니다.

### 3. 자기 인식: "내가 모르는 것을 안다"

세종은 자신과 조선의 한계를 정확히 인식하고 있었습니다.

- **자신의 통치 철학 인식**: 유교 군주로서 백성의 삶을 책임져야 한다는 소명 의식
- **기존 체계의 한계 인식**: 한자가 중국어에는 적합하나 한국어의 구조와는 맞지 않음을 간파
- **지식의 공백 인식**: 기존 학문 체계로는 백성의 고통을 해결할 수 없음을 인정하고, 새로운 접근이 필요함을 자각

### 메타인지적 지식의 독특한 위상

메타인지적 지식은 다른 세 지식 차원(사실적·개념적·절차적)과 근본적으로 다릅니다. 그것은 **지식에 관한 지식(knowledge about knowledge)** 입니다. 세종이 단순히 한글 창제 절차(절차적 지식)를 아는 것에 그치지 않고, **백성의 인지적 한계를 배려한 전략을 체계에 내장한 것**은 이 차원의 진정한 의미를 보여줍니다.

> **요약**: 사실적 지식은 '무엇'을, 개념적 지식은 '왜'를, 절차적 지식은 '어떻게'를 안다면, 메타인지적 지식은 **'내가 그것을 어떻게 알고 있으며, 어떻게 더 잘 알 수 있는가'** 를 아는 것입니다.

# 3. 인지 과정(Cognitive Process) — 한글 창제 사례의 여섯 수준

2장에서 지식 차원(무엇을 아는가)을 살펴보았습니다. 이제 인지 과정(어떻게 사고하는가)의 여섯 수준을 한글 창제 사례를 통해 알아보겠습니다. 각 인지 과정은 이전 수준을 포함하는 위계를 이루며, 같은 지식이라도 어떤 인지 과정으로 다루느냐에 따라 사고의 깊이가 달라집니다.

---

# 3.1 기억(Remember) — 관련 지식을 장기기억에서 인출하기

## 정의

기억(Remember)은 Bloom 분류학의 인지 과정 중 가장 기본적인 수준입니다. 장기기억에 저장된 지식을 인출(retrieve)하는 능력을 말합니다. 이 수준에서는 지식의 의미를 깊이 이해하거나 변형할 필요 없이, 있는 그대로 재생하거나 인식하면 됩니다.

## 한글 창제 사례의 '기억' 수준

한글의 과학적 원리에 대해 '기억' 수준에 해당하는 활동은 다음과 같습니다.

### 사실적 지식 × 기억

- "훈민정음은 몇 년도에 반포되었는가?" → **1446년**이라고 암기하여 답한다.
- "한글의 초성, 중성, 종성을 설명하라" → 용어의 정의를 그대로 재생한다.
- "다섯 가지 기본 자음과 그 발음 기관을 대응하라" → ㄱ(혀뿌리), ㄴ(혀끝), ㅁ(입술), ㅅ(이), ㅇ(목구멍)을 암기하여 나열한다.

### 개념적 지식 × 기억

- "상형의 원리를 정의하라" → "발음 기관의 모양을 본떠 글자를 만들었다"는 정의를 암기하여 말한다.
- "가획의 원리가 무엇인가" → "기본 자음에 획을 더하여 소리를 더 강하게 만드는 원리"라고 교과서적 정의를 재생한다.

### 절차적 지식 × 기억

- "한글 모아쓰기의 순서를 말하라" → 초성 → 중성 → 종성 순으로 결합한다는 단계를 암기하여 나열한다.

### 메타인지적 지식 × 기억

- "세종이 한글을 설계할 때 고려한 학습 전략을 나열하라" → 학습 단위 축소, 시각적 연상 장치 제공, 모아쓰기를 통한 병렬 처리 등 전략의 명칭을 암기하여 재생한다.

## '기억'의 역할과 한계

기억은 모든 고차 인지 과정의 **기초**입니다. 분석하거나 창출하기 위해서는 먼저 관련 사실과 개념을 기억할 수 있어야 합니다. 그러나 **기억 자체는 이해, 적용, 분석, 평가, 창출을 보장하지 않습니다.** 단순히 연도와 원리를 외우는 것만으로는 한글의 과학성을 진정으로 이해했다고 말할 수 없습니다. 이는 다음 인지 과정들로 이어질 때 비로소 의미를 갖습니다.

# 3.2 이해(Understand) — 의미를 구성하고 설명하기

## 정의

이해(Understand)는 기억(Remember)의 다음 수준으로, 암기한 지식의 **의미를 구성(construct meaning)** 하는 인지 과정입니다. 단순히 정보를 재생하는 것을 넘어, 자신의 언어로 바꾸고, 요약하고, 비교하고, 설명할 수 있어야 합니다. 대표적인 하위 유형으로 해석(interpreting), 예시 들기(exemplifying), 분류(classifying), 요약(summarizing), 비교(comparing), 설명(explaining) 등이 있습니다.

## 한글 창제 사례의 '이해' 수준

### 사실적 지식 → 이해

- 단순히 "1443년"이라는 연도를 암기하는 대신, "세종이 즉위한 지 25년 차인 1443년에 문자 창제가 완료되었고, 3년 후인 1446년에 정식 반포했으니, 창제와 반포 사이에는 검증과 보완의 기간이 있었다"고 **자신의 언어로 설명**할 수 있다.
- 암기한 사실들 사이의 시간적 관계를 구성하여 맥락화할 수 있다.

### 개념적 지식 → 이해

- "상형의 원리"를 단순 정의 암기가 아닌, **"왜 ㄱ은 혀뿌리 모양이고 ㄴ은 혀끝 모양인지 그 이유를 풀어 설명**할 수 있다.
- 가획의 원리를 **자신의 언어로 요약**할 수 있다: "기본 글자에 획을 추가하면 소리가 더 세지는 체계적인 관계가 있다."
- 천·지·인 삼재 철학이 모음 체계에 어떻게 반영되었는지를 **비유를 들어 설명**할 수 있다.

### 절차적 지식 → 이해

- 모아쓰기 절차를 단순히 단계로 암기하는 대신, **"왜 일렬로 쓰지 않고 하나의 덩어리로 모아 쓰는지 그 이유를 설명**할 수 있다.
- "알파벳은 자소를 일렬로 배열하지만, 한글은 음절 단위로 블록화한다. 이렇게 하면 같은 정보량을 더 적은 시각적 공간에 담을 수 있고, 음절 단위의 패턴 인식이 가능해져 읽기 속도가 빨라진다"고 **원리와 연결지어 설명**할 수 있다.

### 메타인지적 지식 → 이해

- 세종이 선택한 학습 전략의 효과를 **다른 사례에 비추어 설명**할 수 있다. 예를 들어 "세종이 한글을 설계하면서 쓴 '작은 학습 단위로 쪼개기' 전략은 현대의 'Chunking' 이론과 일치한다"고 연결 지을 수 있다.
- "백성이 열흘이면 배운다"는 말의 함의를 **전략적 관점에서 해석**할 수 있다: 단순한 시간 개념이 아니라 **인지적 접근성의 설계 철학**임을 이해한다.

## '이해'의 중요성

이해는 암기(기억)와 적용 사이의 **가교** 역할을 합니다. 같은 지식이라도 이해 없이 적용하려 하면 기계적인 따라하기에 그치고, 이해를 바탕으로 적용하면 새로운 상황에서도 융통성 있게 대처할 수 있습니다. 예를 들어, 한글의 모아쓰기 원리를 이해한 사람은 낯선 외국어의 발음도 한글 표기 규칙에 맞게 옮길 수 있습니다.

# 3.3 적용(Apply) — 배운 절차를 새로운 상황에서 수행하기

## 정의

적용(Apply)은 배운 지식이나 절차를 **새로운 맥락이나 문제에 대입하여 실행**하는 인지 과정입니다. 이해가 '의미를 아는 것'이라면, 적용은 **그 의미를 실제로 사용하는 것**입니다. 주어진 절차(procedure)를 따라 수행하거나, 학습한 원리를 새로운 상황에 대입하는 하위 유형으로 나뉩니다.

## 한글 창제 사례의 '적용' 수준

### 절차적 지식 → 적용

**절차적 지식은 적용과 가장 자연스럽게 연결됩니다.** 절차는 본질적으로 '실행하기 위해' 존재하기 때문입니다.

- **예시 1 — 새로운 외래어 표기**: 한글의 자모 분절 및 모아쓰기 규칙을 활용하여, 조선 시대에는 없었던 현대 외래어를 한글로 표기해본다.
  - 'Computer' → ㅋ + ㅓ + ㅁ + ㅍ + ㅠ + ㅌ + ㅓ → **컴퓨터**
  - 'Sandwich' → ㅅ + ㅏ + ㄴ + ㄷ + ㅟ + ㅊ → **샌드위치**
  - 처음 들어보는 소리(예: 'Thriller'의 'Thr')도 한글의 자모 조합 규칙을 적용하여 표기할 수 있다.

- **예시 2 — 새로운 글자 만들기**: 가획의 원리를 절차적으로 적용하여, 현대 음성학에 등장하는 낯선 소리를 표기하기 위한 새로운 글자를 체계적으로 설계해본다.

### 메타인지적 지식 → 적용

- 세종의 '인지적 접근성 극대화 전략'을 현대의 다른 맥락에 적용해본다.
  - **디지털 리터러시 교육**: "어르신들이 스마트폰 앱을 쉽게 배울 수 있도록, 세종이 한글을 설계할 때 썼던 전략(작은 학습 단위, 시각적 연상, 패턴 인식)을 모바일 UI 가이드라인으로 재해석해본다."
  - **공공 정보 디자인**: "세종이 백성의 이해 수준을 고려해 문자를 단순화한 것처럼, 복잡한 행정 용어를 쉬운 말로 바꾸는 '공공언어 개선 지침'을 만든다."

### 적용이 가능한 이유

적용이 가능한 이유는 이해가 선행되었기 때문입니다. 한글의 모아쓰기 원리를 **이해**했기 때문에, 처음 보는 외래어(예: 'Quantum')에도 그 규칙을 **적용**할 수 있습니다. 이해 없는 적용은 단순한 패턴 매칭에 불과하며, 규칙의 경계 조건에서 실패합니다. 예를 들어 'Know'의 'K' 묵음 처리 같은 예외 상황에서는 이해가 있어야 올바르게 적용할 수 있습니다.

> **핵심 구분**: '이해'는 "모아쓰기가 소리의 음절 단위를 시각적으로 묶는 방식이다"를 설명하는 것이고, '적용'은 "처음 보는 단어 'Strengths'를 한글로 써보라"는 과제를 실제로 수행하는 것입니다.

# 3.4 분석(Analyze) — 요소를 분해하여 관계와 구조 파악하기

## 정의

분석(Analyze)은 정보를 구성 요소로 **분해(decompose)** 하고, 요소들 간의 관계와 전체 구조를 파악하는 인지 과정입니다. 적용이 '주어진 절차를 실행하는 것'이라면, 분석은 **'그것이 왜 그렇게 작동하는지 내부 구조를 들여다보는 것'** 입니다. 하위 유형으로 요소 구분(differentiating), 조직화(organizing), 귀인(attributing)이 있습니다.

## 한글 창제 사례의 '분석' 수준

### 사실적·개념적 지식 → 분석: 문자 체계 비교

- **과제**: 한글과 세계의 다른 문자 체계(한자, 알파벳, 일본어 가나)를 **문자학적 구조 기준으로 분류하고 비교**한다.
- **분석 내용**:
  - 알파벳: **음소-철자 대응**(phoneme-grapheme) 관계 — 소리의 최소 단위를 글자에 대응
  - 한자: **형태-의미 대응**(logographic) 관계 — 글자가 의미를 직접 표현
  - 한글: **음소-자질 대응**(featural) 관계 — 글자의 모양이 **발음 기관의 특징(자질)** 을 반영
- **분석 결과 도출**: 한글은 단순한 음소문자가 아니라, 자질문자(featural writing system)로서 다른 문자 체계와 **구조적으로 차별화**됨을 논증할 수 있다.

### 개념적 지식 → 분석: 상반된 논리의 대조

- **과제**: 세종의 애민정신(愛民精神)과 최만리의 반대 상소에 담긴 논리를 각각 분해하여 대조한다.
- **분석 내용**:
  - 세종의 논리: "문자는 백성을 위한 도구다 → 백성이 어려워하면 더 쉬운 문자를 만들어야 한다 → 한글 창제는 군주의 의무다"
  - 최만리의 논리: "문자는 중국의 성인(聖人)이 만든 것이다 → 새로운 문자를 만드는 것은 사대(事大)의 예에 어긋난다 → 한글 창제는 명분을 해친다"
- **분석 결과**: 같은 정치적 맥락에서 **전제와 결론이 어떻게 다른지**, 각각의 숨겨진 가정을 드러낼 수 있다.

### 절차적 지식 → 분석: 절차의 효율성 평가 준비

- **과제**: 한글의 모아쓰기 절차를 **음절 인식의 인지적 처리 과정** 측면에서 분석한다.
- **분석 내용**: 알파벳의 선형 배열이 좌뇌의 순차 처리를 요구하는 반면, 한글의 블록화는 **시각적 패턴 인식(우뇌)** 을 동시에 활용하게 한다. 이 차이를 실험 연구나 이론에 기대어 분석할 수 있다.

### 분석의 위치

분석은 적용과 평가 사이에서 **전환점** 역할을 합니다. 적용이 '주어진 도구를 사용하는 것'이라면, 분석은 **'그 도구의 내부를 열어보고 설계를 이해하는 것'** 입니다. 분석이 있어야 비로소 평가(판단)와 창출(새로운 설계)이 가능해집니다.

# 3.5 평가(Evaluate) — 기준에 따라 비판적으로 판단하기

## 정의

평가(Evaluate)는 명시적 **기준(criteria)과 표준(standards)** 에 근거하여 판단을 내리는 인지 과정입니다. 단순한 '의견 말하기'가 아니라, **근거를 가지고 옳고 그름, 좋고 나쁨, 적절하고 부적절함을 논증**하는 것입니다. 하위 유형으로 비판(checking)과 논평(critiquing)이 있습니다.

분석이 '요소를 분해하여 보는 것'이라면, 평가는 **'그렇게 본 결과를 토대로 가치 판단을 내리는 것'** 입니다.

## 한글 창제 사례의 '평가' 수준

### 사실적 지식 → 평가: 한글의 국제적 위상 판단

- **과제**: 유네스코가 문맹 퇴치에 기여한 인물이나 단체에 수여하는 상의 이름이 '세종대왕 문해상(King Sejong Literacy Prize)'인 것이 **근거에 비추어 적절한가** 평가한다.
- **평가 기준**:
  1. 한글은 실제로 문맹 퇴치에 기여했는가? (역사적 데이터)
  2. 세종의 철학(애민정신)은 문맹 퇴치의 가치와 부합하는가?
  3. 다른 후보(예: 다른 문자 창제자)와 비교할 때 세종이 가장 적절한가?
- **평가 방법**: 각 기준에 대해 근거를 제시하고, 종합적으로 판단한다. 예를 들어 "한글 창제 이후 조선의 문해율 추정치"와 "문자 학습에 걸리는 시간(한자는 수년, 한글은 수일~수주)"을 근거로 제시할 수 있다.

### 개념적·메타인지적 지식 → 평가: 역사적 가상 시나리오 판단

- **과제**: "만약 세종이 한글을 창제하지 않고 한자를 계속 사용했다면, 조선의 문해율과 신분 제도는 어떻게 변했을까?"라는 질문에 대해 **역사적·사회적 기준을 세워 자신의 의견을 논증**한다.
- **고려할 기준**:
  - 문자 접근성이 신분 이동에 미치는 영향
  - 한자의 학습 곡선이 계층 구조에 기여한 정도
  - 조선 후기 실학 운동의 문자적 기반
- **평가 결과 예시**: "한글이 없었다면 조선의 문해율은 10% 미만에 머물렀을 것이고, 신분제는 더 경직되었을 것이다. 근거로는 한글 창제 후 민간 문서(언간, 소설 등)의 폭발적 증가를 들 수 있다."

### 절차적 지식 → 평가: 한글 표기법의 일관성 판단

- **과제**: 현대 한글 맞춤법(특히 표준어 규정)이 훈민정음의 원리를 얼마나 충실히 따르고 있는지 **일관성 기준으로 평가**한다.
- **평가 기준**: 소리와 글자의 일대일 대응 원칙, 역사적 표기와의 연속성, 예외 규칙의 합리성
- **평가 예시**: "된소리 표기(ㄲ, ㄸ, ㅃ, ㅆ, ㅉ)는 가획의 원리(획 중복 = 소리 강화)와 일관되나, 'ㄹ' 받침의 다양한 발음(ㄹ→ㄷ 현상 등)은 표기와 소리의 괴리를 보여주므로 원칙의 일관성이 부분적으로 훼손되었다."

### 평가와 의견의 차이

평가는 단순한 의견(opinion)과 다릅니다. 의견은 "한글이 정말 대단한 문자인 것 같아요"에서 멈추지만, 평가는 **"한글은 학습 시간(수일) 대비 정보 밀도라는 기준에서 세계 문자 중 최상위권에 속하며, 유네스코 문해상의 이름이 세종대왕인 것은 역사적·교육적 근거에 비추어 적절하다"** 는 식으로 판단의 **근거**를 제시합니다.

# 3.6 창출(Create) — 요소를 재조합하여 새로운 가치 창출하기

## 정의

창출(Create)은 Bloom 분류학 인지 과정의 **최상위 수준**입니다. 요소들을 재조합하여 **새롭고 일관된 전체(novel coherent whole)** 를 만들어내거나, **독창적인 산물(original product)** 을 생성하는 인지 과정입니다. 하위 유형으로 생성(generating), 계획(planning), 생산(producing)이 있습니다.

**중요한 점**: 창출은 이전의 모든 인지 과정(기억→이해→적용→분석→평가)을 **포함**합니다. 창출을 하려면 관련 지식을 기억하고, 원리를 이해하고, 방법을 적용하고, 구조를 분석하고, 대안을 평가할 수 있어야 합니다. 창출은 이 모든 능력이 **종합적으로 발휘**될 때 가능합니다.

## 한글 창제 사례의 '창출' 수준

### 모든 지식 차원의 융합

창출은 단일 지식 차원에 국한되지 않습니다. 사실적·개념적·절차적·메타인지적 지식이 모두 동원되어 새로운 결과물을 만듭니다.

#### 예시 1: 한글의 원리를 응용한 새로운 코딩 언어 설계

- **사용된 지식**:
  - **개념적**: 상형의 원리 — 개념을 시각적 형태와 연결
  - **절차적**: 조합 규칙 — 기본 요소를 결합하여 복합 의미 생성
  - **메타인지적**: 접근성 전략 — 초보자가 쉽게 배울 수 있는 인터페이스 설계
- **창출 결과물**: "한글의 자음·모음 결합 방식을 차용한, 비프로그래머도 직관적으로 사용할 수 있는 도메인 특화 언어(Domain-Specific Language)의 프로토타입을 기획한다."
  - 예: ㄱ(가져오기)+ㅏ(데이터)+ㅇ(출력) → "가다"라는 시각적 명령어

#### 예시 2: 한글 창제를 소재로 한 역사 드라마 대본 창작

- **사용된 지식**:
  - **사실적**: 역사적 인물, 연대기, 사건
  - **개념적**: 애민정신과 사대주의의 갈등이라는 주제
  - **메타인지적**: 세종과 최만리 등 인물의 전략적 사고와 내적 갈등
- **창출 결과물**: 훈민정음 창제 과정에서 벌어진 정치적 갈등을 다룬 4부작 드라마의 **시놉시스와 1화 대본**을 창작한다.

#### 예시 3: 한글 기반의 새로운 타이포그래피 시스템

- **사용된 지식**:
  - **절차적**: 자모 분절과 조합 규칙
  - **개념적**: 상형과 가획의 원리
  - **메타인지적**: 시각적 인지 효율성 전략
- **창출 결과물**: 한글 자모의 기본 형태에서 출발하여, 가획의 원리를 확장한 **새로운 실험적 폰트 패밀리**와 그 디자인 시스템을 설계한다.

### 창출이 포함하는 하위 과정

창출은 단순히 '뭔가 새롭게 만드는 것'이 아니라, 다음과 같은 하위 과정을 포함합니다:

1. **생성(Generating)**: 가능한 아이디어나 가설을 다양하게 제시 — "한글의 원리로 무엇을 만들 수 있을까?"
2. **계획(Planning)**: 생성된 아이디어 중에서 선택하고 실행 방법을 설계
3. **생산(Producing)**: 실제로 결과물을 구체화하여 산출

## 인지 과정의 최종 목적지

Bloom 분류학에서 창출이 최상위에 위치하는 이유는, **이전의 모든 인지 과정을 통합하여 실제 가치를 창출**하기 때문입니다. 한글의 과학적 원리를 아는 것(기억)에서 시작하여, 그것을 이해하고, 적용하고, 분석하고, 평가한 후에야 비로소 **그 원리를 바탕으로 새로운 무엇인가를 만들어낼 수 있습니다.**

> **비유**: 기억은 벽돌을 쌓아둔 것이고, 이해는 설계도를 읽는 것이며, 적용은 설계도대로 짓는 것이고, 분석은 구조를 해체하여 검토하는 것이며, 평가는 건물의 안전성을 판단하는 것입니다. 그리고 창출은 **전혀 새로운 건축 양식을 설계하여 세상에 없던 건물을 짓는 것**입니다.

# 4. 2차원 매트릭스 통합 — 지식 × 인지 과정이 만나는 구체적 사례

## 매트릭스의 의미

지금까지 우리는 **지식 차원(Knowledge Dimension)** 과 **인지 과정(Cognitive Process)** 을 각각의 축으로 살펴보았습니다. 그러나 Bloom 분류학의 진정한 힘은 이 두 축이 교차하는 **24개의 셀(4 × 6)** 에서 나옵니다. 각 셀은 독립된 학습 목표가 될 수 있으며, 같은 지식이라도 어떤 인지 과정으로 다루느냐에 따라 사고의 깊이가 완전히 달라집니다.

아래 표는 한글 창제 사례를 통해 24개 셀을 모두 구체적인 활동으로 채운 것입니다. 각 셀의 예시는 **해당 지식 차원의 내용을 해당 인지 과정으로 다룰 때** 어떤 학습 활동이 되는지 보여줍니다.

## 통합 매트릭스

| 지식 차원 \ 인지 과정 | **기억 (Remember)** | **이해 (Understand)** | **적용 (Apply)** | **분석 (Analyze)** | **평가 (Evaluate)** | **창출 (Create)** |
|---|---|---|---|---|---|---|
| **사실적 (Factual)** | 훈민정음 반포 연도(1446년)를 암기하여 말한다 | 창제(1443년)와 반포(1446년) 사이에 3년의 검증 기간이 있었던 이유를 설명한다 | 반포 연도를 기준으로 세종 즉위 **후 몇 년 차인지 계산**하여 답한다 | 훈민정음 해례본에 기록된 창제 과정의 **단계를 연대기별로 분해**한다 | 유네스코 세종대왕 문해상의 명칭이 **역사적 사실에 비추어 적절한지 판단**한다 | 한글 창제 과정을 소재로 한 **역사 연대표 인포그래픽**을 새로 설계한다 |
| **개념적 (Conceptual)** | 상형의 원리를 **정의대로 암송**한다 | 상형의 원리를 **자신의 언어로 설명**하고 예를 든다 | 상형의 원리를 적용하여 **가상의 새로운 문자를 체계적으로 분류**한다 | 한글의 상형 원리와 알파벳의 음소 원리를 **비교·대조**한다 | "한글이 세계에서 가장 과학적인 문자인가?"라는 주장을 **논리적으로 평가**한다 | 상형 원리와 현대 음성학을 결합한 **새로운 필기체계**를 제안한다 |
| **절차적 (Procedural)** | 모아쓰기 **규칙의 단계를 순서대로 나열**한다 | 모아쓰기 규칙이 **알파벳 일렬 배열보다 읽기에 유리한 이유**를 요약한다 | 처음 보는 외래어('Quantum')를 **한글 표기 규칙에 따라 써본다** | 모아쓰기와 알파벳 일렬 배열의 **인지적 처리 과정 차이**를 분석한다 | 현대 한글 맞춤법의 예외 규칙이 **훈민정음 원리와 일관되는지 판단**한다 | 한글의 조합 방식을 응용한 **새로운 코딩 언어의 문법 체계**를 설계한다 |
| **메타인지적 (Metacognitive)** | 세종이 사용한 학습 전략의 **명칭을 암기**한다 | "열흘이면 배운다"는 말의 **전략적 함의를 해석**한다 | 세종의 전략을 현대 **디지털 리터러시 교육에 적용**한다 | 세종의 전략과 현대 **Chunking 이론의 공통점과 차이점**을 분석한다 | 세종의 문자 설계 전략이 **현대 UX 디자인 원칙과 비교해 얼마나 효과적인지 평가**한다 | 세종의 접근성 전략을 바탕으로 한 **포용적 UI 디자인 가이드라인**을 창안한다 |

## 매트릭스의 가로 읽기와 세로 읽기

### 가로 읽기: 같은 지식, 다른 인지 깊이

같은 지식 차원의 행(row)을 가로로 따라가면, **같은 내용을 다루지만 사고의 깊이가 달라짐**을 볼 수 있습니다.

예를 들어 **절차적 지식** 행을 가로로 읽어보면:

| 기억 | 이해 | 적용 | 분석 | 평가 | 창출 |
|------|------|------|------|------|------|
| 규칙 단계를 줄줄 외움 | 규칙이 효과적인 이유를 설명함 | 규칙을 새 단어에 적용해 봄 | 다른 체계와 처리 방식을 비교함 | 규칙의 일관성을 비판적으로 검토함 | 규칙을 응용한 새 체계를 창안함 |

같은 '모아쓰기 규칙'이라는 주제라도, 기억 수준에서는 단순 암기로 끝나지만 창출 수준에서는 그 원리를 확장하여 전혀 새로운 체계를 만들어냅니다.

### 세로 읽기: 같은 인지 깊이, 다른 지식 유형

같은 인지 과정의 열(column)을 세로로 따라가면, **같은 수준의 사고라도 어떤 종류의 지식을 다루는지에 따라 활동의 성격이 달라짐**을 볼 수 있습니다.

예를 들어 **분석(Analyze)** 열을 세로로 읽어보면:

| 지식 차원 | 분석 활동의 차이 |
|-----------|----------------|
| 사실적 | 사건의 시간적 인과 관계를 분해한다 |
| 개념적 | 두 체계(한글 vs 알파벳)의 구조적 차이를 비교한다 |
| 절차적 | 두 표기 방식의 인지적 처리 과정을 비교한다 |
| 메타인지적 | 두 전략의 이론적 기반을 분석한다 |

이 차이는 교육 설계에서 중요한 시사점을 줍니다. "분석 능력을 키우겠다"고 할 때, **무엇을 분석할 것인가(지식 차원)** 에 따라 구체적인 학습 활동이 완전히 달라져야 합니다.

## 매트릭스가 주는 교육적 통찰

1. **단일 인지 과정 훈련은 충분하지 않다.** '분석'만 가르친다고 분석 능력이 생기는 것이 아니라, 어떤 지식 차원에서 분석할지도 함께 명시해야 합니다.
2. **수업 설계의 정밀도가 높아진다.** "한글을 배운다"는 모호한 목표 대신, "한글의 개념적 지식을 평가할 수 있다"처럼 **구체적인 셀을 목표로 삼을 수 있습니다.**
3. **학습자의 현재 위치와 다음 목표가 명확해진다.** 학습자가 "한글의 개념을 암기"하는 수준(사실적×기억)이라면, 다음 목표는 "한글의 개념을 자신의 말로 설명하는 것"(개념적×이해)이 됩니다.

# 5. 결론 — 요약 및 교육적 시사점

## 요약

이 문서는 한글(훈민정음) 창제의 과학적 원리를 구체적인 사례로 삼아, Bloom의 분류학(Anderson & Krathwohl, 2001)을 2차원 매트릭스로 이해하는 방법을 살펴보았습니다.

### 지식 차원: 네 가지 '무엇'

| 지식 차원 | 한글 사례의 핵심 |
|-----------|----------------|
| **사실적(Factual)** | 1443년 창제, 1446년 반포, 초·중·종성 구성, 참여 인물 |
| **개념적(Conceptual)** | 상형·가획의 원리, 천지인 철학, 애민·위민 정신 |
| **절차적(Procedural)** | 자모 분절 → 모아쓰기 규칙, 검증 및 보급 절차 |
| **메타인지적(Metacognitive)** | 백성의 인지 한계를 고려한 전략적 설계, 상황 판단력 |

### 인지 과정: 여섯 가지 '어떻게'

| 인지 과정 | 한글 사례의 핵심 질문 |
|-----------|---------------------|
| **기억(Remember)** | "한글은 언제 만들어졌는가?" — 사실을 인출한다 |
| **이해(Understand)** | "왜 ㄱ은 혀뿌리 모양일까?" — 의미를 구성한다 |
| **적용(Apply)** | "처음 보는 외래어를 한글로 써보라" — 절차를 실행한다 |
| **분석(Analyze)** | "한글과 알파벳의 구조적 차이는 무엇인가?" — 요소를 분해한다 |
| **평가(Evaluate)** | "세종대왕 문해상은 적절한가?" — 기준에 따라 판단한다 |
| **창출(Create)** | "한글의 원리를 응용한 새 체계를 설계하라" — 새 가치를 창출한다 |

### 통합 매트릭스

이 두 축이 만나는 24개의 셀은 각각 독립된 구체적 학습 목표가 됩니다. 같은 '분석'이라도 사실을 분석하는 것과 개념을 분석하는 것은 전혀 다른 인지 활동이며, 같은 '절차적 지식'이라도 암기하는 것과 창출하는 것은 하늘과 땅 차이입니다.

## 교육적 시사점

### 1. '상위 인지 과정 = 더 중요함'은 오해

Bloom 분류학에 대한 가장 흔한 오해는 "창출이 가장 중요하고 기억은 낮은 수준이니 덜 중요하다"는 생각입니다. 그러나 창출은 기억을 포함하며, 기억 없이 창출은 공중누각입니다. 중요한 것은 **무엇을 어떤 깊이로 다룰지 맥락에 맞게 설계하는 것**입니다.

### 2. 교육 설계의 정밀한 기준 제공

"비판적 사고를 가르친다"는 막연한 목표는 매트릭스 안에서 "한글의 절차적 지식을 분석할 수 있다"처럼 구체화됩니다. 이는 교수자에게는 명확한 수업 설계의 기준을, 학습자에게는 자신의 현재 위치와 다음 목표를 제공합니다.

### 3. 지식 차원의 중요성

많은 교육 과정이 인지 과정(기억→이해→적용→...)에만 초점을 맞추고 지식 차원을 간과합니다. 그러나 같은 '이해'라도 사실적 지식을 이해하는 것과 절차적 지식을 이해하는 것은 다른 교수 전략을 요구합니다. 지식 차원은 단순한 내용의 구분이 아니라, **인지 과정의 성격 자체를 규정하는 중요한 축**입니다.

## 한글 사례가 특별한 이유

이 문서의 사례인 한글 창제는 Bloom 분류학의 모든 차원과 과정을 설명할 수 있는 **자연스러운 마이크로코즘(microcosm)** 입니다. 한글 안에는:

- 암기할 수 있는 사실(사실적)
- 체계화된 원리(개념적)
- 따라 할 수 있는 규칙(절차적)
- '배우는 사람'을 고려한 전략(메타인지적)

이 모두가 존재하며, 각각을 다른 인지 깊이로 다룰 수 있습니다. 이는 Bloom 분류학이 추상적 이론이 아니라, 실제 교육 현장과 지식의 세계를 정밀하게 분석하는 **실용적인 도구**임을 보여줍니다.

> **최종 메시지**: Bloom의 분류학은 '무엇을(지식)' '얼마나 깊이(인지)' 다룰 것인가를 동시에 고려하는 2차원의 프레임입니다. 한글 창제 사례는 이 프레임이 어떻게 구체적인 학습 활동으로 연결되는지 보여주는 살아있는 예시입니다.
