# Outline: Bloom's Taxonomy × Hangul Creation

**Document Title:** 블룸 분류학, 한글 창제로 이해하기

**Narrative Arc:**
독자는 "나는 무엇을 모르는가?"라는 질문에서 출발해, 블룸 분류학의 2차원 체계를 한글 창제 사례와 함께 익히고, 마지막에는 자신의 지식 수준을 진단하고 학습 목표를 세울 수 있게 된다.

```
모름의 인식 → 분류 체계 이해 → 지식 차원별 탐구(한글+명세언어) → 자기 진단 → 학습 목표 설정
```

### 02. 블룸 분류학 — 2차원 프레임워크

| 항목 | 내용 |
|------|------|
| **목적** | 블룸 분류학의 두 차원(지식 차원, 인지 과정 차원)과 그 관계를 개괄한다. 상세한 사례는 03~06장에서 다룬다 |
| **핵심 메시지** | 블룸 분류학은 "무엇을 알고 있는가(지식)"와 "어느 정도로 알고 있는가(인지 과정)"를 함께 보는 2차원 도구다 |

#### 02-01. 블룸 분류학 개요

| 항목 | 내용 |
|------|------|
| **내용 요소** | • 블룸 분류학의 탄생 배경 (매우 간략히)<br>• 왜 알아야 하는가: 학습 목표 설정, 자기 진단, 수업 설계<br>• **2차원 구조:** 지식 차원(4가지) × 인지 과정 차원(6단계) — 두 축이 교차하여 24칸의 매트릭스를 이룸<br>• 학습 목표는 **동사(인지 과정)** + **목적어(지식 유형)** 로 표현 |

#### 02-02. 지식 차원 (Knowledge Dimension)

| 항목 | 내용 |
|------|------|
| **목적** | 4가지 지식 유형과 그 관계를 개괄한다 (구체적 사례는 03~06장에서) |

| 지식 유형 | 질문 | 본질 | 난이도 |
|-----------|------|------|--------|
| **사실적 (Factual)** | "무엇인가?" | 낱낱의 정보 조각 | 비교적 쉬움 |
| **개념적 (Conceptual)** | "왜 그런가?" | 요소 간 관계와 체계 | 보통 |
| **절차적 (Procedural)** | "어떻게 하는가?" | 방법, 기능, 알고리즘 | 보통 |
| **메타인지 (Metacognitive)** | "내가 무엇을 알고 있는가?" | **"앎에 대한 앎"** — 1차 지식을 대상으로 한 2차 지식 | **가장 어려움** |

**관계 — 구체→추상, 상위는 하위를 포함**

메타인지는 단순히 가장 추상적일 뿐 아니라, **다른 세 차원을 '대상'으로 삼는 상위 차원**이다. 

> ⚠️ **메타인지는 네 차원 중 가장 이해하기 어렵다.** 그 이유는 독립된 내용이 없고, 항상 자신의 다른 지식을 '되돌아보는' 방식이기 때문이다. 06장에서 가장 풍부하게 다룰 예정이니, 지금은 "내가 아는 것에 대해 아는 것"이라는 개념만 기억하자.

| **6. 창안 (Create)** | 알려진 한글 창제 사실을 바탕으로 한글 역사 연표 또는 정보그래픽을 직접 만든다 | 새로운 사실 정보를 수집·조직하는 명세 템플릿을 직접 설계한다 |

### 04. 개념적 지식 (Conceptual Knowledge) — 원리를 알자

| 항목 | 내용 |
|------|------|
| **목적** | '개념적 지식'이 무엇인지 이해하고, 한글 창제의 제자 원리와 구조 명세 언어의 개념 체계를 통해 익힌다 |
| **핵심 메시지** | 낱낱의 사실을 넘어, 그것들이 어떻게 **관계 맺고 체계를 이루는지** 아는 것이 한 단계 더 깊은 학습이다 |
| **관계 환기** | 03장의 사실(28자, 1443년 등)이 이 장에서는 **원리(상형·가획·합자)** 로 연결된다 |

| 인지 과정 | 한글 창제 사례 | 구조 명세 언어 사례 |
|-----------|---------------|-------------------|
| **1. 기억 (Remember)** | • 상형·가획·합자·고전 모방의 정의<br>• 훈민정음의 개념: "백성을 가르치는 바른 소리"<br>• 오음(五音) 분류: 아음·설음·순음·치음·후음<br>• 천·지·인의 개념 (`•`, `ㅡ`, `ㅣ`) | • 구조 명세 언어의 핵심 개념 기억: 구조, 관계, 제약, 흐름, 경계<br>• "명세"의 정의: "정확히 무엇을 할 것인가를 기술한 것"<br>• 상위 개념과 하위 개념의 포함 관계 |
| **2. 이해 (Understand)** | • 상형 원리: 발음 기관 모양을 본뜬 것 이해<br>• 가획 원리: 기본자에 획 추가로 된소리·거센소리 이해<br>• 합자 원리: 초·중·종성의 결합으로 음절 형성<br>• "글자는 비록 간요하나 전환이 무궁하다"는 원리 이해<br>• 성리학(태극·음양·오행)이 문자 이론의 기초가 된 맥락 이해 | • 각 개념이 왜 필요한지 이해<br>• 구조·관계·제약·흐름의 개념적 차이 이해<br>• 명세 언어의 설계 철학 이해 (왜 이런 개념들이 필요한가) |
| **3. 적용 (Apply)** | 제자 원리(상형·가획·합자)를 활용해 익숙하지 않은 글자의 발음과 뜻을 추론해본다 | 명세 언어의 개념 체계를 활용해 새로운 도메인의 요구사항을 구조화한다 |
| **4. 분석 (Analyze)** | • 한글과 한자의 문자적 특성을 여러 기준으로 비교·분석한다<br>• 훈민정음 해례본의 구조(예의 vs 해례)를 분석한다<br>• 최만리 반대 상소의 논리적 구조를 분석한다<br>• 상형·가획·합자의 원리가 28자 각각에 어떻게 적용되었는지 분석한다 | • 서로 다른 명세 언어의 개념 체계를 비교·분석한다<br>• 명세의 계층 구조와 요소 간 관계를 분석한다<br>• 특정 명세가 어떤 개념적 모델에 기반하는지 분석한다 |
| **5. 평가 (Evaluate)** | • "과연 한글이 세계에서 가장 과학적인 문자인가?"를 판단 기준에 따라 평가한다<br>• 최만리의 반대 논리가 타당했는지 평가한다<br>• 훈민정음이 창제 당시에 가진 혁신성을 평가한다 | • 특정 목적에 가장 적합한 명세 접근법을 판단 기준에 따라 평가한다<br>• 명세 언어의 표현력과 사용 용이성 간의 트레이드오프를 평가한다 |
| **6. 창안 (Create)** | • **상형·가획·합자의 원리를 조합해 새로운 문자 체계를 설계한다** ★<br>• (확장) 기존 문자 체계의 한계를 극복할 새로운 개념 체계를 구상한다 | • **한글의 제자 원리에서 영감을 받아 AI 시대의 구조 명세 언어 개념 체계를 설계한다** ★★<br>• 상형→직관적 매핑, 가획→점진적 정밀화, 합자→조합적 확장, 간요·전환무궁→단순함과 표현력의 균형을 원리로 삼는다 |

### 06. 메타인지 지식 (Metacognitive Knowledge) — 나를 알자

| 항목 | 내용 |
|------|------|
| **목적** | 가장 추상적이고 이해하기 어려운 지식 유형인 '메타인지 지식'이 **무엇이며, 어떤 하위 유형으로 구성되는지** 명확히 이해한다. 나아가 **메타인지 지식을 학습 목표로 삼는 것**이 왜 최고 수준인지 깨닫는다 |
| **핵심 메시지** | 아는 것(사실), 관계(개념), 방법(절차)을 넘어, **내가 무엇을 알고 무엇을 모르는지 아는 것** — 이것이 가장 높은 지식 차원이다. **"I know that I know nothing"** |
| **관계 환기 — 가장 중요** | 메타인지는 03·04·05장에서 배운 **모든 지식을 '대상'으로 삼는다**: "'내가' 그 사실을 아는가?(03) 그 원리를 아는가?(04) 그 방법을 아는가?(05)" — **메타인지는 독립된 내용이 아니라, 이미 배운 내용에 대한 '2차 지식'이다** |

#### 06-01. 메타인지 지식이란? — "앎에 대한 앎"

| 항목 | 내용 |
|------|------|
| **목적** | 메타인지 지식의 본질을 명확히 이해시킨다. 이 장의 기초 |
| **내용 요소** | • **"메타(meta) = ~에 관한"** → 메타인지 = **"인지에 관한 인지"**, "생각에 대한 생각"<br>• **1차 지식 vs 2차 지식:**<br>  - 1차: "1443년에 세종이 한글을 창제했다" — 사실을 아는 것<br>  - 2차(메타인지): "**나는** 그 사실을 안다" 또는 "**나는** 그 사실을 모른다" — 앎 자체를 아는 것<br>• **비유:** 지도(1차) vs 지도 위에서 자신의 위치를 아는 탐험가(메타인지)<br>• 소크라테스의 "I know that I know nothing" — '아무것도 모른다'는 1차 지식이 아니라, **'아무것도 모르는 상태'를 안다**는 2차 지식이 메타인지다 |

#### 06-02. 메타인지 지식의 세 가지 하위 유형 ★

메타인지 지식은 크게 세 가지 유형으로 구성된다. 각 유형을 **정의 + 한글 창제 사례**로 이해하자.

##### D1. 전략적 지식 (Strategic Knowledge)

| 항목 | 내용 |
|------|------|
| **정의** | 학습, 사고, 문제 해결을 위한 **방법과 전략**에 대한 지식. "어떻게 하면 더 잘 배울 수 있을까?" |
| **한글 창제 사례** | • "한글 28자를 외울 때는 **암기 전략**으로 연상법(ㄱ은 혀뿌리 모양)이 효과적이다"<br>• "상형 원리를 이해할 때는 **정교화 전략**으로 발음 기관 모양을 떠올리면서 글자를 보는 것이 좋다"<br>• "제자 원리를 학습할 때는 **조직화 전략**으로 상형·가획·합자로 분류해서 공부하는 것이 낫다"<br>• "내가 이 개념을 이해했는지 모르겠다면 **초인지 전략**으로 스스로 설명해보고 막히는 부분을 점검한다" |
| **구조 명세 언어 사례** | "명세를 작성할 때는 빈 문서보다 템플릿부터 채우는 전략이 효과적이다" |

##### D2. 인지 과제에 관한 지식 (Knowledge about Cognitive Tasks)

| 항목 | 내용 |
|------|------|
| **정의** | **과제의 성격과 요구 조건**에 대한 지식. 각 과제가 어떤 인지적 노력을 요구하는지 아는 것. |
| **한글 창제 사례** | • **과제 난이도 인식:** "한글 창제 연도를 **기억하는 것(A1)** 보다 상형 원리를 **설명하는 것(B2)** 이 더 어렵다"<br>• **전략 선택:** "사실 암기(A1)에는 반복 학습이 효과적이지만, 원리 이해(B2)에는 예시 만들기가 더 효과적이다"<br>• **맥락 인식:** "한국인에게 한글을 설명할 때는 제자 원리(개념)가 흥미롭고, 외국인에게는 읽기 방법(절차)이 더 필요하다" |
| **구조 명세 언어 사례** | "단순 요구사항 명세보다 여러 관계가 얽힌 명세가 더 높은 인지적 노력을 요구한다는 것을 안다" |

##### D3. 자기 자신에 대한 지식 (Self-Knowledge)

| 항목 | 내용 |
|------|------|
| **정의** | 자신의 **인지적 강점과 약점, 동기, 신념**에 대한 지식. |
| **한글 창제 사례** | • **자기 인식(강점·약점):** "나는 사실 암기는 잘하지만(03장 강점), 원리 설명은 약하다(04장 약점)"<br>• **자기 효능감:** "제자 원리를 이해하는 것은 할 수 있을 것 같다. 하지만 창안 수준은 아직 어렵다"<br>• **동기 인식:** "나는 단순히 한글을 읽고 쓰는 것을 넘어, 한글이 왜 과학적인 문자인지 증명할 수 있을 정도로 깊이 알고 싶다" |
| **구조 명세 언어 사례** | "나는 세부 요구사항을 자주 놓치는 편이다. 명세 작성 시 체크리스트를 활용해야겠다" |

#### 06-03. 메타인지 지식을 학습 목표로 삼는다는 것 ★★

| 항목 | 내용 |
|------|------|
| **목적** | 메타인지 지식이 왜 최고 수준의 학습 목표인지 이해시킨다 |
| **핵심** | **자신의 지식 상태를 정확히 아는 것**은 단순히 많이 아는 것보다 더 높은 차원의 학습이다 |

**왜 메타인지 지식이 최고 수준인가?**

| 이유 | 설명 |
|------|------|
| **통합적이다** | 메타인지는 사실·개념·절차 지식을 모두 **통합적으로 조망**해야 가능하다. 03~05장 각각을 '이해'하는 것과, 그것들을 **하나의 지도 위에 놓고 자신의 위치를 아는 것**은 질적으로 다르다 |
| **자기 주도적이다** | 메타인지 지식은 남이 가르쳐줄 수 없다. **스스로 자신에게 질문**하고 자신의 상태를 진단해야만 얻을 수 있다 |
| **전이 가능하다** | 한글 창제 사례에서 얻은 메타인지는 "수요와 공급의 법칙", "구조 명세 언어", 그 외 어떤 주제로든 **전이(transfer)** 될 수 있다. 방법을 아는 것은 내용을 아는 것보다 오래간다 |
| **무지의 정확한 인식** | 소크라테스의 "I know that I know nothing"은 무지를 인식하는 것 자체가 지혜의 시작임을 말한다. **무엇을 모르는지 정확히 아는 것**이 학습의 출발점이다 |

**메타인지 학습 목표의 예:**

| 코드 | 학습 목표 |
|------|----------|
| **D2 (이해 수준)** | "Taxonomy Table의 각 셀이 요구하는 인지적 노력의 차이를 설명할 수 있다" |
| **D3 (평가 수준)** | "Taxonomy Table 위에서 자신의 현재 지식 상태를 정확히 평가할 수 있다" |
| **D3 (창안 수준)** | "자신의 강점과 약점을 분석해 6개월간의 구체적 학습 계획을 스스로 설계할 수 있다" |

**이 장을 읽는 당신에게:** 지금까지 03(사실) → 04(개념) → 05(절차)를 배웠다. 06장은 '새로운 내용'이 아니라, **03~05장의 내용을 되돌아보며 "나는 이것들을 얼마나 아는가?"를 묻는 것**이다. 그리고 07장에서 본격적으로 자신의 학습 목표를 설계해본다.

**의존성:** 05 이후
**추정 분량:** 약 4-5쪽

### 07. 나의 학습 목표 설계하기

| 항목 | 내용 |
|------|------|
| **목적** | 독자가 배운 2차원 분류 체계를 **자신의 관심 주제**에 적용하며 학습 목표를 설계해본다 |
| **핵심 메시지** | 당신은 지금 어디에 있고, **어디로 가고 싶은가?** |
| **내용 요소** | • **연습 문제: "수요와 공급의 법칙"** <br>  - "이에 대해서 당신은 얼마나 알고 있나요?" — 스스로에게 질문<br>  - 사실(A): 수요·공급의 정의, 수요 법칙, 공급 법칙 등 기본 개념을 말할 수 있는가?<br>  - 개념(B): 수요와 공급이 만나 균형 가격을 형성하는 원리를 설명할 수 있는가?<br>  - 절차(C): 수요·공급 곡선을 그리고, 변화 요인에 따라 곡선을 이동시킬 수 있는가?<br>  - 메타인지(D): 자신이 이 주제에 대해 어디까지 알고 있고, 무엇을 모르는지 아는가?<br>• **Taxonomy Table에서 자신의 위치 찾기** — 지금 어느 셀에 있는가?<br>• **학습 목표 설정 워크시트 (자신의 주제로):**<br>  - "나는 **\_\_** 을/를 **\_\_** 할 수 있게 되고 싶다" (목적어 + 동사)<br>  - 이 목표는 Taxonomy Table의 어느 셀에 해당하는가?<br>  - 지금 내가 있는 셀과 목표 셀의 차이는 무엇인가?<br>  - 그 차이를 극복할 구체적 계획은? |

**의존성:** 06 이후
**추정 분량:** 약 2-3쪽

### 08. 마치며 — 아는 것이 힘이 아니라, 모르는 것을 아는 것이 힘이다

| 항목 | 내용 |
|------|------|
| **목적** | 문서를 철학적으로 마무리하며 독자에게 지속적인 동기를 부여 |
| **핵심 메시지** | 블룸 분류학은 "**나는 무엇을 모르는가?**"를 질문하는 도구다 |
| **내용 요소** | • "I know that I know nothing"으로 돌아가기 — 이제 이 문장의 의미가 다르게 들릴 것이다<br>• 이제 당신은 24개의 셀로 **자신의 지식 지도**를 그릴 수 있다<br>• 모르는 셀이 두렵지 않다 — 이제 **어디를 채워야 할지 알기 때문**이다<br>• 03→04→05→06장의 흐름이 내 삶의 어떤 영역에든 적용될 수 있다<br>• 읽을거리 안내 |

**의존성:** 07 이후
**추정 분량:** 약 1쪽
