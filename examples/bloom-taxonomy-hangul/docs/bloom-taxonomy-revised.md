# Research: Bloom's Taxonomy (Revised) — Anderson & Krathwohl (2001)

## 개요

Bloom(1956)의 교육목표분류학을 Lorin W. Anderson과 David R. Krathwohl 등이 2001년에 개정한 버전.
가장 큰 변화는 **단일 차원(인지 과정)** 에서 **2차원(지식 차원 × 인지 과정 차원)** 구조로 확장된 것.

---

## 2차원 분류 체계 (The Taxonomy Table)

| ↓ 지식 차원 / 인지 과정 차원 → | 1. 기억 | 2. 이해 | 3. 적용 | 4. 분석 | 5. 평가 | 6. 창안 |
|---|---|---|---|---|---|---|
| **A. 사실적 지식** | | | | | | |
| **B. 개념적 지식** | | | | | | |
| **C. 절차적 지식** | | | | | | |
| **D. 메타인지 지식** | | | | | | |

각 칸은 하나의 학습 목표 유형을 나타냄.
예: `B2` = 개념적 지식을 이해하는 것, `C3` = 절차적 지식을 적용하는 것.

---

## 지식 차원 (Knowledge Dimension)

| 차원 | 설명 | 하위 유형 |
|------|------|-----------|
| **A. 사실적 지식 (Factual)** | 기본 요소 — 알고 있어야 하는 개별 정보 | • 용어에 대한 지식<br>• 구체적 사실과 요소에 대한 지식 |
| **B. 개념적 지식 (Conceptual)** | 더 큰 구조 속에서 기본 요소들이 상호 관계를 맺는 방식 | • 분류와 범주에 대한 지식<br>• 원리와 일반화에 대한 지식<br>• 이론, 모형, 구조에 대한 지식 |
| **C. 절차적 지식 (Procedural)** | 무엇을 하는 방법, 탐구 방법, 기능, 알고리즘, 기법 | • 구체적 기능과 알고리즘에 대한 지식<br>• 특정 기법과 방법에 대한 지식<br>• 적절한 절차 사용 시기를 결정하는 기준에 대한 지식 |
| **D. 메타인지 지식 (Metacognitive)** | 인지에 대한 지식, 자신의 인지에 대한 자각. **"앎에 대한 앎"** — 1차 지식(사실·개념·절차)을 대상으로 한 2차 지식 | • 전략적 지식 (D1)<br>• 인지 과제에 관한 지식 (D2)<br>• 자기 자신에 대한 지식 (D3) |

### 메타인지 지식 — 상세 해설

메타인지 지식(Metacognitive Knowledge)은 네 가지 지식 차원 중 가장 추상적이며, 학습자와 독자가 가장 이해하기 어려워하는 차원이다. 그 이유는 **메타인지가 '독립된 내용'을 가지지 않고, 항상 다른 지식을 '대상'으로 삼기 때문**이다.

**핵심 이해: '1차 지식'과 '2차 지식'**

```
1차 지식 (사실·개념·절차):   "1443년에 세종이 한글을 창제했다" (사실)
                                    ↓ (이 지식을 '대상'으로 삼음)
2차 지식 (메타인지):           "나는 그 사실을 알고 있다" — 또는 — "나는 그 사실을 모른다"
```

메타인지는 이처럼 항상 **자신의 다른 지식(사실·개념·절차)에 대한 '이차적' 지식**이다. 이것이 메타인지가 다른 세 차원과 근본적으로 다른 점이다.

#### D1. 전략적 지식 (Strategic Knowledge)

학습, 사고, 문제 해결을 위한 **방법과 전략**에 대한 지식.

| 유형 | 설명 | 예시 |
|------|------|------|
| **암기 전략** | 정보를 기억하기 위한 방법 | "한글 28자를 외울 때는 연상법을 쓰는 것이 효과적이다" |
| **정교화 전략** | 새로운 정보를 기존 지식과 연결 | "상형 원리를 이해하려면 발음 기관의 모양을 떠올리면서 글자를 보는 것이 좋다" |
| **조직화 전략** | 정보를 구조화하여 파악 | "제자 원리를 상형·가획·합자로 분류해서 공부하는 것이 암기보다 낫다" |
| **초인지 전략** | 자신의 이해를 점검하고 조절 | "나는 이 개념을 이해했는가? 다시 읽어야겠다" |

#### D2. 인지 과제에 관한 지식 (Knowledge about Cognitive Tasks)

**과제의 성격과 요구 조건**에 대한 지식. 즉, 각 과제가 어떤 인지적 노력을 요구하는지 아는 것.

| 내용 | 설명 | 예시 |
|------|------|------|
| **과제의 난이도 인식** | 어떤 과제가 더 많은 인지적 노력을 요구하는지 앎 | "한글 창제 연도를 **기억하는 것(A1)** 보다 상형 원리를 **설명하는 것(B2)** 이 더 어렵다" |
| **과제 유형에 따른 전략 선택** | 과제의 성격에 따라 필요한 전략이 다름을 앎 | "사실 암기에는 반복 학습이 효과적이지만, 원리 이해에는 예시 만들기가 더 효과적이다" |
| **맥락 인식** | 어떤 상황에서 어떤 지식이 필요한지 앎 | "외국인에게 한글을 가르칠 때는 절차적 지식(읽기 방법)이 더 중요하고, 한국인에게는 개념적 지식(원리)이 더 흥미롭다" |

#### D3. 자기 자신에 대한 지식 (Self-Knowledge)

자신의 **인지적 강점과 약점, 동기, 신념**에 대한 지식.

| 내용 | 설명 | 예시 |
|------|------|------|
| **자기 인식** | 자신이 잘하는 것과 못하는 것을 앎 | "나는 사실 암기는 잘하지만, 원리를 설명하는 것은 약하다" |
| **자기 효능감** | 자신이 특정 과제를 수행할 수 있다는 믿음 | "제자 원리를 이해하는 것은 할 수 있을 것 같다" |
| **동기 인식** | 자신이 왜 배우고자 하는지 아는 것 | "나는 한글의 과학성을 증명할 수 있을 정도로 깊이 알고 싶다" |

#### 메타인지가 다른 차원과 어떻게 다른가 — 비유

| 비유 | 설명 |
|------|------|
| **지도와 탐험가** | 사실·개념·절차 지식은 **지도(1차 지식)** 이고, 메타인지는 **"내가 지도 위에서 어디에 있는지 아는 탐험가"** (2차 지식)이다. 지도가 아무리 상세해도 탐험가가 자신의 위치를 모르면 길을 찾지 못한다. |
| **도구 상자와 기술자** | 사실·개념·절차는 **도구(1차 지식)** 이고, 메타인지는 **"어떤 도구를 언제 써야 할지 아는 기술자"** (2차 지식)이다. |
| **소크라테스** | "I know that I know nothing" — 아는 것이 없음을 **아는 것** 자체가 메타인지. 스스로의 무지를 아는 것이 학습의 출발점. |

#### 지식 차원 간 관계 — 메타인지의 위치

지식 차원은 **구체적(concrete)** 에서 **추상적(abstract)** 으로 이동함:
사실적 → 개념적 → 절차적 → 메타인지

그러나 메타인지는 단순히 '가장 추상적'일 뿐만 아니라, **다른 세 차원을 질적으로 다른 방식으로 포함**한다:

```
사실적 (무엇)  \
개념적 (왜)    ──→  이들의 대상이 된다  ──→  메타인지 (나는 이것들을 얼마나 아는가?)
절차적 (어떻게) /
```

이것이 메타인지가 다른 차원과 단순히 연속선상에 있는 것이 아니라 **'상위 차원'** 이라고 불리는 이유다.

## 인지 과정 차원 (Cognitive Process Dimension)

| 차원 | 설명 | 핵심 인지 과정 |
|------|------|---------------|
| **1. 기억 (Remember)** | 장기기억에서 관련 지식을 인출 | • 인지(recognizing)<br>• 회상(recalling) |
| **2. 이해 (Understand)** | 교육적 메시지로부터 의미 구성 | • 해석(interpreting)<br>• 예시(exemplifying)<br>• 분류(classifying)<br>• 요약(summarizing)<br>• 추론(inferring)<br>• 비교(comparing)<br>• 설명(explaining) |
| **3. 적용 (Apply)** | 주어진 상황에서 절차 수행 | • 실행(executing)<br>• 구현(implementing) |
| **4. 분석 (Analyze)** | 자료를 구성 요소로 나누고 부분 간, 부분과 전체 간 관계 파악 | • 구별(differentiating)<br>• 조직(organizing)<br>• 귀속(attributing) |
| **5. 평가 (Evaluate)** | 기준과 표준에 근거하여 판단 | • 점검(checking)<br>• 비판(critiquing) |
| **6. 창안 (Create)** | 요소들을 모아 새롭고 일관된 전체를 구성하거나 독창적 산출물 생성 | • 생성(generating)<br>• 계획(planning)<br>• 산출(producing) |

인지 과정 차원은 **낮은 순서의 사고(lower order thinking)** 에서 **높은 순서의 사고(higher order thinking)** 로 이동함.

> **참고:** 각 범주는 위계적이지만 절대적이지는 않음. 예를 들어, '분석'이 '평가'보다 항상 쉬운 것은 아니며, '창안'이 항상 가장 어려운 것도 아님. 그러나 일반적으로 하위 수준의 인지 과정은 상위 수준의 기초가 됨.

## 학습 목표의 구조

Anderson & Krathwohl에 따르면 학습 목표는 일반적으로 **동사(verb)** 와 **목적어(object/noun)** 로 구성됨:

- **동사** → 인지 과정 차원을 나타냄 (예: "설명한다", "비교한다", "창안한다")
- **목적어** → 지식 차원을 나타냄 (예: "사실을", "원칙을", "방법을")

예: "한글 창제의 **역사적 사실을 기억**한다" → A1 (사실적 지식 × 기억)
예: "훈민정음의 **제자 원리를 설명**한다" → B2 (개념적 지식 × 이해)
