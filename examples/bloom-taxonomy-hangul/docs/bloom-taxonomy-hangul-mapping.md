# Bloom's Taxonomy × Hangul Creation: 2차원 매핑

## Taxanomy Table — 한글 창제 사례 매트릭스

| ↓ 지식 차원 \ 인지 과정 차원 → | 1. 기억 (Remember) | 2. 이해 (Understand) | 3. 적용 (Apply) | 4. 분석 (Analyze) | 5. 평가 (Evaluate) | 6. 창안 (Create) |
|---|---|---|---|---|---|---|
| **A. 사실적 지식 (Factual)** | 한글 창제 연도(1443)를 기억한다 | 한글 창제 연도의 역사적 의미를 이해한다 | — | 창제 연도 기록의 신뢰성을 분석한다 | 서로 다른 연도 기록(실록 vs 해례본)을 평가한다 | — |
| **B. 개념적 지식 (Conceptual)** | '훈민정음'의 뜻을 기억한다 | 상형·가획·합자의 제자 원리를 이해한다 | 제자 원리를 다른 문자와 비교하는 데 적용한다 | 훈민정음과 한자의 차이점을 분석한다 | 훈민정음의 독창성을 평가한다 | 한글 제자 원리에서 영감을 받아 **AI 시대의 구조 명세 언어를 설계**한다 |
| **C. 절차적 지식 (Procedural)** | 한글 합자 순서(초+중+종)를 기억한다 | 한글 읽기·쓰기 절차의 원리를 이해한다 | 한글 맞춤법 규칙을 글쓰기에 적용한다 | 한글 표기 방식의 구조적 차이를 분석한다 | 다양한 한글 표기 방식의 적절성을 평가한다 | 자연어 요청을 구조적 명세로 변환하는 **절차와 규칙 체계를 창안**한다 |
| **D. 메타인지 지식 (Metacognitive)** | "백성이 글을 몰라 뜻을 펴지 못한다"는 문제 인식을 기억한다 | 자신의 한글 지식 수준을 이해한다 | 한글 학습 전략을 자신의 상황에 적용한다 | 자신의 문자 사용 습관을 분석한다 | 자신의 한글 이해 수준을 평가한다 | "자연어는 모호하다"는 문제 인식에서 출발해 **사용자- AI 간 정확한 소통을 위한 구조를 창안**한다 |

## 지식 차원(Knowledge Dimension)별 상세 매핑

### A. 사실적 지식 (Factual Knowledge)

| 세부 유형 | 한글 창제 사례 | 설명 |
|----------|---------------|------|
| A1. 용어 지식 | • "훈민정음", "언문", "한글"의 각 용어<br>• "초성", "중성", "종성" 용어<br>• "상형", "가획", "합자", "방괘형" 용어 | 한글을 설명할 때 사용하는 기본 용어들 |
| A2. 구체적 사실 | • 1443년 창제, 1446년 반포<br>• 창제자: 세종대왕<br>• 기본자 28자<br>• 집현전 학사 8명 참여<br>• 《세종실록》 25년 12월 기사<br>• 《훈민정음 해례본》은 국보 제70호 | 한글 창제와 관련된 개별 사실들 |

### B. 개념적 지식 (Conceptual Knowledge)

| 세부 유형 | 한글 창제 사례 | 설명 |
|----------|---------------|------|
| B1. 분류와 범주 | • 자음(초성)과 모음(중성)의 분류 체계<br>• 소리(예: 아음·설음·순음·치음·후음)의 오음 분류<br>• 한글을 표음문자로 분류함 | 소리의 종류에 따른 문자 분류 체계 |
| B2. 원리와 일반화 | • **상형의 원리:** 발음 기관의 모양을 본떠 글자 만듦<br>• **가획의 원리:** 기본자에 획을 더하면 소리가 더 거세짐<br>• **합자의 원리:** 초·중·종성이 모여 하나의 음절을 이룸<br>• **천지인 원리:** `•`(천), `ㅡ`(지), `ㅣ`(인)의 형상<br>• "글자는 비록 간요하나 전환이 무궁하다"는 원리 | 한글 창제의 근본 원리들 |
| B3. 이론·모형·구조 | • **훈민정음 해례본의 이론 체계** — 성리학(태극·음양·오행)에 기반한 문자 이론<br>• **오음(五音) 체계** — 발성 위치에 따른 자음 분류 이론<br>• **초·중·종성 3분 구조** — 음절의 구조적 모형 | 한글 창제를 뒷받침한 이론적 체계 |

### C. 절차적 지식 (Procedural Knowledge)

| 세부 유형 | 한글 창제 사례 | 설명 |
|----------|---------------|------|
| C1. 기능과 알고리즘 | • 한글 읽기: 초성→중성→종성 순으로 소리 내는 방법<br>• 한글 쓰기: 초성+중성+(종성)을 방괘형에 조합하여 쓰는 방법<br>• 연서·병서·부서의 규칙<br>• 종성(받침) 표기 규칙 | 한글을 실제로 읽고 쓰는 절차적 기능 |
| C2. 기법과 방법 | • 한글 창제 과정 자체: 음성 분석 → 글자 설계 → 보완 → 반포<br>• 《동국정운》 편찬을 통한 한자음 표기법 정립<br>• 외국어(중국어, 몽골어 등)의 한글 표기법 | 한글을 연구하고 정립한 방법 |
| C3. 기준 결정 | • 어떤 소리를 어떤 글자로 표기할지 결정하는 기준<br>• 맞춤법 규범을 적용하는 시점과 방법 | 적절한 표기 절차를 선택하는 기준 |

### D. 메타인지 지식 (Metacognitive Knowledge)

| 세부 유형 | 한글 창제 사례 | 설명 |
|----------|---------------|------|
| D1. 전략적 지식 | • **세종의 전략:** "백성이 쉽게 배울 수 있는 문자"라는 학습자 중심 접근<br>• 단계적 도입 전략: 먼저 《용비어천가》로 시범 적용 → 해례본 반포<br>• 집현전 학사들을 활용한 이론 체계 구축 전략 | 문자 창제·보급의 전략적 접근 |
| D2. 과제에 관한 지식 | • 세종이 인식한 과제: "백성의 말은 있어도 글이 없어 뜻을 펴지 못한다"<br>• 자신의 언어학적 지식의 한계를 인식하고 관련 서적을 구해 연구함<br>• 기존 이두(吏讀)의 한계 인식 | 창제 과정에서의 자기 인식 |
| D3. 자기 자신에 대한 지식 | • 학습자로서: "내가 한글을 어느 수준까지 이해하고 있는가?"<br>• 사용자로서: "나는 한글을 읽고 쓰는 데 어떤 어려움이 있는가?"<br>• 교수자로서: "다른 사람에게 한글 원리를 어떻게 설명할 것인가?" | 한글을 대하는 자신의 인지 상태 자각 |

## 인지 과정 차원(Cognitive Process Dimension)별 상세 매핑

### 1. 기억 (Remember)

| 인지 과정 | 한글 창제 사례 예시 |
|----------|-------------------|
| **인지** (Recognizing) | "훈민정음"이라는 단어를 보고 한글의 옛 이름임을 알아본다 |
| **회상** (Recalling) | 한글 창제 연도가 1443년임을 스스로 떠올린다 |

| 지식 유형 | 구체적 예시 |
|-----------|------------|
| A. 사실적 | 한글 자음 14자의 이름과 순서를 암기한다 |
| B. 개념적 | '상형'의 정의를 기억한다 |
| C. 절차적 | 'ㄱ→ㅋ→ㄲ'으로 가획되는 순서를 기억한다 |
| D. 메타인지 | 자신이 한글에 대해 아는 수준을 떠올려본다 |

### 2. 이해 (Understand)

| 인지 과정 | 한글 창제 사례 예시 |
|----------|-------------------|
| **해석** (Interpreting) | "훈민정음"을 "백성을 가르치는 바른 소리"로 바꾸어 말한다 |
| **예시** (Exemplifying) | 상형 원리의 예로 'ㄱ'(혀뿌리), 'ㅁ'(입 모양)을 든다 |
| **분류** (Classifying) | 'ㄱ, ㄴ, ㅁ, ㅅ, ㅇ'을 각각 아음·설음·순음·치음·후음으로 분류한다 |
| **요약** (Summarizing) | 한글 창제의 동기와 원리를 세 문장으로 요약한다 |
| **추론** (Inferring) | 가획 원리로 보아 'ㄷ'에 획을 더하면 'ㅌ'이 될 것이라고 추론한다 |
| **비교** (Comparing) | 한글의 초·중·종성 구조와 영어 알파벳의 자모 배열 방식을 비교한다 |
| **설명** (Explaining) | "왜 한글이 과학적인 문자인가"를 상형 원리로 설명한다 |

| 지식 유형 | 구체적 예시 |
|-----------|------------|
| A. 사실적 | 《세종실록》의 관련 기사 내용을 자신의 말로 다시 설명한다 |
| B. 개념적 | 상형 원리와 가획 원리의 차이를 설명한다 |
| C. 절차적 | 연서(連書)와 병서(並書)의 차이를 이해하고 설명한다 |
| D. 메타인지 | "내가 한글 맞춤법을 어느 정도 이해하고 있는지"를 설명한다 |

### 3. 적용 (Apply)

| 인지 과정 | 한글 창제 사례 예시 |
|----------|-------------------|
| **실행** (Executing) | 한글 맞춤법 규칙에 따라 글을 쓴다 |
| **구현** (Implementing) | 한글 창제 원리를 새로운 언어(예: 외국어)의 표기에 적용한다 |

| 지식 유형 | 구체적 예시 |
|-----------|------------|
| A. 사실적 | 아는 한글 지식을 이용해 낯선 고유어의 발음을 적어본다 |
| B. 개념적 | 상형 원리를 이용해 가상의 문자를 만든다 |
| C. 절차적 | '초성+중성+종성' 합자 절차에 따라 새 단어를 읽어본다 |
| D. 메타인지 | 자신의 한글 오류 패턴을 인식하고 교정 전략을 적용한다 |

### 4. 분석 (Analyze)

| 인지 과정 | 한글 창제 사례 예시 |
|----------|-------------------|
| **구별** (Differentiating) | 훈민정음의 예의(例義, 세종 직필)와 해례(解例, 학사들 집필)를 구별한다 |
| **조직** (Organizing) | 한글 제자 원리를 상형·가획·합자·고전 모방으로 체계화한다 |
| **귀속** (Attributing) | 최만리의 반대 상소가 어떤 가치관(중화주의, 사대주의)에 기반했는지 분석한다 |

| 지식 유형 | 구체적 예시 |
|-----------|------------|
| A. 사실적 | 《세종실록》 1443년 기사와 1446년 기사의 차이점을 분석한다 |
| B. 개념적 | 한글과 한자의 문자적 특성을 여러 기준으로 비교·분석한다 |
| C. 절차적 | 현대 한글 맞춤법과 훈민정음 당시 표기법의 차이를 분석한다 |
| D. 메타인지 | "내가 한글의 어떤 측면을 잘 이해하고 있고, 어떤 측면이 부족한지"를 분석한다 |

### 5. 평가 (Evaluate)

| 인지 과정 | 한글 창제 사례 예시 |
|----------|-------------------|
| **점검** (Checking) | 《세종실록》의 기술("이번 달에 언문 28자를 만들었다")의 정확성을 여러 사료와 대조해 점검한다 |
| **비판** (Critiquing) | 최만리의 반대 논리("중국과의 외교 관계에 문제가 생긴다")가 타당했는지 판단한다 |

| 지식 유형 | 구체적 예시 |
|-----------|------------|
| A. 사실적 | 서로 다른 역사 기록 간의 일치 여부를 평가한다 |
| B. 개념적 | "과연 한글이 세계에서 가장 과학적인 문자인가?"를 판단 기준에 따라 평가한다 |
| C. 절차적 | 여러 한글 표기 방식(두음법칙 적용 전/후) 중 어느 것이 더 적절한지 평가한다 |
| D. 메타인지 | 자신의 한글 학습 방법이 효과적인지 스스로 평가한다 |

### 6. 창안 (Create)

| 인지 과정 | 한글 창제 사례 예시 |
|----------|-------------------|
| **생성** (Generating) | 세종이 기존 문자와 다른 완전히 새로운 문자 체계를 구상한 것 — 가장 대표적인 '창안' 사례 |
| **계획** (Planning) | 집현전 학사 8명을 조직하고 《동국정운》·《용비어천가》 편찬 등 문자 보급을 계획한 것 |
| **산출** (Producing) | 《훈민정음 해례본》이라는 완성된 문자 해설서를 산출한 것 |

| 지식 유형 | 구체적 예시 (한글 창제) | 확장 예시: AI 시대 구조 명세 언어 |
|-----------|------------------------|--------------------------------|
| A. 사실적 | 알려진 한글 지식을 바탕으로 한글 역사 연표를 직접 만든다 | 기존 명세 언어(YAML, JSON, XML 등)의 특징을 조사해 비교표를 만든다 |
| B. 개념적 | 가상의 문자를 설계할 때, 상형이나 가획 같은 새로운 원리 조합을 창안한다 | 한글의 **간요하면서 전환무궁** 원리에서 영감을 받아, **적은 기본 요소로 복잡한 구조를 표현할 수 있는 명세 언어의 개념 체계를 설계**한다 |
| C. 절차적 | 기존 맞춤법과 다른 새로운 표기 규칙을 제안해본다 | 자연어로 표현된 사용자 의도를 **일관된 구조로 변환하는 명세 작성 절차와 규칙 체계를 창안**한다 |
| D. 메타인지 | 자신의 한글 학습 계획을 스스로 설계하고 목표를 설정한다 | **"자연어는 모호해서 사용자와 AI가 정확히 소통하기 어렵다"** 는 문제를 인식하고, 이를 해결할 구조 명세 언어의 **설계 원칙과 학습 목표를 스스로 수립**한다 |

## 학습 목표 예시 (Learning Objectives)

학습 목표는 **동사(인지 과정)** + **목적어(지식 유형)** 구조로 표현된다.

| 코드 | 학습 목표 예시 | 동사 → 인지 과정 | 목적어 → 지식 차원 |
|------|--------------|-----------------|------------------|
| **A1** | 한글 창제 연도(1443)를 말할 수 있다 | 말할 수 있다 → 기억 | 창제 연도 → 사실적 지식 |
| **B2** | 훈민정음의 상형 원리를 설명할 수 있다 | 설명할 수 있다 → 이해 | 상형 원리 → 개념적 지식 |
| **C3** | 새로 접한 외래어를 한글 표기 규칙에 따라 적을 수 있다 | 적을 수 있다 → 적용 | 외래어 표기 규칙 → 절차적 지식 |
| **B4** | 한글과 한자의 문자적 특성을 비교·분석할 수 있다 | 분석할 수 있다 → 분석 | 문자적 특성 → 개념적 지식 |
| **C5** | 서로 다른 한글 표기 방식 중 상황에 맞는 것을 선택할 수 있다 | 선택할 수 있다 → 평가 | 표기 방식 → 절차적 지식 |
| **D6** | 자신의 한글 이해 수준을 진단하고 학습 계획을 수립할 수 있다 | 수립할 수 있다 → 창안 | 학습 계획 → 메타인지 지식 |

  │    ├─ 발성 기관의 형태 분석 (상형의 기초)
  │    └─ 기존 문자(한자, 이두)의 한계 분석
  │
  ├─ [이해 - B2] ← 성리학·음운학 이론 이해
  │    ├─ 태극·음양·오행의 원리 이해
  │    └─ 동아시아 음운학(운서) 지식 이해
  │
  ├─ [적용 - C3] ← 원리를 설계에 적용
  │    ├─ 상형 원리를 글자 모양에 적용
  │    ├─ 가획 원리를 음소 체계에 적용
  │    └─ 합자 원리를 음절 구조에 적용

  ├─ [평가 - C5] ← 완성된 문자의 적절성 평가

## 확장: 한글 창제의 교훈을 AI 시대 구조 명세 언어 창안에 적용하기

### 문제 인식 — 메타인지적 출발점

| 시대 | 문제 상황 | 인식된 한계 |
|------|----------|-----------|
| **1443년 조선** | 백성은 말(한국어)은 있으나, 그 말을 적을 **문자**가 없음 | 한자는 배우기 어려워 백성이 뜻을 펴지 못함 |
| **AI 시대 (현재)** | 사용자는 의도(자연어)는 있으나, 그 의도를 정확히 전달할 **구조 명세 언어**가 부재함 | 자연어는 모호함이 많아 AI와의 정확한 의사소통이 어려움 |

### 한글 창제 원리의 구조 명세 언어 설계로의 적용

| 한글 창제 원리 | 본질 | 구조 명세 언어 설계에의 적용 |
|---------------|------|---------------------------|
| **상형 (象形)** | 발음 기관의 모양을 본떠 글자 창제 — **직관적 매핑** | 구조적 개념(경계, 관계, 제약, 흐름)을 **시각적으로 직관적인 기호**에 매핑 |
| **가획 (加劃)** | 기본자에 획을 더해 더 거센 소리 표현 — **점진적 정밀화** | 기본 구조 요소에서 **세부 사항을 추가**할수록 더 정밀한 명세가 되도록 계층 설계 |
| **합자 (合字)** | 초·중·종성이 모여 하나의 음절 — **조합적 확장** | 기본 요소들이 **조합되어 복잡한 구조를 표현**할 수 있도록 합성 규칙 설계 |
| **간요·전환무궁** | 28자로 무한한 소리 표현 — **단순함과 확장성의 균형** | 적은 수의 기본 구조体로 **다양한 도메인의 복잡한 요구사항**을 표현 가능하게 설계 |
| **학습자 중심** | "사람마다 하여금 쉬워서 편안히 쓰게" — **사용자 중심 설계** | 전문가가 아닌 **일반 사용자도 쉽게 배우고 쓸 수 있는** 명세 언어 설계 |

### 이를 블룸의 Create 차원으로 보면

#### 생성 (Generating) — B6, D6

세종이 "기존 문자(한자)로는 백성의 뜻을 펴기 어렵다"는 문제를 인식하고 **전혀 새로운 접근(표음문자)** 을 구상한 것처럼,
AI 시대의 설계자는 "자연어로는 사용자와 AI 간 정확한 의사소통이 어렵다"는 문제를 인식하고 **기존과 다른 새로운 명세 방식**을 구상한다.

```
1443년: "백성의 말은 있어도 글이 없어 뜻을 펴지 못한다" → 새로운 문자(훈민정음) 구상
AI 시대: "사용자의 의도는 있어도 정확한 명세 언어가 없어 뜻을 전하지 못한다" → 새로운 구조 명세 언어 구상
```

#### 계획 (Planning) — D1, B6

세종이 집현전 학사들을 조직하고, 먼저 《용비어천가》로 시범 적용한 후 해례본을 반포하는 **단계적 계획**을 세운 것처럼,
구조 명세 언어도 **단계적 도입 전략**이 필요하다:

1. **기본 원리 설계** — 핵심 구조 요소(상형에 해당)와 조합 규칙(합자에 해당) 정의
2. **시범 적용** — 간단한 도메인(예: 할 일 목록, 일정 관리)에 먼저 적용
3. **피드백 기반 개선** — 실제 사용 경험을 바탕으로 보완 (세종의 3년간 보완 기간에 대응)
4. **해설서(문서화) 반포** — 사용 방법과 원리를 체계적으로 문서화 (해례본에 대응)

#### 산출 (Producing) — C6

세종이 《훈민정음 해례본》이라는 **완성된 산출물**을 남긴 것처럼,
구조 명세 언어의 설계자는 다음을 산출한다:

- **명세 언어의 문법과 의미론 정의** (해례본의 예의·해례에 대응)
- **명세 작성 가이드라인과 예제 모음** (용비어천가류의 실제 적용 사례)
- **명세를 AI가 해석하고 실행할 수 있는 변환 도구**

### 한글 창제가 구조 명세 언어 창안에 주는 근본적 교훈

1. **문제의 본질을 꿰뚫어라:** 세종은 "글자가 없다"는 표면적 문제 뒤에 **"백성이 뜻을 펴지 못한다"** 는 근본 문제를 보았다. 구조 명세 언어도 "명세 언어가 없다"는 표면을 넘어 **"사용자와 AI가 정확히 소통하지 못한다"** 는 본질을 봐야 한다.

2. **사용자의 수준에 맞추어라:** 세종은 전문가(사대부)가 아닌 **일반 백성**을 대상으로 삼았다. 구조 명세 언어도 개발자/엔지니어가 아닌 **일반 사용자**가 쉽게 쓸 수 있어야 한다.

3. **단순함과 표현력의 균형:** 28자로 무한한 소리를 표현한 것처럼, **적은 규칙으로 다양한 구조를 표현**할 수 있어야 한다. 복잡할수록 강력한 것이 아니라, 단순할수록 널리 쓰인다.

4. **이론을 갖추어라:** 세종은 성리학(태극·음양·오행)이라는 당대 최고의 이론 체계를 문자 창제의 기초로 삼았다. 구조 명세 언어도 단순한 아이디어가 아니라 **견고한 개념적 기초** 위에 세워져야 한다.

5. **반대와 검증을 겸허히 받아들여라:** 최만리의 반대 상소, 3년간의 보완 기간을 거치며 문자가 더 완성되었다. 구조 명세 언어도 **초기 설계를 고집하지 않고 실제 사용을 통한 검증과 개선**을 받아들여야 한다.
