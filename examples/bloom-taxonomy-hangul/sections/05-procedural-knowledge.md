# 05. 절차적 지식 (Procedural Knowledge) — 방법을 알자

> *"원리를 아는 것(개념)에서 더 나아가, 실제로 실행할 수 있는 것 — 그것이 절차적 지식이다."*

**관계 환기:** 03장의 사실(28자), 04장의 원리(상형·가획·합자)가 이 장에서는 **실제로 읽고 쓰는 절차**로 이어집니다. 상형이 무엇인지 설명할 수 있는 것과, 실제로 한글을 읽고 쓸 수 있는 것은 다른 능력입니다. 바로 이 "할 수 있는 것"이 절차적 지식입니다.

## 절차적 지식이란?

**절차적 지식(Procedural Knowledge)** 은 **무언가를 하는 방법**, 즉 기능, 기술, 알고리즘, 방법론에 대한 지식입니다. 세 가지 하위 유형이 있습니다:

- **구체적 기능과 알고리즘에 대한 지식 (Knowledge of Subject-Specific Skills and Algorithms):** "초성+중성+종성을 합자하여 한 음절을 만드는 방법"
- **기법과 방법에 대한 지식 (Knowledge of Subject-Specific Techniques and Methods):** "한글 맞춤법 규칙에 따라 글을 쓰는 방법"
- **적절한 절차 사용 시기에 대한 지식 (Knowledge of Criteria for Determining When to Use Appropriate Procedures):** "어떤 상황에서 연서를 쓰고, 어떤 상황에서 병서를 써야 하는지"

절차적 지식은 "**어떻게?** (How?)"라는 질문에 답합니다. "한글로 '감'을 어떻게 쓰나요?" → "ㄱ + ㅏ + ㅁ을 왼쪽에서 오른쪽, 위에서 아래로 합자합니다."

### 한글의 핵심 절차들

| 절차 | 설명 | 예시 |
|------|------|------|
| **합자 순서** | 초성→중성→종성 순서로 결합 | ㄱ+ㅏ+ㅁ = 감 |
| **모아쓰기** | 자모를 한 음절로 조합 | ㅎ+ㅏ+ㄴ=한, ㄱ+ㅡ+ㄹ=글 |
| **연서법** | 같은 자음을 겹쳐 쓰기 | ㅅ+ㅅ = ㅆ (쌍시옷) |
| **병서법** | 다른 자음을 나란히 쓰기 | ㄱ+ㅅ (한글 초기, 현대에는 거의 사용 안 함) |
| **맞춤법 규칙** | 표준 발음과 표기법 일치 규칙 | "한글" → [한글] 발음과 일치 |
| **외래어 표기법** | 외국어를 한글로 옮기는 절차 | computer → 컴퓨터 |

## 인지 과정별로 살펴보는 절차적 지식

### C1. 기억 (Remember) — "합자 순서를 기억한다"

예시:
> "한글 음절을 만들 때 초성, 중성, 종성을 어떤 순서로 결합하나요?"
> "초성 다음에 중성, 마지막에 종성을 결합합니다."

학습 목표: "초·중·종성의 **합자 순서를 말할 수 있다**" = **C1 셀**

**구조 명세 언어 관점:** 명세 작성의 기본 단계를 기억하는 것. "명세는 요구사항 수집→구조화→검증→승인 순서로 작성한다" 같은 절차적 순서에 대한 암기.

### C2. 이해 (Understand) — "모아쓰기 원리를 이해한다"

단순 순서 암기를 넘어, **왜 그런 순서인지**, 각 단계의 **의미**를 이해하는 수준입니다.

예시:
> "왜 한글은 자모를 풀어쓰지 않고 모아쓰기를 할까요? 모아쓰기가 음절 단위 인식에 어떤 이점을 주는지 설명해보세요."

학습 목표: "초·중·종성이 음절을 형성하는 **과정을 이해한다**" = **C2 셀**

### C3. 적용 (Apply) — "한글 맞춤법 규칙에 따라 글을 쓸 수 있다"

절차적 지식의 핵심입니다. 배운 절차를 **실제로 수행**할 수 있어야 합니다.

예시:
> "다음 문장을 한글 맞춤법 규칙에 따라 올바르게 써보세요."
> "한글은 세종대왕이 창제하신 과학적인 문자입니다."
> (직접 써보기 — 실제로 손이나 키보드로 작성)

학습 목표: "한글 맞춤법 규칙에 따라 **글을 쓸 수 있다**" = **C3 셀**

### C4. 분석 (Analyze) — "현대와 당시 표기법의 차이를 분석할 수 있다"

여러 절차와 방법을 **비교하고, 차이를 식별**하는 수준입니다.

예시:
> "15세기 훈민정음 당시의 표기법과 현대 한글 맞춤법을 비교해보세요. 어떤 차이가 있는가? 받침 표기, 연서·병서 사용, 띄어쓰기 측면에서 구체적인 예를 들어 분석하세요."

핵심 차이점 예시:
- 당시에는 **연서·병서·부서**가 활발히 사용됨
- 현대에는 **된소리(ㄲ, ㄸ, ㅃ, ㅆ, ㅉ)** 만 연서 형태로 남음
- 띄어쓰기는 현대에 정착된 규칙

학습 목표: "현대와 15세기 한글 표기법의 차이를 **분석할 수 있다**" = **C4 셀**

### C5. 평가 (Evaluate) — "표기 방식의 적절성을 평가할 수 있다"

여러 절차 중 **어느 것이 상황에 더 적합한지** 판단하는 수준입니다.

예시:
> "한글을 처음 배우는 외국인 학습자에게는 어떤 표기 방식(풀어쓰기 vs 모아쓰기)이 더 효과적일까? 학습자의 인지적 부담, 읽기 효율성, 발음 학습 용이성 등을 기준으로 평가해보세요."

학습 목표: "상황에 따른 표기 방식의 적절성을 **평가할 수 있다**" = **C5 셀**

### C6. 창안 (Create) — "새로운 표기 규칙을 제안할 수 있다"

기존의 절차를 **새로운 방식으로 조합**하거나, 전혀 새로운 절차를 고안하는 수준입니다.

예시:
> "스마트폰 시대에 더 효율적인 한글 입력 방식을 제안해보세요. 또는 '한국어 발음을 가장 정확히 표현할 수 있는 새로운 로마자 표기법'을 설계해보세요. 기존 표기법(국어의 로마자 표기법, 매큔-라이샤워 표기법)의 한계를 분석하고, 이를 극복할 새로운 절차를 고안하세요."

학습 목표: "기존과 다른 새로운 표기 규칙을 **제안할 수 있다**" = **C6 셀**

**구조 명세 언어 관점 — 응용:** "기존 명세 시스템의 한계를 분석하고, 이를 극복할 새로운 명세 작성 절차 체계를 창안하세요." 예를 들어 복잡한 마이크로서비스 아키텍처를 위한 새로운 명세 템플릿과 작성 방법론을 설계하는 것이 C6 수준에 해당합니다.

### Taxonomy Table — 절차적 지식 (C열)

| 인지 과정 | 학습 목표 예시 | 한글 사례 |
|-----------|---------------|-----------|
| C1. 기억 | 합자 순서를 말할 수 있다 | 초성→중성→종성 |
| C2. 이해 | 음절 형성 과정을 이해한다 | 자모→음절 조합 원리 |
| C3. 적용 | 맞춤법 규칙에 따라 글을 쓸 수 있다 | 실제 문장 쓰기 |
| C4. 분석 | 현대와 당시 표기법 차이를 분석할 수 있다 | 연서·병서 사용 변화 분석 |
| C5. 평가 | 표기 방식의 적절성을 평가할 수 있다 | 모아쓰기 vs 풀어쓰기 평가 |
| C6. 창안 | 새로운 표기 규칙을 제안할 수 있다 | 새 로마자 표기법 설계 |
