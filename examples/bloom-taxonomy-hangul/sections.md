# Architecture Design: Bloom's Taxonomy × Hangul Creation

## Overview

문서 규모: **중간(~20쪽)** — 각 섹션은 단일 파일로 작성하며, 하위 섹션은 `###` 제목으로 구분한다.
의존성 그래프에 따라 **순차적으로 작성**한다. 각 섹션 승인 후 다음 섹션으로 진행한다.

```
01 → 02 → 03 → 04 → 05 → 06 → 07 → 08
                                        ↓
                                   (부록)
```

---

## Dependency Graph

```
01. 들어가며
   └── 02. 블룸 분류학
         ├── 03. 사실적 지식
         │     └── 04. 개념적 지식
         │           └── 05. 절차적 지식
         │                 └── 06. 메타인지 지식
         │                       └── 07. 학습 목표 설계
         │                             └── 08. 마치며
         └── (부록: 독립적)
```

후속 섹션은 선행 섹션에 정의된 개념과 사례를 **참조**할 수 있다. (예: 04장은 03장의 사실을 "28자"로 인용)

## Section 01: 들어가며 — "나는 내가 모르는 것을 안다"

| 항목 | 내용 |
|------|------|
| **파일명** | `01-introduction.md` |
| **목적** | 독자의 호기심을 자극하고, "모르는 것을 아는 것"이 학습의 출발점임을 전달한다. 문서 전체의 분위기와 방향성을 설정한다 |
| **핵심 메시지** | 메타인지(자신의 지식을 아는 것)가 학습의 첫걸음이다. 이 문서를 통해 당신은 자신이 무엇을 알고 무엇을 모르는지 알게 될 것이다 |
| **내용 요소** | • 소크라테스의 "I know that I know nothing" 인용과 짧은 해설<br>• "당신은 한글 창제에 대해 얼마나 알고 있나요?" — 독자에게 직접 질문<br>• 블룸 분류학을 배우면 무엇을 할 수 있게 되는지 Validation Criteria 미리보기<br>  - 2차원 분류 체계를 설명할 수 있다<br>  - 한글 창제 사례를 분류할 수 있다<br>  - 자신의 지식 수준을 진단할 수 있다<br>  - 학습 목표를 설정할 수 있다<br>• 이 문서의 구성 안내 (8개 장 + 부록) |
| **하위 섹션** | 없음 (단일 흐름) |
| **의존성** | 없음 (문서의 시작) |
| **추정 분량** | 약 1~1.5쪽 |
| **톤/스타일** | 철학적이면서도 친근한 어조. 독자에게 말을 거는 방식. "여기 앉아서 커피 한 잔과 함께 읽어보자"는 분위기 |

## Section 02: 블룸 분류학 — 2차원 프레임워크

| 항목 | 내용 |
|------|------|
| **파일명** | `02-bloom-taxonomy-framework.md` |
| **목적** | 블룸 분류학의 두 차원(지식 차원, 인지 과정 차원)과 그 관계를 **간결하게** 개괄한다. 상세한 사례는 03~06장에서 다루므로, 여기서는 프레임워크만 제시한다 |
| **핵심 메시지** | 블룸 분류학은 "무엇을 알고 있는가(지식)"와 "어느 정도로 알고 있는가(인지 과정)"를 함께 보는 2차원 도구다. 상위는 하위를 포함한다 |
| **하위 섹션** | 02-01, 02-02, 02-03 |

### 02-01. 블룸 분류학 개요

| 항목 | 내용 |
|------|------|
| **목적** | 블룸 분류학이 무엇인지, 왜 알아야 하는지, 기본적인 2차원 구조를 소개 |
| **핵심 메시지** | 단순한 "암기→이해→적용" 단계가 아니라, **지식의 유형**이라는 또 다른 축이 있다 |
| **내용 요소** | • 블룸 분류학의 탄생 배경 (매우 간략히 2~3문장)<br>• 왜 알아야 하는가: 학습 목표 설정, 자기 진단, 더 나은 학습 설계<br>• **2차원 구조 소개:** 지식 차원(4가지) × 인지 과정 차원(6단계) = 24칸 매트릭스<br>• 학습 목표는 **동사(인지 과정)** + **목적어(지식 유형)** 로 표현됨을 짧은 예시와 함께 제시 |

### 02-02. 지식 차원 (Knowledge Dimension)

| 항목 | 내용 |
|------|------|
| **목적** | 4가지 지식 유형과 그 관계를 간단히 정의. 구체적 사례는 03~06장에서 |
| **핵심 메시지** | 지식에도 종류가 있다. 사실, 개념, 절차, 메타인지 — 구체에서 추상으로, 상위는 하위를 포함한다 |
| **내용 요소** | • 4가지 지식 유형 요약 표 (유형, 질문, 본질, 난이도)<br>  - 사실적: "무엇인가?" / 낱낱의 정보 / 비교적 쉬움<br>  - 개념적: "왜 그런가?" / 관계와 체계 / 보통<br>  - 절차적: "어떻게 하는가?" / 방법과 기술 / 보통<br>  - 메타인지: "내가 무엇을 알고 있는가?" / "앎에 대한 앎" / **가장 어려움**<br>• 관계: 구체→추상의 연속선, 메타인지는 나머지 세 차원을 대상으로 하는 상위 차원<br>• ⚠️ 메타인지 주의 문구 — "지금은 '내가 아는 것에 대해 아는 것'만 기억하자, 06장에서 자세히 다룸" |

### 02-03. 인지 과정 차원 (Cognitive Process Dimension)

| 항목 | 내용 |
|------|------|
| **목적** | 6단계 인지 과정과 그 관계를 간단히 정의. 구체적 사례는 03~06장에서 |
| **핵심 메시지** | 인지 과정은 하위(기억)에서 상위(창안)로 이어지며, 상위 단계는 하위 단계를 포함한다 |
| **내용 요소** | • 6단계 목록과 1문장 정의<br>  - 기억: 장기기억에서 인출<br>  - 이해: 의미 구성<br>  - 적용: 절차 수행<br>  - 분석: 부분과 전체의 관계 파악<br>  - 평가: 기준에 따른 판단<br>  - 창안: 새로운 전체 구성<br>• 관계: 하위→상위, 상위는 하위를 포함 (예: 평가하려면 분석·이해·기억이 선행되어야 함)<br>• 단, 실제 학습은 반드시 선형적이지 않으며 순환적일 수 있음을 짧게 언급 |

| **의존성** | 01 이후 |
| **추정 분량** | 약 2~2.5쪽 (02-01 0.5쪽, 02-02 1쪽, 02-03 1쪽) |
| **톤/스타일** | 간결하고 명확한 설명. 전문 용어를 처음 소개할 때는 반드시 풀어서 씀. "너무 자세하지 않게, 프레임워크만" |

| **목적** | 가장 추상적이고 이해하기 어려운 '메타인지 지식'이 **무엇이며, 어떤 하위 유형(D1/D2/D3)으로 구성되는지** 명확히 이해한다. 나아가 **메타인지 지식을 학습 목표로 삼는 것**이 왜 최고 수준인지 깨닫는다 |
| **핵심 메시지** | 아는 것(사실), 관계(개념), 방법(절차)을 넘어, **내가 무엇을 알고 무엇을 모르는지 아는 것** — 이것이 가장 높은 지식 차원이다. **"I know that I know nothing"** |
| **관계 환기 (도입부 — 가장 강조)** | 메타인지는 03·04·05장에서 배운 **모든 지식을 '대상'으로 삼는다**: "'내가' 그 사실을 아는가?(03) 그 원리를 아는가?(04) 그 방법을 아는가?(05)" 메타인지는 독립된 내용이 아니라, 이미 배운 내용에 대한 **'2차 지식'** 이다 |

### 06-01. 메타인지 지식이란? — "앎에 대한 앎"

| 항목 | 내용 |
|------|------|
| **목적** | 메타인지 지식의 본질을 명확히 이해시킨다 |
| **핵심 메시지** | 메타인지는 '무엇을 아는 것'이 아니라, **'내가 그것을 안다는 사실'을 아는 것**이다 |
| **내용 요소** | • "메타(meta) = ~에 관한" → 메타인지 = "인지에 관한 인지", "생각에 대한 생각"<br>• **1차 지식 vs 2차 지식 비유:**<br>  - 1차: "1443년에 세종이 한글을 창제했다" (사실)<br>  - 2차(메타인지): "**나는** 그 사실을 안다" 또는 "**나는** 그 사실을 모른다"<br>• **비유:** 지도(1차 지식) vs 지도 위에서 자신의 위치를 아는 탐험가(메타인지)<br>• 소크라테스 "I know that I know nothing" — '아무것도 모른다'는 1차 지식이 아니라, **'아무것도 모르는 상태'를 안다**는 2차 지식이 메타인지다 |

### 06-02. 메타인지 지식의 세 가지 하위 유형 ★

**D1. 전략적 지식 (Strategic Knowledge)**

| 항목 | 내용 |
|------|------|
| **정의** | 학습, 사고, 문제 해결을 위한 **방법과 전략**에 대한 지식. "어떻게 하면 더 잘 배울 수 있을까?" |
| **한글 창제 사례** | • 암기 전략: "한글 28자를 외울 때 연상법(ㄱ=혀뿌리)이 효과적이다"<br>• 정교화 전략: "상형 원리를 이해하려면 발음 기관 모양을 떠올리며 글자를 보자"<br>• 조직화 전략: "제자 원리를 상형·가획·합자로 분류해 공부하는 것이 낫다"<br>• 초인지 전략: "내가 이 개념을 이해했는지 스스로 설명해보고 막히는 부분을 점검한다" |
| **명세 언어 사례** | "명세 작성은 빈 문서보다 템플릿부터 채우는 전략이 효과적이다" |

**D2. 인지 과제에 관한 지식 (Knowledge about Cognitive Tasks)**

| 항목 | 내용 |
|------|------|
| **정의** | **과제의 성격과 요구 조건**에 대한 지식. 각 과제가 어떤 인지적 노력을 요구하는지 아는 것 |
| **한글 창제 사례** | • 난이도 인식: "연도를 **기억하는 것(A1)** 보다 원리를 **설명하는 것(B2)** 이 더 어렵다"<br>• 전략 선택: "사실 암기(A1)에는 반복 학습, 원리 이해(B2)에는 예시 만들기가 효과적"<br>• 맥락 인식: "한국인에게는 제자 원리가, 외국인에게는 읽기 방법이 더 필요하다" |
| **명세 언어 사례** | "단순 명세보다 여러 관계가 얽힌 명세가 더 높은 인지적 노력을 요구한다" |

**D3. 자기 자신에 대한 지식 (Self-Knowledge)**

| 항목 | 내용 |
|------|------|
| **정의** | 자신의 **인지적 강점과 약점, 동기, 신념**에 대한 지식 |
| **한글 창제 사례** | • 자기 인식: "나는 사실 암기는 잘하지만(강점), 원리 설명은 약하다(약점)"<br>• 자기 효능감: "제자 원리 이해는 할 수 있겠지만, 창안 수준은 아직 어렵다"<br>• 동기 인식: "한글이 왜 과학적인 문자인지 증명할 수 있을 정도로 알고 싶다" |
| **명세 언어 사례** | "나는 세부 요구사항을 자주 놓친다. 체크리스트를 활용해야겠다" |

### 06-03. 메타인지 지식을 학습 목표로 삼는다는 것 ★★

| 항목 | 내용 |
|------|------|
| **목적** | 메타인지 지식이 왜 최고 수준의 학습 목표인지 이해시킨다 |
| **핵심** | **자신의 지식 상태를 정확히 아는 것**은 단순히 많이 아는 것보다 더 높은 차원의 학습이다 |

**네 가지 이유:**
1. **통합적이다** — 사실·개념·절차를 모두 통합적으로 조망해야 가능
2. **자기 주도적이다** — 남이 가르쳐줄 수 없고 스스로만 얻을 수 있음
3. **전이 가능하다** — 한글→수요공급→명세언어 등 어떤 주제로든 적용 가능
4. **무지의 정확한 인식** — 무엇을 모르는지 아는 것이 학습의 진정한 출발점

**메타인지 학습 목표 예시:**
| 코드 | 목표 |
|------|------|
| D2 (이해) | Taxonomy Table 각 셀의 인지적 요구 차이를 설명할 수 있다 |
| D3 (평가) | Taxonomy Table 위에서 자신의 지식 상태를 정확히 평가할 수 있다 |
| D3 (창안) | 자신의 강점과 약점을 분석해 6개월 학습 계획을 설계할 수 있다 |

| **의존성** | 05 이후 |
| **추정 분량** | 약 4~5쪽 (06-01 1쪽, 06-02 2.5쪽, 06-03 1.5쪽) |
| **톤/스타일** | 가장 배려 깊은 어조. **"이해하기 어려운 게 당연하다"** 는 전제로 시작. 비유와 사례를 가장 많이 사용. 독자가 "아, 이제 알겠다"고 느낄 때까지 반복 설명 |

## Section 07: 나의 학습 목표 설계하기

| 항목 | 내용 |
|------|------|
| **파일명** | `07-designing-learning-goals.md` |
| **목적** | 독자가 배운 2차원 분류 체계를 **자신의 관심 주제**에 적용하며 학습 목표를 직접 설계해본다 |
| **핵심 메시지** | 당신은 지금 어디에 있고, **어디로 가고 싶은가?** |
| **내용 요소** | • **도입 질문:** "당신은 지금 이 문서를 다 읽었다. 그렇다면 당신은 블룸 분류학의 어디쯤에 있는가?"<br>• **연습 문제: "수요와 공급의 법칙"** <br>  - "이에 대해서 당신은 얼마나 알고 있나요?" — 스스로에게 질문<br>  - 사실(A): 수요·공급의 정의, 수요 법칙, 공급 법칙 등 기본 개념을 말할 수 있는가?<br>  - 개념(B): 수요와 공급이 만나 균형 가격을 형성하는 원리를 설명할 수 있는가?<br>  - 절차(C): 수요·공급 곡선을 그리고, 변화 요인에 따라 곡선을 이동시킬 수 있는가?<br>  - 메타인지(D): 자신이 이 주제에 대해 어디까지 알고 있고, 무엇을 모르는지 아는가?<br>• **Taxonomy Table에서 자신의 위치 찾기** — 지금 당신은 어느 셀에 있는가?<br>• **학습 목표 설정 워크시트 (자신의 주제로):**<br>  - "나는 **\_\_** 을/를 **\_\_** 할 수 있게 되고 싶다"<br>  - 이 목표는 Taxonomy Table의 어느 셀인가?<br>  - 지금 있는 셀과 목표 셀의 차이는 무엇인가?<br>  - 그 차이를 극복할 구체적 계획은?<br>• **예시 답안:** "나는 경제학을 전공하지 않았지만, '수요와 공급의 균형 가격 형성 원리'를 **설명(B2)** 할 수 있게 되고 싶다. 지금 나는 A1(용어 기억) 수준이므로, 먼저 핵심 개념을 이해(B2)하는 것부터 시작하겠다" |
