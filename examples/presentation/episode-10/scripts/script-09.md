# Script: Slide 09 — 2️⃣ 2단계 — ADR: 대안 분석 및 결정

**Time:** ~1분

**[SLIDE 09: Left (개념 + 3가지 규칙) / Right (ADR 예시 mockup — 선택지 A vs B, 장단점, 결정 ✅) / 하단: 포인트 박스]**

(두 번째 단계로 넘어가며)

2단계는 **ADR — Architecture Decision Record**입니다. 우리말로 하면 '구조적 의사결정 기록'이죠.

(화면을 가리키며)

ADR의 핵심은 단순합니다. **최소 2~3가지 명확한 선택지를 제시하고, 각각의 장단점을 비교한 뒤, 사용자가 직접 결정하도록 하는 것.**

(비유로 설명)

건축 비유로 보면 이렇습니다. 건축가가 이렇게 말하는 거죠.

"지붕으로 하면 비용은 저렴하지만 옥상 공간을 전혀 활용할 수 없습니다. 옥상으로 하면 테라스나 정원을 만들 수 있지만 방수 관리가 필요합니다. 옥탑방으로 하면 추가 생활 공간이 생기지만 공사비가 가장 높습니다. 어떤 것을 선택하시겠습니까?"

(잠시 멈춤)

여기서 중요한 건, 건축가가 **결정을 대신하지 않는다**는 점입니다. 선택지를 제시할 뿐, 선택은 항상 **사용자**의 몫입니다.

(세 가지 규칙을 짚으며)

ADR에는 세 가지 핵심 규칙이 있습니다.

**하나,** 사용자가 결정을 위임한 영역에 대해서만 진행합니다. 사용자가 모르는 영역까지 AI가 임의로 결정해서는 안 됩니다.

**둘,** 각 선택지의 장단점을 명확히 비교합니다. 애매모호한 비교는 사용자의 결정을 돕지 못합니다.

**셋,** 사용자의 모든 결정은 Audit Trail로 기록됩니다. 나중에 "왜 이렇게 결정했죠?"라는 질문에 답할 수 있어야 합니다.

(강조하며)

**💡 ADR은 AI의 독단을 막고 사용자가 설계도의 주인으로 남게 하는 장치입니다.** AI는 조력자일 뿐, 결정권자는 항상 사용자입니다.

(화면 전환)

좋습니다. 이제 모든 결정이 끝났습니다. 그런데 개별 결정들만 흩어져 있으면 안 되겠죠? 이 결정들을 하나로 통합할 필요가 있습니다.
