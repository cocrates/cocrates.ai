# Slide 05: 🔍 ASR — 구조적으로 중요한 요구사항

## Key Message
ASR이 무엇인지 이해하고, ASR → ADR → Spec → Generation의 흐름을 여행 계획 비유로 파악하라.

## Content
- **Section badge (상단):** 🔍 2.3 ASR — 구조적으로 중요한 요구사항
- **Definition box:**
  - **Term:** ASR = Architecturally Significant Requirement
  - **Body:** 최종 산출물의 구성, 품질, 전개 방식에 실질적 지각변동을 일으키는 핵심 제약조건
- **🧳 여행 계획 4단계 흐름도 (중앙):**
  - **ASR** (어떤 호텔? 어떤 식당? → 여행 계획 구조 결정) → **ADR** ("시내 호텔 vs 교외 리조트" 대안 검토) → **Spec** (최종 결정: 호텔 A, 식당 B 명세서) → **Generation** (Spec 기반 최종 여행 계획)
- **💡 Point box:** ASR을 명시하는 것이 곧 구조 설계의 시작이다

## Visual Elements
- 4단계 흐름도: 각 상자 다른 색상, 화살표 연결
- 여행 관련 아이콘 (호텔, 식당, 지도)

## Layout Notes
- Definition (상단) + Flow diagram (중앙) + Point (하단)
- 흐름도가 시각적 중심
