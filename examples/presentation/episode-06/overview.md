# S1: Episode-06 발표 정의

## 제목

**구조 기반 워크플로우 생성 실습 — "스킬을 만드는 스킬"**

부제: "보고서 써줘" 대신 "보고서 쓰는 법"을 AI에게 가르치는 법

## 목적

Cocrates의 **스킬 생성 파이프라인(Kernel → Frame → Outline → Spec → Skill → Verification)** 을 실제 `document-authoring` 스킬 개발 사례를 통해 경험한다. 시청자는 단순히 "문서를 작성해 달라"고 요청하는 것이 아니라, **AI에게 문서를 작성하는 방법(워크플로우) 자체를 가르치는 방법**을 이해하게 된다.

특히 다음을 강조한다:
- AI의 **'침묵의 기본값(고정관념)'** 을 사용자가 깨고 유연한 아키텍처로 진화시키는 과정
- Snowflake 점진적 확장(작은 단위로 쪼개어 검토하며 작성)의 실전 적용
- 스킬이 만들어지면 이후 동일한 유형의 요청에 재사용 가능하다는 **지속 가능한 자산** 개념

## 청중

- Ep5까지 시청한 개발자 (구조 기반 생성 파이프라인을 경험함)
- AI에게 "보고서 써줘", "설명문 작성해줘" 같은 요청을 해본 경험이 있는 사람
- "AI가 쓴 문서를 그냥 넘어간" 경험이 있는 사람
- 반복되는 문서 작업을 자동화하고 싶은 사람

## 예상 시간

약 22분 (슬라이드 수 미정, ~1.5분/슬라이드 기준)

## 핵심 메시지

> AI에게 일회성 답변을 요청하는 것을 넘어, **AI의 '일하는 방식' 자체를 설계하고 가르칠 수 있다.** 이것이 스킬(Skill)이다. 사용자의 통찰이 AI의 고정관념을 깨고, 유연한 마스터 아키텍처로 진화시킨다.

스킬은 단순한 코드 조각이 아니라 **공동의 행동 규범**이다. 한 번 만들어두면 같은 유형의 요청에 계속 재사용할 수 있는 **지속 가능한 자산**이다.

## 성공 기준

시청자가 이 발표를 마친 후:
1. **스킬(Skill)과 일회성 요청의 차이**를 이해하고, 스킬이 왜 지속 가능한 자산인지 설명할 수 있다
2. **스킬 생성 파이프라인 6단계**(Kernel → Frame → Outline → Spec → Skill → Verification)를 설명할 수 있다
3. Snowflake 점진적 확장(작은 단위로 쪼개고 검토하며 진행)의 실전 적용 방법을 이해한다

## 에피소드 맥락

### 시리즈 내 위치

```
Part 1. 소개 ── Ep1 (같은 LLM, 다른 결과)
                 Ep2 (원칙 선언)
Part 2. 실습 ── Ep3 (설치)
                 Ep4 (소크라테스식 학습)
                 Ep5 (구조 기반 생성) ← 지난 편
            ▶  Ep6 (워크플로우 생성) ◀ 현재
Part 3. 구조 ── Ep7~12 (원리와 스킬)
Part 4. 결론 ── Ep13~14 (선언)
```

### 이전 에피소드와의 연결

- **Ep5 (생성):** ADR → Spec → Generation → Verification 파이프라인을 배웠다. Ep6는 이 파이프라인을 **스킬 생성**에 적용한 사례다.
- **Ep2 (원칙):** "검토되지 않은 산출물은 생성할 가치가 없다" — 스킬 생성 과정의 모든 단계에서 사용자 승인 게이트가 이 원칙을 지킨다.
- **Ep4 (학습):** Snowflake 점진적 확장 방식이 학습뿐 아니라 스킬 생성에도 동일하게 적용된다.

## 슬라이드 구성 (초안)

### 도입 (~3분)
1. **타이틀 페이지** — Ep5 연결 (생성 파이프라인 → 스킬 생성 파이프라인)
2. **학습 목표** — 4가지 목표 (파이프라인 설명, 스킬의 가치, AI 고정관념 깨기, Snowflake)

### 본론 — 스킬 생성 6단계 경험 (~16분)
3. **시작: "보고서 써줘" 대신 "보고서 쓰는 법"** — 일회성 요청 vs 스킬의 차이
4. **Kernel 단계** — 핵심 목적을 단 한 문장으로 압축 (Cocrates의 질문 3종 세트)
5. **Frame 단계** — 산출물 뼈대와 작업 파일 트리 설계 (Snowflake 시작)
6. **Outline 단계 — 가장 극적인 순간** — AI의 "서론→본론→결론" 고정관념을 사용자가 깨다
7. **Outline 교훈** — AI의 침묵의 기본값을 깨는 사용자의 통찰
8. **Spec 단계** — 6가지 맞춤형 구성 방식 선택지 + 7가지 금지 사항
9. **Skill 생성 단계** — Spec을 기반으로 `.opencode/skills/document-authoring/SKILL.md` 생성
10. **Verification 단계** — 7개 항목 검증 (전원 PASS)
11. **스킬 완성 후 변화** — "보고서 써줘" 한 마디의 변화된 의미
12. **Snowflake 점진적 확장의 힘** — 작은 단위로 쪼개고 검토하는 방법론

### 결론 (~3분)
13. **핵심 요약 + 학습 확인** — 4-point 요약 + 자기 질문
14. **다음 에피소드 연결** — Ep7: Cocrates 프레임워크 구조와 원리 (아키텍처 딥다이브)
