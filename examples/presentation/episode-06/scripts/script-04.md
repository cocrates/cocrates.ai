# Script: Slide 04 — 🎯 Kernel — 핵심 목적을 단 한 문장으로 압축

**Time:** ~1.5분

**[SLIDE 04: Kernel 개념도 — 질문 3종 → 사용자 응답 → Kernel 한 문장 (스킬의 헌법)]**

스킬 생성의 첫 단계는 Kernel입니다. Kernel이 무엇이냐? 스킬의 **헌법**입니다. 이 한 문장이 앞으로 모든 결정의 기준이 됩니다.

Cocrates는 이 Kernel을 만들기 위해 사용자에게 세 가지 질문을 던집니다.

첫째, "어떤 종류의 보고서를 주로 쓰실 건가요?" — 기술 문서? 제안서? 분석 보고서?

둘째, "주요 독자는 누구인가요?" — 개발자? 경영진? 일반 사용자?

셋째, "산출물은 어떤 파일 형태로 저장되나요?" — PDF? Word? Markdown?

여기서 사용자의 응답은 이렇습니다. "일반적인 문서, 누구나 독자, .md 파일"

이 응답을 바탕으로 Kernel이 수립됩니다. "이 스킬은 Markdown 형식의 설명문/보고서를 검토 가능한 단계를 거쳐 생성하도록 돕는다."

이 한 문장이 앞으로의 모든 결정을 좌우합니다. 이 Kernel이 모호하면 스킬 전체가 흔들립니다. 반대로, 이 Kernel이 명확하면 이후 단계가 훨씬 수월해집니다. 모호한 요청을 단 한 문장으로 압축하는 것, 이것이 스킬 설계의 첫걸음입니다.
