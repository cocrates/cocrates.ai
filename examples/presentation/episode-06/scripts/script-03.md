# Script: Slide 03 — "보고서 써줘" 대신 "보고서 쓰는 법"을 AI에게 가르친다

**Time:** ~1.5분

**[SLIDE 03: 좌우 비교 — 좌측 "보고서 써줘" (일회성 요청) vs 우측 🦉 "보고서 자동화 스킬 만들어줘" (스킬)]**

자, 화면을 봐주세요. 왼쪽과 오른쪽, 두 가지 접근법을 비교해보겠습니다.

왼쪽은 일반적인 접근법입니다. "보고서 써줘" — 이렇게 매번 요청할 때마다 AI는 매번 다른 품질의 결과물을 줍니다. 오늘은 괜찮은데 내일은 형편없고, 같은 주제인데도 구조가 제각각이죠. 반복 피로가 쌓입니다. 이것이 **일회성 요청**의 한계입니다.

오른쪽은 Cocrates의 접근법입니다. "보고서를 작성하는 스킬을 만들어줘" — 한 번 만들어두면 영원히 재사용할 수 있습니다. 같은 유형의 요청이 들어오면 AI가 항상 같은 워크플로우로 작업을 수행하고, 일관된 품질을 보장합니다.

마치 "물고기를 잡아달라"고 매번 부탁하는 것과, "물고기 잡는 법을 배우라"고 가는르치는 것의 차이와 같습니다.

핵심은 이것입니다. **스킬이란 AI의 행동 규범, 즉 표준 워크플로우입니다.** 같은 유형의 요청에 대해 AI가 일관된 방식으로 작업하도록 만드는 규칙이 바로 스킬입니다.

오늘 우리가 만들 것은 `document-authoring`이라는 스킬입니다. 보고서나 설명문 작성을 자동화하는 워크플로우입니다. 이제 본격적으로 어떻게 만드는지 알아보겠습니다.
