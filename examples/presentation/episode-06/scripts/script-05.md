# Script: Slide 05 — 🧱 Frame — 산출물의 뼈대와 작업 파일 트리

**Time:** ~1.5분

**[SLIDE 05: 문서 레이아웃 개념도 + Snowflake 파일 트리 구조]**

Kernel이 완성되었으면, 이제 Frame 단계입니다. Frame은 산출물의 뼈대를 만드는 단계입니다.

먼저 문서의 레이아웃을 정의합니다. 메타정보, 개요, 본문 — 여기서 본문은 다시 섹션과 서브섹션으로 나뉘고 — 그다음 결론, 부록 순서입니다.

그런데 여기서 중요한 것은 어떻게 작업할 것인가입니다. Cocrates는 한 번에 통으로 결과물을 내놓지 않습니다. 대신 Snowflake 방식으로 점진적으로 확장해 나갑니다.

화면에 보이는 파일 트리를 봐주세요.

`docs/{slug}/` 디렉토리 아래에, 첫 단계로 `outline.md` — 문서 개요와 메타정보를 저장합니다.
두 번째 단계로 `sections.md` — 구조 도면을 저장합니다.
세 번째 단계로 `sections/` 디렉토리 — 섹션별 본문을 개별 파일로 저장합니다.
마지막으로 `{slug}.md` — 최종 통합본입니다.

Cocrates가 이렇게 제안하는 이유가 뭘까요? "한 번에 통으로 쓰는 것보다 섹션 단위로 쪼개어 검토하며 작성하는 게 안전하겠죠?"

이것이 Snowflake의 핵심입니다. **작은 단위로 쪼개고, 단계별로 파일에 저장하고, 검토하고, 승인받고, 다음 단계로.** 채팅창에서 모든 걸 주고받지 않습니다. 파일이 작업의 단위입니다.
