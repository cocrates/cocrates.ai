# Script: Slide 12 — ❄️ Snowflake 점진적 확장의 힘

**Time:** ~1.5분

**[SLIDE 12: Snowflake 원칙 4가지 + Ep4-5-6 방법론 비교표 — 공통 패턴 시각화]**

지금까지 우리는 스킬 생성의 전 과정을 살펴봤습니다. 여기서 한 가지 중요한 패턴이 보입니다. 바로 **Snowflake 점진적 확장**입니다.

Snowflake 원칙을 정리하면 네 가지입니다.

첫째, 작게 시작한다. Kernel 단 한 문장에서 시작했습니다.
둘째, 단계별로 확장한다. Frame, Outline, Spec으로 점진적으로 확장해 나갔습니다.
셋째, 각 단계에서 검토와 승인을 받는다. 사용자가 매 단계를 승인해야만 다음으로 넘어갔습니다.
넷째, 중간 산출물을 파일로 저장한다. 채팅창에서 모든 걸 주고받지 않고, 각 단계의 결과물을 파일로 보존했습니다.

그런데 흥미로운 점은, 이 Snowflake 패턴이 Ep6에만 있는 것이 아니라는 겁니다.

Ep4의 학습 과정을 보세요. Education → Knowledge Capture → Reflection. 작은 개념에서 시작해서 점진적으로 깊이를 더해갔습니다.

Ep5의 생성 과정도 마찬가지입니다. ADR → Spec → Generation → Verification. 작은 결정에서 시작해서 전체 산출물로 확장해 나갔습니다.

그리고 오늘 Ep6의 스킬 생성. Kernel → Frame → Outline → Spec → Skill → Verification. 모두 같은 패턴입니다.

**Snowflake는 Cocrates의 공통 패턴입니다. 배움, 생성, 스킬 — 모든 활동에 동일한 방법론이 적용됩니다.**
