# Script — Slide 3 (1분 30초)

## Slide: 두 가지 접근법 — 구조를 아는 것과 모르는 것

**Time:** ~1분 30초

"Episode 10에서 배운 핵심을 기억하시나요?

산출물 생성에는 두 가지 접근법이 있습니다. **구조를 모르는 경우**와 **구조를 아는 경우**.

여행 계획을 예로 들어보겠습니다.

여행 계획을 세울 때, 구조를 모르면 어떻게 하나요?

'어떤 호텔?', '어떤 식당?' — 개별 항목을 하나하나 검토합니다. 그때마다 ADR로 결정하고 Spec에 반영합니다. 이것이 **spec-driven-generation**의 접근법입니다. 구조가 없으니 구조를 먼저 만들어야 하니까요.

그런데, 여행 계획에는 사실 **명확한 구조**가 있습니다.

여행 계획은 **timeline**입니다.
숙박 → 조식 → 활동 → 중식 → 활동 → ... 이라는 일별 timeline이 있고,
여행지1 → 여행지2 → 여행지3 → ... 이라는 경로 구조가 있습니다.

이 구조를 알면? 먼저 여행지 경로를 결정하고, 각 여행지에서 구체적인 timeline을 계획합니다. 그리고 각 timeline 단계에서 호텔, 식당, 활동을 검토합니다.

**더 구체적인 설계 결정을 내릴 수 있습니다.** 구조가 있으니까요.

이 '여행 계획의 timeline 구조' 자체를 **재사용 가능한 템플릿, 즉 스킬**로 만드는 것이 **generating-skill-creation**의 역할입니다.

(잠시 멈춤)

두 접근법의 차이는 단순합니다. '구조를 아는가, 모르는가'. generating-skill-creation은 **알게 된 구조를 영구적인 스킬로 전환**합니다."
