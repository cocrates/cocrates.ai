# Script — Slide 7 (1분 30초)

## Slide: Meta Snowflake 7단계 개관

**Time:** ~1분 30초

"지금까지 우리는 generating-skill-creation이 **어떤 원리**로 움직이는지 배웠습니다. 5대 프리즘으로 산출물을 분해하고, Snowflake 5단계로 점진 구체화한다는 것.

그렇다면 generating-skill-creation **자신**은 새 스킬을 어떻게 만들까요?

바로 **Meta Snowflake 7단계**를 따릅니다.

함께 보시죠.

(슬라이드 포인트하며)

**Kernel → Components → Frame → Outline → Spec → Skill → Verification**

이 7단계가 generating-skill-creation이 새 SKILL.md를 만드는 절차입니다.

각 단계는 앞서 배운 Snowflake 5단계에 매핑됩니다.

- **Kernel**은 define. 생성 대상을 **한 문장**으로 정의합니다. 이게 흔들리면 모든 게 흔들립니다.
- **Components**는 plan. 앞서 배운 5대 차원을 식별합니다.
- **Frame**은 architecture design. 워크플로우, 파일 구조, 승인 지점을 설계합니다.
- **🚨 Outline은 detail design** — 생성 전 최종 게이트입니다.
- **Spec → Skill → Verification**은 generation. 명세를 확정하고 파일을 생성하고 검증합니다.

소설 창작 비유로 다시 말하면: Outline은 '장면 리스트'입니다. 장면 리스트 없이 집필하지 않는 것처럼, **Outline 없이 SKILL.md를 생성하지 않습니다**."

(잠시 멈춤)

"이 7단계는 이 발표자료의 슬라이드 구조를 설계할 때와 **동일한 논리**로 작동합니다."
