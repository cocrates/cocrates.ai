# Slide Content Specification — Slide 5

## Slide 5: ❄️ Snowflake 5단계 — 소설 창작의 점진 구체화

**Section:** 본론 1 — 메타-스킬의 정체성
**Type:** 개념/프로세스
**Time:** ~1분 30초

### Visual Design
- **Background:** 시리즈 공통 템플릿
- **Layout:** 수평 프로세스 다이어그램
  - 상단: 제목 "Snowflake 5단계" (32pt)
  - 중단: 5개 단계를 왼쪽→오른쪽으로 연결하는 프로세스 박스
  - 각 박스: 단계명 + 소설 예시 텍스트
  - detail design과 generation 사이: 🚨 게이트 아이콘 강조
  - 하단: 게이트 설명 + 💡 포인트

### Content Elements
| 요소 | 내용 | 비고 |
|------|------|------|
| 제목 | ❄️ Snowflake 5단계 — 점진 구체화 워크플로우 | 상단 |
| define | 장르(판타지 스릴러), 목표 독자(성인), 분량(400쪽), 핵심 테마 결정 | 1번 박스 |
| plan | 전체 스토리 아크, 3막 구조 개관, 주요 등장인물 개요 | 2번 박스 |
| architecture design | 챕터별 개요, 각 막의 전환점, 인물 관계도, 서브플롯 구조 | 3번 박스 |
| **🚨 detail design** | 각 챕터의 장면 리스트, 대화/설명/액션 비율, 세부 플롯 포인트 | 4번 박스 (빨간 강조) |
| generation | 실제 집필 | 5번 박스 |
| 🚨 게이트 | "장면 리스트(detail design) 완료 전까지 집필(generation) 금지" | 4-5 사이 게이트 |
| 위반 결과 | 게이트 없음 → '세계관 덤프' 소설 탄생 | 작은 경고 텍스트 |
| 💡 포인트 | "이 Snowflake은 presentation-authoring도, 소설 창작도 동일한 구조로 설계될 수 있다" | 하단 박스 |

### Layout Notes
- 5단계는 등간격 수평 배열. 각 박스는 화살표로 연결.
- detail design 박스는 색상/테두리로 강조 (빨간 계열).
- detail design → generation 사이 🚨 게이트는 크게 표시.
- 게이트 위반 결과는 회색/작은 글씨로 부가 설명.
