# Slide 10: 🎯 Pull-only 평가 — 가르치지 않고 측정한다

## Key Message
Reflection은 절대 Push(가르치기)로 전환하지 않는다. Pull-only로 측정하며, Gap 발견 시 기록만 하고 Education 환류를 제안한다. 측정과 교수는 분리되어야 한다.

## Content
### Headlines / text
- **왜 Pull-only인가?** Reflection의 목적은 **성장**이 아니라 **측정**. 가르치면 현재 수준이 왜곡됨
- **질문 유형 (SKILL.md 실제 규칙):**
  - "이 원칙을 완전히 다른 도메인의 예시를 들어 설명해보세요" (전이)
  - "이 시나리오에서 원칙이 깨지는 경계 조건은 어디인가요?" (경계)
  - "KB에 기록된 과거의 그 틀렸던 가정, 지금도 똑같이 실수하고 있지 않나요?" (메타인지)
- **Gap 발견 시 행동 원칙:**
  - 그 자리에서 **강의하지 않음**
  - `⚠️ 제대로 알지 못했던 것` 리스트로 기록만
  - 세션 마무리 시 "별도의 Education을 진행하시겠습니까?" 제안
- **💡 포인트:** "모르는 걸 가르쳐주는 것"이 친절처럼 보이지만, Reflection의 순수성을 해친다. **측정과 교수는 분리되어야 한다.**

### Visual elements
- 상단: 🎯 타이틀과 핵심 원칙
- 중앙: 3가지 질문 유형 말풍선 카드
- 하단 좌측: Gap 처리 프로세스 (기록→제안)
- 하단 우측: 💡 강조 박스

## Layout Notes
- 질문 유형은 실제 질문을 인용문 스타일로 (청중이 직접 질문받는 느낌)
- Gap 처리 프로세스는 간단한 2단계 다이어그램
- "가르치지 않는다"는 빨간색 강조
