# Slide 06: 💾 Knowledge Capture Skill — 핵심만 캡처한다

## Key Message
Knowledge Capture의 핵심은 "기억하는 것"을 기록하는 것이 아니라 "내가 몰랐던 것"을 기록하는 것. 장황한 설명은 금지, 회상 가능한 최소 단위만 저장한다.

## Content
### Headlines / text
- **핵심 원칙:** 백과사전식 장황한 설명이나 전체 코드는 **일절 저장하지 않음**
- **저장 위치:** `kb/{topic-slug}.md` (kebab-case, 영어)
- **저장 대상:**
  - **한 줄 정의** — 추후 회상을 위한 최소 단위
  - **비유** — 개념을 내 것으로 만드는 이미지
  - ✅ **잘 이해한 것**
  - ⚠️ **"내가 처음에 착각했던 오개념과 무지(Ignorance)"** ← `## Wrong Assumptions / Gaps` 섹션 (가장 중요)
- **💡 포인트:** "아는 것"이 아니라 **"몰랐던 것"을 기록하는 것**이 Knowledge Capture의 본질

### Visual elements
- 상단: 💾 타이틀과 핵심 원칙 한 줄
- 중앙: 저장 대상 4개를 카드 배열
  - ⚠️ 항목은 강조 색상 (크게, 눈에 띄게)
- 하단: 저장 위치 코드 스타일로 표시
