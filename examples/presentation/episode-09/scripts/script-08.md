# Script: Slide 08 — 🔄 Merge 전략 — 지식 중복 방지

**Time:** ~1분

**[SLIDE 08: Merge 전략 3단계 프로세스 — 검색 → 발견 시 Merge(덮어쓰지 않음) → 미발견 시 신규 생성]**

Knowledge Capture의 마지막 퍼즐, Merge 전략입니다.

만약 매번 새로운 파일을 만든다면, KB는 수천 개로 불어나고 회상이 불가능해집니다. 이 문제를 해결하기 위해 Knowledge Capture 스킬은 Merge 전략을 사용합니다.

**1단계:** 저장 요청이 들어오면 동일 주제의 기존 KB 파일이 있는지 검색합니다. 파일명, 제목, Tags를 비교합니다.

**2단계:** 발견하면 덮어쓰지 않습니다. 기존 내용은 그대로 두고, 새로운 인사이트만 `## Update History`와 함께 추가합니다.

**3단계:** 발견되지 않으면 새 파일을 생성합니다.

(슬라이드 예시를 가리키며)

```markdown
## Update History
- 2026-06-26: Push/Pull 전략의 판단 규칙 추가
- 2026-06-20: 최초 작성 (3블록 구조)
```

지식은 교체가 아니라 누적되어야 합니다. Merge 전략은 지식의 연속성을 보장합니다. 한 번 배운 내용이 사라지지 않고, 새로운 통찰이 계속해서 쌓여가는 거죠.
