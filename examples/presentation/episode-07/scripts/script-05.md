# Script: Slide 05 — 📜 Cocrates Agent Prompt — 6개 섹션 개관

**Time:** ~1.5분

**[SLIDE 05: 6개 섹션을 가로로 연결한 순서도 — Persona → Principle → Harness Architecture → Request Handling → Core Activities → Success Criteria]**

Cocrates Agent, 즉 `cocrates.md`는 6개 섹션으로 구성되어 있습니다. 이 6개 섹션이 순차적으로 Cocrates라는 에이전트의 행동 전반을 정의합니다.

첫 번째 섹션은 **Persona**, 정체성입니다. "불확실성을 체계적인 탐구로 전환하고, 사용자가 결과물을 완전히 이해할 때까지 안내하는 존재" — 이것이 Cocrates의 정체성입니다. 단순한 정보 제공자가 아니라, 사용자의 성장을 돕는 페이스메이커인 거죠.

두 번째는 **Principle**, 핵심 원칙입니다. 바로 **Harness Ignorance** — 무지를 통제하라. 검토 없는 생성은 금지입니다. 이 원칙이 모든 결정의 기준이 됩니다.

세 번째는 **Harness Architecture**, 구조입니다. 방금 설명한 Agent + Skills 분리 구조를 이 섹션에서 선언합니다.

네 번째는 **Request Handling**, 요청 처리입니다. 여기에 **Intent-To-Skill Routing**이라는 핵심 매커니즘이 들어있습니다. 잠시 후 자세히 다루겠습니다.

다섯 번째는 **Core Activities**, 핵심 활동입니다. Cocrates가 수행하는 두 가지 주요 파이프라인 — 산출물 생성과 학습 — 이 이 섹션에 정의되어 있습니다.

여섯 번째는 **Success Criteria**, 성공 기준입니다. 가장 중요한 질문: "사용자가 이 산출물을 자기 입으로 설명할 수 있는가?" 이 질문에 Yes라고 답할 수 있을 때만 Cocrates는 세션을 성공적이라고 판단합니다.

지금부터 이 중 가장 핵심적인 **Request Handling**을 자세히 들여다보겠습니다.
