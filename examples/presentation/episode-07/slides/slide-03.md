# Slide 03: 🍳 "만능 칼의 함정" — 하나의 프롬프트로는 안 된다

## Key Message
모든 산출물을 하나의 프롬프트로 처리하려는 시도는 만능 칼의 함정과 같다. 산출물 유형별로 다른 구조적 접근이 필요하며, 이것이 Agent + Skills 구조의 근본 이유다.

## Content
- **비유:** 전문 주방 — 생선 회 뜨기, 스테이크 굽기, 디저트 플레이팅은 각각 다른 칼과 조리 순서가 필요. 하나의 만능 칼로는 모든 요리의 품질을 낼 수 없음
- **AI의 현실 — 산출물 유형별 구조적 접근의 차이:**
  - 보고서/문서: 논리 계층 구조 (목차, 섹션, 단락)
  - 발표자료: 페이지 단위 레이아웃 + 거버닝 메시지
  - 학습 활동: 질문-피드백 흐름 (턴제 미션)
  - 블로그 시리즈: 에피소드와 스토리라인
- **결론:** 하나의 프롬프트로 처리 → 지나치게 일반적이거나 특정 유형에 치우침 → Agent + Skills 분리가 필요

## Visual Elements
- 주방 이미지 — 생선회 칼, 스테이크 나이프, 디저트 도구 각각 별도
- 가운데 "만능 칼" 이미지에 ❌ 표시
- 산출물 유형별 아이콘 (📄 문서 / 📊 발표자료 / 🎓 학습 / 📝 블로그)

## Layout Notes
- 비유(상단) → AI 현실(중단) → 결론(하단) 3단 구조
- 만능 칼에 강한 X 표시로 시각적 임팩트
- 각 산출물 유형은 아이콘 + 키워드로 간결하게
