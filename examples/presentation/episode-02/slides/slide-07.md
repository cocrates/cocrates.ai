# Slide 07: 검토의 진짜 본질 — U → A → E → A

## Key Message
검토는 단순한 오타 확인이 아니다. 이해 → 분석 → 평가 → 승인의 4단계 과정이다.

## Content
- **상단 (취소선):** "대부분의 사람들이 생각하는 검토: 대충 읽어보고 오타 확인"
- **중단 — 4단계 흐름도:**
  - **U — 이해 (Understand):** "내 말로 다시 설명할 수 있을 때 비로소 이해한 것"
  - **A — 분석 (Analyze):** "왜 이렇게 구현되었는가?" — 설계 의도, 구조, 로직 파악
  - **E — 평가 (Evaluate):** "이게 정말 최선인가?" — 프로젝트 목적/기준에 부합하는지 판단
  - **A — 승인 (Approve):** "내가 확인했다. 이걸로 간다. 책임은 나다."
- **하단:** "검토 = U → A → E → A"
- **Visual Elements:**
  - 4단계 수평 흐름도 (U → A → E → A)
  - 각 단계마다 아이콘 (예: 👁️ / 🔍 / ⚖️ / ✅)
  - 단계 간 화살표

## Layout Notes
- 취소선 문장은 상단에 작게, 대비를 주기 위해 회색 처리
- 4단계는 수평으로 배치, 각 단계 동일한 너비
- 각 단계의 색상은 구분 가능하도록 다른 색조 사용
