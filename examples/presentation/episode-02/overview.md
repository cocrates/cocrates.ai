# Presentation Definition — Episode 02

## Topic (S1)
The Unexamined Code Is Not Worth Generating: 검토되지 않은 산출물은 생성할 가치가 없다

## Purpose
이 에피소드는 시리즈를 관통하는 핵심 원칙을 선언한다: **"The unexamined code is not worth generating."** AI가 생성한 모든 산출물은 검토(이해→분석→평가→승인)를 거쳐야 비로소 가치를 가진다.

- **검토의 4단계**를 정의한다: 이해(Understand) → 분석(Analyze) → 평가(Evaluate) → 승인(Approve)
- **"Code"의 확장된 정의**를 제시한다: 소스 코드뿐 아니라 AI가 만드는 모든 산출물(보고서, 슬라이드, 블로그, 학습 노트)이 검토 대상이다
- **무지의 제거(Harnessing Ignorance)** 가 검토의 궁극적 목적임을 설명한다
- Cocrates Harness가 사용자의 무지한 승인을 방지하는 페이스메이커 역할을 함을 소개한다

> 참고: AI-native Engineer 정의는 `presentation/docs/ai-native-engineer.md` 참조. 검토(U→A→E→A)는 AI-native Engineer의 핵심 역량이며, 이 에피소드는 그 원칙을 선언한다.

## Audience
- **주 시청자:** 소프트웨어 개발자(초급~중급), AI 도구를 업무에 사용 중인 사람
- **함께 볼 시청자:** AI 도구를 활용하는 모든 지식 노동자 (Code의 확장 정의와 연결)
- **사전 지식:** Ep1 시청 또는 "AI-assisted vs AI-native" 개념에 대한 기본 이해
- **시청 동기:** "AI가 만들어준 결과물을 그냥 쓰면 안 되나?"라는 의문, 또는 "검토한다는 게 구체적으로 뭘 의미하지?"라는 궁금증

## Logistics
- **형식:** 온라인 교육용 동영상 (녹화/배포)
- **예상 길이:** 약 10분
- **톤/스타일:** 선언적이면서도 친근한 대화체. 소크라테스 명언에서 영감을 받은 철학적 무게감을 유지하되, 블로그 시리즈 제작 예시로 현실감을 더함.
- **슬라이드 구성 (고정 템플릿):**
  1. 타이틀 페이지 (이전 에피소드 맥락 언급)
  2. 학습 목표
  3~N. 에피소드 내용 (블로그 스토리라인에 따라 유연)
  N+1. 학습 목표 확인 (소크라테스식 질문/미션)
  N+2. 다음 에피소드 연결

## Success Criteria
- 시청자가 **"The unexamined code is not worth generating"** 원칙을 이해하고 설명할 수 있다
- 시청자가 **"Code"의 확장된 정의**를 이해한다 (소스 코드 외 모든 산출물)
- 시청자가 **검토의 4단계(U→A→E→A)** 를 설명할 수 있다
- 시청자가 **무지의 제거(Harnessing Ignorance)** 가 검토의 궁극적 목적임을 이해한다
- 시청자가 Cocrates Harness가 무지한 승인을 방지하는 도구임을 인식한다

## Content Notes

### 흐름 제안 (12슬라이드, ~10분)

1. **타이틀 페이지** — Ep1 복습: "AI-assisted vs AI-native 차이를 알게 됐다. 그렇다면 AI-native의 핵심 역량은?"
2. **학습 목표** — (1) 핵심 원칙 이해 (2) Code의 확장 정의 (3) U→A→E→A 검토
3. **선언: "The Unexamined Code Is Not Worth Generating"** — 시리즈의 심장
4. **소크라테스의 영감** — "성찰하지 않는 삶은 살 가치가 없다"의 연결
5. **'Code'의 확장** — 소스 코드, 보고서, 슬라이드, 블로그, 학습 노트
6. **뼈대(Architecture)의 중요성** — 블로그 시리즈 제작 4단계 예시
7. **검토의 진짜 본질** — 이해(U) → 분석(A) → 평가(E) → 승인(A)
8. **무지의 제거 (Harnessing Ignorance)** — 검토의 궁극적 목적
9. **Cocrates Harness의 리듬** — 깐깐한 페이스메이커, 가드레일
10. **핵심 요약** — 4가지 포인트
11. **학습 목표 확인** — 소크라테스식 질문
12. **다음 에피소드 연결** — Cocrates Harness 설치 (Ep3)

## Constraints & Taboos
- **범위 안:** 원칙 선언, Code 확장 정의, U→A→E→A 설명, 블로그 예시, 무지의 제거, Cocrates 리듬 소개
- **범위 밖:** Cocrates Harness의 구체적 설치 방법(Ep3), Learning/Generation 파이프라인 상세(Ep4~12), 다른 도구와의 비교
- **금지:** "AI를 쓰지 마라"는 부정적 메시지. AI의 가치는 인정하되, 검토 없는 사용의 위험성을 지적하는 방향
- **기술적 상세:** 블로그 시리즈 제작 예시(overview → outline → episodes.md → 본문)는 구체적이되, 각 단계의 상세한 설명은 피함
