# Script: Slide 06 — 뼈대(Architecture)의 중요성

**Time:** ~1분

**[SLIDE 06: 4단계 수직 흐름 다이어그램. 각 단계: (1) 전체 그림 설계 → 🔍 검토 → ✅ 승인, (2) 각 에피소드 뼈대 → 🔍 검토 → ✅ 승인, (3) 상세 설계 → 🔍 검토 → ✅ 승인, (4) 본문 생성 → 🔍 검토 → ✅ 승인]**

자, 그럼 구체적으로 어떻게 하는 걸까요?

Cocrates의 접근법을 이해하기 위해,
실제 예시를 하나 들어볼게요.

여러분이 지금 보고 계신 이 **블로그 시리즈** 자체가
바로 Cocrates의 방식으로 만들어지고 있습니다.

(다이어그램을 따라가며)

첫 번째 단계. 전체 그림을 설계합니다.
overview.md라는 파일로 시리즈 전체의 구조를 잡고,
**검토하고, 승인**받습니다.

두 번째 단계. 각 에피소드의 뼈대를 만듭니다.
outline.md로 구조를 잡고, 다시 **검토하고, 승인**받습니다.

세 번째 단계. 상세 설계를 합니다.
각 에피소드의 내용을 구체화하고, 또 **검토와 승인**.

마지막으로 본문을 생성하고, 최종 **검토와 승인**.

(손을 모으며)

보이시나요?
모든 단계에서 **검토(🔍)와 승인(✅)** 이 들어갑니다.
Cocrates는 절대 "일단 만들어 놓고 보자"를 허용하지 않습니다.
**"뼈대를 검토하는 1시간이, 일주일짜리 버그 수정을 막아준다"**
이것이 Cocrates의 방식입니다.
