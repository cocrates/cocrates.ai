# Script: Slide 08 — 지속적 개선의 원칙 — The Unexamined Code

**Time:** ~1분 30초

**[SLIDE 08: 상단 — "The unexamined code is not worth generating" 인용문 강조 / 중단 — 3가지 원칙 카드 / 하단 — 핵심 메시지 요약 상자]**

여러분, 이 시리즈의 첫 에피소드에서부터 우리를 관통해 온 한 문장이 있습니다.

(슬라이드 상단을 가리키며, 천천히)

**"The unexamined code is not worth generating."**
**검토되지 않은 산출물은 생성할 가치가 없다.**

지금까지 우리는 이 원칙을 AI가 생성하는 최종 산출물에 적용해왔습니다. 그런데 오늘 우리는 이 원칙이 **Cocrates Harness 자신에게도 적용**된다는 사실을 이야기하고자 합니다.

(청중을 보며)

Cocrates가 완벽하지 않아서 불편하신가요?

(잠시 멈추고, 단호한 톤으로)

**그럼 고치면 됩니다.**

AI-native Engineer는 단순한 사용자가 아닙니다. 우리 팀 도구의 지속적 개선자, Continuous Improver입니다.

(슬라이드 중단을 가리키며)

한 번에 완벽한 스킬을 만들려는 욕심은 버리세요. 끊임없이 사용하고, 검토하고, 작은 개선을 굴려 나가는 끈기가 필요합니다.

"지금보다 나은 내일의 Cocrates" — 이것이 지속적 개선의 정신입니다.

(슬라이드 하단을 가리키며, 집약하는 톤으로)

**오늘 에피소드의 핵심을 한 문장으로 정리하겠습니다.**

Cocrates는 AI-native Engineer인 여러분과 AI 팀원이 함께 일하는 도구입니다.

하지만 사용하는 것으로 충분하지 않습니다.
Cocrates Harness를 우리 팀이 일을 제대로 하는 도구로 지속적으로 진화시켜야 합니다.

이것이 바로 Harnessing Ignorance의 실천입니다. 무지를 활용하여 나와 우리 팀의 도구를 계속 개선하는 것.

(시리즈 3원칙을 하나씩 짚으며)

**질문하라. 검토하라. 개선하라.**

이 세 가지가 시리즈 전체를 관통하는 원칙입니다.
