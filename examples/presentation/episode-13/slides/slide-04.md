# Slide Content Specification — Slide 4

## Slide 4: 검토와 승인의 습관화 — 구조 검토에 집중하라

**Section:** 본론 1 — 사용자의 진화
**Type:** 개념 설명 + 경고
**Time:** ~1분 30초

### Visual Design
- **Background:** 시리즈 공통 템플릿 (약간 어두운 톤 — 경고/강조 분위기)
- **Layout:** 3단 수직 분할
  - 상단: PR vs AI 검토 비교표
  - 중단: 건축 설계도 비유 + 3가지 구조 중심 질문
  - 하단: "구조 검토 → 세부 검토" 흐름 다이어그램

### Content Elements
| 요소 | 내용 | 비고 |
|------|------|------|
| 제목 | ✅ 검토와 승인의 습관화 — **구조 검토에 집중하라** | 상단, 굵은 강조 |
| PR vs AI 비교 | **코드 PR:** 한 줄 한 줄 꼼꼼히 검토 ↔ **AI 결과물:** "그냥 써볼까?" 하고 복붙 → **이중 잣대를 깨라** | 2열 비교표, 화살표로 충돌 표시 |
| 구조 중요성 (박스) | 🎯 **더 중요한 것: 구조(Architecture)의 검토** | 강조 배경 박스 |
| 구조 중요성 내용 | • 구조는 최종 산출물의 품질을 결정한다<br>• 구조 변경의 영향이 크다<br>• 따라서 최종 산출물 검토보다 **구조의 검토와 승인이 더욱 중요하다** | 3줄 불릿 |
| 구조 질문 1 | ❓ **이 구조가 요구사항을 가장 잘 충족하는가?** (구조의 적합성) | 질문 카드 1 |
| 구조 질문 2 | ❓ **다른 구조적 대안은 없었는가? 그 대안보다 나은 이유는?** (구조의 tradeoff) | 질문 카드 2 |
| 구조 질문 3 | ❓ **이 구조를 변경해야 한다면, 영향 범위는 어디까지인가?** (구조 변경의 영향도) | 질문 카드 3 |
| 비유 | 🏗️ 건축 설계도(구조) 검토 vs 완공된 건물(최종 산출물) 검토. 설계도가 잘못되면 아무리 마감을 잘해도 건물은 흔들린다. | 건축 설계도 아이콘 + 텍스트 |
| 흐름 다이어그램 | **구조 검토(더 중요)** → **세부 설계 검토** → **최종 산출물 검토** | 좌→우 화살표, 첫 단계 굵게 강조 |
| 포인트 | 검토는 불신이 아니라 책임감이다. 그리고 그 책임감은 **구조에서부터** 시작된다. | 하단 강조 상자 |

### Visual Elements Description
- 구조 중요성 박스: 배경색이 다른 영역, 테두리 강조
- 3가지 질문 카드: 세로 리스트, 각각 체크박스 스타일 아이콘
- 구조→세부→최종 흐름: 굵은 화살표, 첫 단계(구조)가 가장 크고 굵게

### Transitions
- **In:** 이전 슬라이드에서 푸시
- **Out:** 다음 슬라이드로 푸시
- **Notes:** "구조 검토가 더 중요하다"는 메시지에 도달할 때 시각적 강조 효과. 발표자가 건축 비유를 설명할 때 설계도 이미지 등장.
