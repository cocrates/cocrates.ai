# Script: Slide 02 — 학습 목표

**Time:** ~1.5분

**[SLIDE 02: 학습 목표 페이지]**

**도입 질문:** "여러분, AI에게 '이것 좀 만들어줘'라고 요청한 후, 그대로 쓰신 적 있나요?"
**핵심 전제:** AI의 제안이 항상 내 요구와 맞는 것은 아니다.
**4가지 학습 목표:**
1. 구조 기반 생성 파이프라인 4단계 설명
2. AI 제안이 부적절한 3가지 이유 (일반적이다 / 쉽고 간단하다 / 품질 미반영)
3. ADR과 Spec이 함께 살아있는 문서(Living Document) 개념
4. Undocumented ASR 개념과 검증의 중요성

---

자, 먼저 여러분께 한 가지 질문을 드리고 싶습니다.

"여러분, AI에게 '이것 좀 만들어줘'라고 요청해본 적 있으신가요?"

(잠시 멈추며 시청자가 생각할 시간을 준다)

아마 대부분 있으실 겁니다. 그런데 그 결과물을 **검토 없이 그대로 사용**하신 적은 없으신가요?

오늘 발표를 통해 여러분은 이렇게 바뀌게 됩니다.

**첫째,** 구조 기반 생성 파이프라인의 4단계 — ADR, Spec, Generation, Verification — 를 스스로 설명할 수 있게 됩니다.

**둘째,** AI의 제안이 왜 항상 내 요구와 맞지 않는지, 그 이유를 세 가지 측면으로 실제 사례와 함께 말할 수 있게 됩니다. 바로 **일반적**이어서, **쉽고 간단**해서, 또는 **품질을 충분히 반영하지 못해서**입니다.

**셋째,** ADR과 Spec이 단발성 문서가 아니라, 함께 진화하는 **살아있는 문서**라는 개념을 이해하게 됩니다.

**넷째,** Undocumented ASR — 즉 문서화되지 않은 설계 결정이 왜 생기고, 검증이 왜 중요한지 설명할 수 있게 됩니다.

그럼 본격적으로 시작해볼까요?
