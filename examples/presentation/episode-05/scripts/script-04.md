# Script: Slide 04 — ADR ① — 저장 모델 논쟁

**Time:** ~1.5분

**[SLIDE 04: ADR ① 저장 모델 논쟁]**

**상황:** "데이터를 어떻게 저장할까?"
**AI의 제안:** "MongoDB처럼 Collection-Document 구조가 일반적이고 쉽습니다"
→ **핵심 문제: 일반적이다 (Generic)**
**사용자의 브레이크:** "내가 지정한 경로가 그대로 파일명이 되는 Path-Addressable 방식이 필요해"
→ 예: `Set("episode/e1.json")` → 실제 `episode/e1.json` 파일
**결과:** 사용자의 요구대로 경로 매핑 구조로 결정
**💡 교훈:** 일반적이다 ≠ 내 요구에 맞다.

---

첫 번째 논쟁입니다. 데이터를 어떻게 저장할까?

AI가 먼저 제안을 합니다. "MongoDB처럼 Collection-Document 구조가 일반적입니다."

여기서 주목할 점은 AI가 **"일반적"** 이라는 단어를 썼다는 겁니다. AI가 가장 많이 학습한 패턴, 수많은 예제에서 봐온 구조를 제안한 거죠.

하지만 생각해보세요. "일반적"이라는 말은 "보통은 그렇다"는 뜻이지, **"내 상황에 꼭 맞다"** 는 뜻이 아닙니다.

사용자가 브레이크를 밟습니다.

"아니요. 제가 원하는 건 경로를 직접 지정하는 방식이에요. `Set("episode/e1.json")`이라고 하면 실제로 파일 시스템에 `episode/e1.json` 파일이 생기는 그런 구조가 필요해요."

이것이 바로 **Path-Addressable 방식**입니다. AI가 처음에 제안한 Collection-Document와는 완전히 다른 접근법이죠.

결국 사용자의 요구대로 경로 매핑 구조로 결정되었습니다.

여기서 얻는 교훈은 이것입니다. **일반적인 것이 항상 내 요구에 맞는 것은 아닙니다.** AI는 가장 흔한 패턴을 제안했을 뿐입니다. 진짜 사용 맥락을 아는 사람은 오직 **사용자**뿐입니다.
