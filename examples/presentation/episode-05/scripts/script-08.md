**[SLIDE 08: Spec — 결정을 하나의 헌법으로]**

**Spec의 탄생:** 모든 ADR 결정을 하나의 설계 명세서로 통합
**Self-Containment:** `spec/jsondb.md` 하나만 읽으면 전체 구조 파악 가능
- 10개 REST 엔드포인트 기능
- API 시그니처와 CLI 명령어 스펙
- 테스트 요구사항까지 모두 포함
**Spec = 설계의 헌법**
**💡 포인트:** Spec 없이 생성은 있을 수 없다. Spec이 곧 설계의 완성이다.

---

세 번의 ADR 논쟁을 마쳤습니다. 이제 이 결정들을 **하나의 문서로 통합**할 차례입니다.

이것이 바로 **Spec**, 설계 명세서입니다.

모든 ADR 결정을 `spec/jsondb.md`라는 하나의 파일로 통합합니다.

이 Spec의 가장 큰 특징은 **Self-Containment**, 즉 자기 완결성입니다. 이 문서 하나만 읽으면 jsondb의 전체 구조를 파악할 수 있습니다.

어떤 내용이 들어있을까요? 10개 REST 엔드포인트의 기능과 API 시그니처, CLI 명령어 스펙, 그리고 테스트 요구사항까지 — 모든 것이 이 문서 하나에 담겨 있습니다.

(잠시 멈춤)

초기의 모호한 요청 — "jsondb 만들어줘" — 는 이제 잊힙니다. 오직 이 통합된 Spec만이 유일한 진실 공급원, Single Source of Truth가 됩니다.

명심하세요. **Spec 없이 생성은 있을 수 없습니다.** Spec이 곧 설계의 완성입니다.
