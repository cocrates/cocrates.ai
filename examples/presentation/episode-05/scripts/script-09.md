# Script: Slide 09 — ⚡ Generation — 오직 Spec만 보고 달린다

**Time:** ~1.5분

**[SLIDE 09: Generation — 오직 Spec만 보고 달린다]**

**모드:** `spec-driven-generation` 활성화
**원칙:** 초기 프롬프트("jsondb 만들어줘")는 잊는다. **오직 합의된 Spec만** 유일한 법전으로 삼는다.
**결과물 (Go 언어):**
- 코어 라이브러리 Go 파일 7개
- REST API 서버
- CLI 도구
- 단위 테스트 + 통합 테스트
- **컴파일 오류 0개** (go build, go vet 패스)
**핵심:** Spec의 품질이 결과물의 품질을 결정한다.

---

Spec이 준비되었습니다. 이제 생성 단계로 넘어갑니다.

여기서 Cocrates는 `spec-driven-generation` 모드로 전환합니다. 이 모드의 핵심 원칙은 하나입니다.

**초기 프롬프트는 잊는다.**

맨 처음 여러분이 했던 말 — "jsondb 만들어줘" — 는 이제 더 이상 유효하지 않습니다. 오직 합의된 Spec만이 유일한 법전입니다.

그 결과는 어땠을까요?

(잠시 멈춤)

AI는 Spec만 보고 Go 언어로 다음과 같은 결과물을 만들어냈습니다.

코어 라이브러리 Go 파일 7개, REST API 서버, CLI 도구, 그리고 단위 테스트와 통합 테스트까지.

그리고 놀라운 점은 **컴파일 오류가 하나도 없었다**는 겁니다. `go build`와 `go vet`을 모두 통과했습니다.

이것이 의미하는 바는 분명합니다. **Spec을 잘 설계하면 AI가 코드를 잘 만든다.** Spec의 품질이 결과물의 품질을 결정합니다.
