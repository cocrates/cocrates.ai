# Slide 13: 핵심 요약 + 학습 확인

## Key Message
발표의 핵심을 4가지 포인트로 요약하고, 시청자가 스스로 이해를 확인할 수 있는 자기 질문을 제시한다.

## Content
- **Headlines / text:**
  - **4-Point 핵심 요약:**
    1. **🔍 AI의 제안을 검토하라.** 할루시네이션만 조심할 게 아니다. AI의 제안이 부적절한 이유는 다양하다 — 일반적이어서, 쉽고 간단해서, 또는 품질을 반영하지 못해서.
    2. **📄 ADR과 Spec은 함께 살아있는 문서다.** 검증/요구사항 변경 시 구조 설계(ADR)로 돌아가 결정을 재검토한다. 구조는 한 번 정해지면 변하지 않는 것이 아니다.
    3. **✅ 검증이 곧 품질이다.** 72개 검증으로도 모든 걸 잡을 수 없지만, 검증 없이는 그조차 발견할 수 없다.
    4. **🦉 Cocrates는 여러분의 구조 설계 역량을 키우는 도구다.** 올바르게 AI를 사용하고 설계 능력을 키울 수 있도록 안내한다.
  - **학습 확인 (자기 질문):**
    - ❓ 파이프라인 4단계(ADR→Spec→Generation→Verification)를 말할 수 있는가?
    - ❓ AI 제안이 부적절한 3가지 이유를 실제 사례와 함께 설명할 수 있는가?
    - ❓ Undocumented ASR이 무엇이고 왜 발생하는가?
    - ❓ 구조 기반 개발이 Waterfall이 아닌 이유와 ADR+Spec의 진화를 설명할 수 있는가?

## Visual Elements
- 4-Point는 숫자 아이콘(①, ②, ③, ④) 또는 아이콘(🔍, 📄, ✅, 🦉)과 함께
- 학습 확인은 별도 박스로 구분 (시청자가 멈춰서 생각할 시간을 주는 디자인)
- 질문은 물음표 아이콘과 함께

## Layout Notes
- 상단: 4-Point 요약 (2×2 그리드 권장)
- 하단: 학습 확인 (별도 섹션으로 구분)
- 전체적으로 깔끔하고 정리된 느낌
