# Script: Slide 06 — 🔄 Learning 파이프라인 개관

**Time:** ~1분

**[SLIDE 06: Learning 파이프라인 3단계 순환 다이어그램 — Education → Knowledge Capture → Reflection → (Education으로 환류)]**

자, 지금까지 우리는 Cocrates가 왜 질문으로 답하는지, 그 배후에는 어떤 철학이 있는지를 살펴봤습니다. 이제 이 철학들이 실제로 구현된 **Learning 파이프라인**의 전체 구조를 보여드리겠습니다.

이 파이프라인은 3단계로 구성됩니다.

**Education — 탐구와 발견.** Cocrates가 질문과 미션으로 사용자의 생각을 유도하는 단계입니다.

**Knowledge Capture — 핵심과 무지의 기록.** 배운 내용의 핵심과, 무엇보다 "내가 무엇을 몰랐는지"를 기록합니다.

**Reflection — 실전 검증.** 진짜로 이해했는지 기습 검증합니다.

(다이어그램을 가리키며)

그리고 이 구조의 핵심은 **순환**입니다. Reflection에서 Gap이 발견되면, 다시 Education 단계로 돌아가 보완합니다.

이 파이프라인이 Cocrates 학습의 핵심 엔진입니다. 각 단계가 독립적으로 동작하면서도 하나의 선순환을 이룹니다.
