# Script: Slide 07 — 🎯 파이널 미션 — "Git 브랜치 전략 커리큘럼을 짜보라"

**Time:** ~1.5분

**[SLIDE 07: 큰 미션 박스 (강조 테두리). 피라미드 하단에 창조(Create) 단계 강조 표시. 물음표 일러스트]**

세 가지 깨달음을 얻은 유저.
하지만 Education 단계는 아직 끝나지 않았습니다.

Cocrates가 **파이널 미션**을 던집니다.

(말풍선을 가리키며, 힘주어)

"지금까지 배운 내용을 바탕으로,
**신입 개발자를 위한 Git 브랜치 전략
4시간 커리큘럼을 설계해 보세요.**"

(잠시 멈추며)

이 미션은 블룸 분류학의 가장 높은 단계인
**창조(Create)** 를 요구합니다.

단순히 개념을 설명하는 수준(이해)을 넘어,
직접 설계(Create)해야 합니다.

이 미션을 통해 유저는 배운 개념을
완전히 자기 것으로 소화할 수 있었습니다.

(청중을 바라보며, 진지하게)

**"설명할 수 있다"와 "설계할 수 있다"는 다릅니다.**

여러분이 어떤 개념을 안다고 말할 때,
과연 설계할 수 있는 수준인가요?

아니면 그냥 설명할 수 있는 정도인가요?

이것이 **'내가 안다'는 것의 진짜 의미**입니다.

(잠시 침묵, 생각할 시간을 줌)

여러분이라면 이 Git 커리큘럼을 어떻게 설계하시겠습니까?
잠시 상상해보세요.
