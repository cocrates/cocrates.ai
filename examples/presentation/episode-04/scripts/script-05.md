# Script: Slide 05 — 🎯 미션 2 — "매트릭스에 어떻게 매핑될까?" (두 번째 깨달음)

**Time:** ~1.5분

**[SLIDE 05: 왼쪽에 1D 피라미드 → 오른쪽에 2D 매트릭스 표 (Knowledge Dimension 4행 × Cognitive Process 6열). 아래에 `map` 함수 예시 4개 셀 매핑]**

첫 번째 깨달음을 얻은 유저.
Cocrates가 한 단계 더 깊이 파고듭니다.

(연기하듯)

Cocrates가 말합니다.

"블룸 분류학은 2001년에 개정되었습니다.
사실은 한 축이 더 있습니다.
예를 들어 자바스크립트의 `map` 함수를 배울 때...

이 4가지 지식 차원과 6가지 인지 과정을
조합해보면 어떻게 될까요?"

(매트릭스를 가리키며)

여기서 엄청난 반전이 일어납니다.

블룸 분류학은 우리가 알던 1차원 피라미드가 아니었습니다.
**2차원 매트릭스**였던 겁니다.

**세로축 — Knowledge Dimension (지식 차원):**
사실적 → 개념적 → 절차적 → 메타인지

**가로축 — Cognitive Process (인지 과정):**
기억 → 이해 → 적용 → 분석 → 평가 → 창조

이렇게 4 × 6 = **24개 셀**의 매트릭스입니다.

(map 예시를 가리키며)

예를 들어 `map` 함수 하나만 봐도 이렇게 입체적으로 쪼개집니다.

**사실적 지식:** `map` 함수의 기본 문법을 암기한다 (기억).
**개념적 지식:** 원본 배열과 입력 함수의 유기적 관계 원리를 파악한다 (이해).
**절차적 지식:** 실무 프로젝트에서 직접 `map` 코드를 구현한다 (적용).
**메타인지 지식:** "난 `map`을 쓸 때 this 바인딩 실수를 자주 하더라"며
내 코딩 습관을 인지한다 (분석).

(잠시 멈추며)

저를 포함해 많은 분이 이 2차원 매트릭스의 존재를
모르고 1차원 피라미드만 알고 계셨을 겁니다.

내가 안다고 생각한 것이,
사실 진짜 '아는 것'이 아니었음을 깨닫는 순간.

이것이 두 번째 깨달음입니다.
