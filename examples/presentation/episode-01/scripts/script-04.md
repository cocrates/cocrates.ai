# Script: Slide 04 — AI-assisted Engineer — A님의 사례

**Time:** ~1분

**[SLIDE 04: 좌측에 "A님의 하루"라는 타임라인. 아침: "로그인 모듈 추가해줘" → 점심: "복붙 완료, 잘 돌아감" → 일주일 후: "보안 리스크 발견!" → "밤샘 재작업". 우측에 "결과: 코드↑ 이해도↓"라는 요약]**

우리 주변에서 흔히 볼 수 있는 사례, A님부터 살펴볼게요.

A님은 바쁜 주니어 개발자입니다. 오늘도 마감에 쫓기고 있네요. 급한 마음에 AI에게 요청합니다. "로그인 모듈 추가해줘."

AI가 순식간에 그럴듯한 코드를 내놓습니다. A님은 깊은 고민 없이 그대로 프로젝트에 붙여넣습니다. "AI가 잘 짰겠지, 뭐." 당장은 잘 돌아가니 다행이다 싶어 가슴을 쓸어내리죠.

(목소리 톤을 낮추며)

하지만 진짜 문제는 일주일 뒤에 터집니다.

보안 리뷰에서 세션 관리에 취약점이 발견됐다거나, 다른 모듈과 인터페이스가 안 맞는다는 경고가 날아옵니다. 결국 A님은 코드를 이해하지 못한 채 붙여넣었기 때문에, 밤을 새워가며 처음부터 다시 갈아엎어야 했습니다.

코드는 늘어났지만, 정작 A님의 머릿속에는 '모르는 것'만 눈덩이처럼 쌓여갔습니다.

여기서 드는 생각 없으신가요? "아, 나도 가끔 이렇게 하는데..."
